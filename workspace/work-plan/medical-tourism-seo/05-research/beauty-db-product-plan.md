## 1. beauty-db 데이터 자산 분석

### 1.1 보유 데이터 요약

| 데이터셋 | 건수 | 활용 가치 | 고유성 |
|----------|------|-----------|--------|
| 전국 피부과 DB | 4,351개 (웹사이트 3,681개) | 전수 데이터 - 경쟁사 없음 | 극히 높음 |
| 외국인유치기관 매핑 | 424개 | 등록기관 타겟팅 | 높음 |
| 시술 상세 데이터 | 324개 | 콘텐츠 자동 생성 원본 | 극히 높음 |
| 다국어 번역 | 1,150건 (일/중/영/태/베) | 즉시 사용 가능한 다국어 콘텐츠 | 극히 높음 |
| 표준시술 + 시장가격 | 126개 시술 + 149건 가격 | 한/일/중 가격비교 | 유일무이 |
| 국제 가격비교 | 33건 | 가격 경쟁력 마케팅 | 높음 |
| 검색 사전 | 36,410개 | SEO 키워드 매핑 | 극히 높음 |
| 추천 병원 | 84개 (메타데이터 포함) | 큐레이션 기반 추천 | 높음 |

### 1.2 데이터 고유 경쟁력

**핵심 해자 (Moat):**

1. **전수 데이터**: 대한민국 피부과 4,351개 전수 크롤링 -- 이 규모의 구조화된 데이터는 시장에 없음
2. **시술-병원 매핑**: 324개 시술 x 4,351개 병원의 조합 가능성
3. **3개국 가격비교**: 한국/일본/중국 시술 가격을 동시에 비교할 수 있는 데이터셋은 공개된 것이 없음
4. **검색 사전 36K**: 시술명의 동의어/별칭/외래어 매핑 -- 다국어 SEO의 핵심 자산
5. **다국어 전문 콘텐츠**: 의료 시술에 대한 정확한 일/중/영 번역은 제작 비용이 높음 (건당 5-10만원 수준)

**데이터 가치 추정:**

| 항목 | 재생산 비용 추정 |
|------|-----------------|
| 4,351개 병원 크롤링 + 구조화 | 2,000-3,000만원 |
| 324개 시술 상세 데이터 작성 | 1,500-2,000만원 |
| 1,150건 전문 의료 번역 | 1,000-1,500만원 |
| 36,410개 검색 사전 | 500-800만원 |
| 국제 가격비교 수집 | 300-500만원 |
| **합계** | **5,300-7,800만원** |

## 2. 데이터 통합 전략

### 2.1 아키텍처 통합

```
현재 (Phase A):
CheckYourHospital (Next.js) → SEO 진단 15개 항목
                             → 리드 수집 → 이메일 시퀀스

통합 후:
CheckYourHospital (Next.js)
├── SEO 진단 (기존) ─────────────────── 무료 진단 + 유료 최적화
├── beauty-db API (신규) ────────────── 시술/병원/가격 데이터 서빙
│   ├── /api/procedures/:id           시술 상세 (다국어)
│   ├── /api/hospitals/:id            병원 정보 + 시술 목록
│   ├── /api/prices/compare           한/일/중 가격비교
│   └── /api/search/synonyms          검색 사전 조회
├── 콘텐츠 생성 엔진 (신규) ─────────── beauty-db 기반 자동 생성
└── 병원 마케팅 도구 (신규) ─────────── 셀프서비스 SaaS
```

### 2.2 데이터 동기화

- beauty-db (PostgreSQL 70MB) → Supabase로 마이그레이션 또는 read-replica 연결
- 가격 데이터: 분기 1회 업데이트 (수동 + 크롤링 하이브리드)
- 시술 데이터: 신규 시술 추가 시 업데이트 (월 1-2건 예상)
- 병원 데이터: 월 1회 변동 체크 (폐업/신규/웹사이트 변경)

## 3. 확장 상품 기획

### 상품 1: 다국어 시술 페이지 자동 생성 (Content Factory)

**개요:** 병원이 제공하는 시술 목록을 입력하면, beauty-db의 시술 상세 데이터 + 다국어 번역을 기반으로 SEO 최적화된 다국어 시술 페이지를 자동 생성하는 서비스.

**문제:**
- 피부과 홈페이지의 90% 이상이 한국어 전용
- 외국어 페이지가 있어도 기계 번역 수준으로 신뢰도 낮음
- 시술 설명 페이지 1개 제작에 전문 번역 + 디자인 + SEO 최적화로 30-50만원 소요

**솔루션:**
- 병원이 제공하는 시술 10-30개를 선택
- beauty-db의 324개 시술 상세 데이터에서 매칭
- 다국어(일/중/영) SEO 최적화 페이지를 자동 생성
- 구조화 데이터(MedicalProcedure Schema), hreflang, 메타태그 자동 삽입
- 병원 브랜딩에 맞춘 커스터마이징

**데이터 활용:**
- 시술 데이터 324개 (원리, 효과, 부작용, 통증, 다운타임)
- 다국어 번역 1,150건
- 검색 사전 36,410개 (타겟 키워드 자동 매핑)

**산출물 예시:**
```
/procedures/thermage/           (한국어)
/ja/procedures/thermage/        (일본어) ← beauty-db 전문 번역 활용
/zh/procedures/thermage/        (중국어) ← beauty-db 전문 번역 활용
/en/procedures/thermage/        (영어)

각 페이지에 포함:
- 시술 원리 설명 (전문 의료 콘텐츠)
- 기대 효과, 시술 시간, 다운타임, 통증 레벨
- FAQ (검색 사전에서 자주 묻는 질문 자동 생성)
- MedicalProcedure 구조화 데이터
- 가격 범위 (국제 비교 포함, 선택적)
```

**수익 모델:**

| 구분 | 가격 | 비고 |
|------|------|------|
| 기본 패키지 (10개 시술 x 3개 언어) | 150만원 | 30페이지 생성 |
| 표준 패키지 (20개 시술 x 3개 언어) | 250만원 | 60페이지 생성 |
| 프리미엄 패키지 (30개 시술 x 4개 언어) | 400만원 | 120페이지 생성 |
| 월 유지보수 (콘텐츠 업데이트 + 순위 모니터링) | 30만원/월 | 선택 |

**구현 난이도:** 중 (3-4주)
- 콘텐츠 템플릿 엔진 개발
- 시술-병원 매칭 로직
- 다국어 페이지 생성 파이프라인
- 커스터마이징 UI

### 상품 2: 가격 경쟁력 마케팅 도구 (Price Advantage)

**개요:** 한국 시술 가격이 일본/중국 대비 얼마나 저렴한지를 시각적으로 보여주는 마케팅 에셋을 자동 생성하는 서비스.

**문제:**
- 한국 의료관광의 최대 셀링 포인트는 "품질 대비 가격"
- 그러나 대부분 병원이 구체적 비교 데이터 없이 "저렴합니다"만 주장
- 외국인 환자가 가격 비교를 직접 해야 하는 불편

**솔루션:**
- beauty-db의 149건 시장 가격 + 33건 국제 가격비교 데이터 활용
- 병원 맞춤형 가격 비교 인포그래픽 자동 생성
- 다국어 랜딩 페이지에 임베드 가능한 위젯 제공
- SNS 공유용 이미지 자동 생성

**산출물 예시:**
```
써마지 FLX (전체 얼굴):
┌─────────────────────────────────────────┐
│  일본: ¥350,000 (약 320만원)  ████████████████████
│  중국: ¥18,000 (약 350만원)   █████████████████████
│  한국: ₩1,200,000 (120만원)   ████████
│
│  한국에서 최대 65% 절약!
│  "같은 써마지 FLX, 한국이 200만원 이상 저렴합니다"
└─────────────────────────────────────────┘
```

**데이터 활용:**
- 시장가격 149건 (한/일/중)
- 국제 가격비교 33건
- 표준시술 126개

**수익 모델:**

| 구분 | 가격 | 비고 |
|------|------|------|
| 가격비교 위젯 (웹사이트 임베드) | 50만원 (설치) + 10만원/월 | 자동 업데이트 포함 |
| 마케팅 에셋 패키지 (인포그래픽 10종 + SNS 이미지 20종) | 80만원 | 분기 1회 업데이트 |
| 상품 1(Content Factory) 번들 | +30만원 추가 | 시술 페이지에 가격비교 자동 삽입 |

**구현 난이도:** 하 (1-2주)
- 가격비교 차트 컴포넌트
- 인포그래픽 템플릿 (5-10종)
- SNS 이미지 자동 생성 (OG Image 방식)

### 상품 3: 피부과 SEO 콘텐츠 허브 (Beauty SEO Hub)

**개요:** 병원 홈페이지가 아닌, CheckYourHospital 자체 미디어로 운영하는 다국어 시술 정보 허브. 외국인 환자가 시술 정보를 검색할 때 유입 -> 추천 병원으로 연결하는 어필리에이트 모델.

**문제:**
- 외국인 환자가 한국 피부 시술을 검색하면 신뢰할 만한 정보원이 부족
- 기존 정보는 블로그/SNS에 산재 -- 구조화되지 않고 언어도 제한적
- 에이전시 사이트는 편향되어 있고, 병원 자체 사이트는 한국어 전용

**솔루션:**
- beauty-db 전체 데이터를 기반으로 한 피부 시술 백과사전 구축
- 324개 시술 x 3-4개 언어 = 1,000-1,300개 고품질 페이지
- 36,410개 검색 사전으로 롱테일 키워드 커버
- 추천 병원 84개와 연결 (CPA 모델)

**콘텐츠 구조:**
```
checkyourhospital.com/procedures/
├── /thermage-flx/                    ← 시술 상세 (한국어)
│   ├── ?lang=ja                      ← 일본어
│   ├── ?lang=zh                      ← 중국어
│   └── ?lang=en                      ← 영어
├── /potenza-rf-microneedling/
├── /categories/anti-aging/           ← 카테고리 페이지
├── /compare/thermage-vs-ulthera/     ← 시술 비교
├── /prices/thermage-korea-vs-japan/  ← 가격 비교 콘텐츠
└── /hospitals/gangnam/               ← 추천 병원 (CPA)
```

**SEO 전략:**
- 각 시술 페이지가 `[시술명] korea price`, `[시술명] 韓国 費用`, `[시술명] 韩国 价格` 키워드를 타겟
- 36,410개 검색 사전으로 동의어/별칭 키워드도 커버
- 구조화 데이터(MedicalProcedure, FAQPage, HowTo)로 리치 스니펫 확보
- GEO/AEO 최적화: AI 검색에서 한국 피부 시술 권위 소스로 인식

**수익 모델:**

| 구분 | 단가 | 목표 | 월 매출 |
|------|------|------|---------|
| CPA (상담 연결) | 5-10만원/건 | 월 30-50건 | 150-500만원 |
| 프리미엄 리스팅 (추천 병원 상위 노출) | 30-50만원/월 | 20-30개 병원 | 600-1,500만원 |
| 배너 광고 (시술 페이지) | 50-100만원/월 | 5-10개 광고주 | 250-1,000만원 |

**장기 가치:**
- SEO 트래픽이 쌓이면 "한국 피부 시술"의 권위 있는 정보원으로 포지셔닝
- CheckYourHospital의 브랜드 인지도 + SEO 도메인 권위 동시 강화
- 자체 미디어 자산으로서 반복 수익 창출

**구현 난이도:** 중-상 (4-6주)
- 324개 시술 페이지 템플릿 + 자동 생성
- 다국어 라우팅 (i18n)
- SEO 메타데이터 자동화
- 병원 추천 알고리즘
- CPA 트래킹 시스템

### 상품 4: 경쟁 분석 리포트 (Competitive Intelligence)

**개요:** 특정 병원의 SEO/온라인 가시성을 동일 지역 경쟁 병원과 비교 분석하는 유료 리포트. beauty-db의 전수 데이터를 활용한 포지셔닝 분석.

**문제:**
- 병원 원장이 가장 궁금한 것: "내 경쟁 병원은 어떻게 하고 있나?"
- 기존 SEO 진단은 자기 병원만 분석 -- 경쟁 맥락 부재
- 강남/서초 797개 피부과 중 자신의 위치를 모름

**솔루션:**
- 무료 진단(Phase A)에서 수집된 데이터 + beauty-db 전수 데이터 결합
- 동일 지역(예: 강남구), 동일 규모, 동일 시술 카테고리 기준 비교
- 5-10개 경쟁 병원과의 상세 비교 리포트 제공

**리포트 구성:**
```
1. 포지셔닝 맵
   - 내 병원 vs 동일 지역 경쟁 병원 SEO 점수 분포
   - 상위 10%/25%/50% 기준선

2. 항목별 경쟁 비교 (익명화)
   - 기술 SEO: "귀원 52점 / 강남구 상위 25% 기준 68점"
   - 다국어 지원: "경쟁 병원 중 42%가 일본어 페이지 보유"
   - 구조화 데이터: "상위 병원 대비 MedicalProcedure 스키마 미적용"

3. 시술별 키워드 경쟁
   - 주요 시술 키워드에서의 검색 노출 현황
   - 검색 사전 기반 미활용 키워드 기회

4. 갭 분석 + 액션 플랜
   - 우선순위별 개선 항목 3-5개
   - 예상 효과 및 투자 대비 수익
```

**데이터 활용:**
- 병원 4,351개 전수 데이터 (지역/규모별 필터링)
- 외국인유치기관 424개 (동종 비교)
- 웹사이트 3,681개 (기술 SEO 벤치마크)
- 검색 사전 36,410개 (키워드 기회 분석)

**수익 모델:**

| 구분 | 가격 | 비고 |
|------|------|------|
| 기본 경쟁분석 리포트 (1회) | 50만원 | 5개 경쟁 병원 비교 |

### 상품 6: AI 시술 상담 챗봇 (Procedure Concierge)

**개요:** 외국인 환자가 자신의 피부 고민을 입력하면, beauty-db의 시술 데이터를 기반으로 적합한 시술을 추천하고 한국 병원으로 연결하는 AI 챗봇.

**문제:**
- 외국인 환자가 한국 시술을 알아보려면 에이전시에 의존 (수수료 20-40%)
- 시술명을 모르는 경우가 대부분 ("얼굴 주름 없애고 싶어요")
- 언어 장벽으로 직접 병원 상담이 어려움

**솔루션:**
- 324개 시술 데이터를 RAG(검색 증강 생성)로 구축
- 증상/고민 → 시술 추천 → 가격 정보 → 병원 연결 플로우
- 일본어/중국어/영어 네이티브 대화 지원
- LINE/WeChat 봇으로 확장 가능 (Phase D와 연계)

**대화 예시:**
```
환자(일본어): 顔のたるみが気になります。韓国でおすすめの施術はありますか？
봇: 顔のたるみには、以下の施術がおすすめです：

1. サーマジェンFLX (써마지 FLX)
   - 原理：RF エネルギーでコラーゲンを刺激
   - ダウンタイム：ほぼなし
   - 韓国価格：約120万ウォン（日本の約65%OFF）

2. ウルセラ (울쎄라)
   - 原理：HIFU で深部からリフティング
   - ダウンタイム：1-2日の軽い腫れ
   - 韓国価格：約150万ウォン（日本の約60%OFF）

ご希望の施術がありましたら、通訳対応の病院をご紹介できます。
[病院を探す] [もっと詳しく] [価格を比較する]
```

**데이터 활용:**
- 시술 데이터 324개 (RAG 인덱스)
- 다국어 번역 1,150건 (다국어 응답)
- 시장 가격 149건 (가격 정보 제공)
- 검색 사전 36,410개 (증상-시술 매핑)
- 추천 병원 84개 (병원 연결)

**수익 모델:**

| 구분 | 단가 | 비고 |
|------|------|------|
| 병원 웹사이트 임베드 (White-label) | 20만원/월 | 병원 브랜딩으로 제공 |
| CPA (상담 연결) | 3-5만원/건 | CheckYourHospital 자체 채널 |
| LINE/WeChat 봇 구축 (병원별) | 100만원 + 15만원/월 | Phase D 연계 |

**구현 난이도:** 중 (3-4주)
- RAG 파이프라인 구축 (beauty-db → 벡터 DB)
- 다국어 챗 UI
- 병원 연결 로직
- LINE/WeChat 연동 (Phase D에서)

### 상품 7: 병원 데이터 API (B2B Data API)

**개요:** beauty-db의 구조화된 데이터를 API로 제공하는 B2B 데이터 서비스. 의료관광 에이전시, 보험사, 여행사 등이 활용.

**문제:**
- 의료관광 관련 사업자들이 시술/가격/병원 데이터를 자체 수집해야 함
- 구조화된 데이터가 시장에 없어 각자 크롤링/수동 조사
- 데이터 품질과 최신성 유지 비용이 높음

**솔루션:**
- RESTful API로 beauty-db 데이터 제공
- 사용량 기반 과금 (API 콜 수)
- SDK 제공 (JavaScript, Python)

**API 엔드포인트:**
```
GET /api/v1/procedures                    시술 목록 (필터: 카테고리, 언어)
GET /api/v1/procedures/:id                시술 상세 (다국어)
GET /api/v1/procedures/:id/prices         시술 가격 (한/일/중)
GET /api/v1/hospitals                     병원 목록 (필터: 지역, 외국인유치기관)
GET /api/v1/hospitals/:id                 병원 상세
GET /api/v1/search/synonyms?q=            검색 사전 조회
GET /api/v1/prices/compare                국제 가격비교
```

**타겟 고객:**
- 의료관광 에이전시 (자체 플랫폼에 데이터 통합)
- 여행 플랫폼 (의료관광 섹션 구축)
- 보험사 (해외 시술 가격 참조)
- 리서치 기관 (시장 분석)

**수익 모델:**

| 플랜 | 가격 | API 콜 | 포함 데이터 |
|------|------|--------|-------------|
| Starter | 30만원/월 | 10,000콜 | 시술 + 가격 |
| Business | 80만원/월 | 50,000콜 | + 병원 + 검색사전 |
| Enterprise | 협의 | 무제한 | + 커스텀 데이터 + SLA |

**구현 난이도:** 중 (3-4주)
- API 게이트웨이 (rate limiting, 인증)
- API 문서 (Swagger/OpenAPI)
- SDK 개발
- 사용량 트래킹 + 과금
