# WeddingCraft - 종합 사업화 계획 요약

## Executive Summary

### 한줄 요약

**기존에 직접 개발한 프리미엄 모바일 청첩장 프로젝트를, 섹션 기반 에디터 SaaS로 전환하여 사업화한다.**

### 핵심 수치

| 항목 | 수치 |
|------|------|
| TAM (전체 시장) | ~30억원/년 (한국 모바일 청첩장) |
| SAM (접근 가능 시장) | ~12억원/년 (프리미엄 세그먼트) |
| SOM Year 1 | ~4,121만원 (월 25-150건, 낮은 가격→높은 전환) |
| SOM Year 3 | ~3.3억원 (월 500-800건) |
| 손익분기점 | **월 23건** (1인 운영, 월 고정비 80만원 기준) |
| MVP 개발 기간 | **13주 + 2주 버퍼** (1인 개발, 약 3.5개월) |
| 초기 투자 | **~0원** (기존 코드 + 무료 티어 인프라) |

---

## 1. 왜 이 사업인가

### 문제

기존 모바일 청첩장 시장은 **획일화된 템플릿 + 제한된 커스터마이징**이 주류. 예비부부의 개성 표현 욕구를 충족시키지 못하고 있다.

### 솔루션

18개 섹션을 자유롭게 조합하는 에디터 + 시장에 유일한 아케이드 모드(RPG 게임 청첩장) + GuestSnap(하객 사진 수집) 등 프리미엄 인터랙티브 기능을 제공한다.

### 최대 자산

이미 **프로덕션 레벨의 18개 섹션 + 아케이드 모드**가 구현되어 있다. 에디터 + 결제 레이어만 추가하면 사업화 가능하다.

---

## 2. 시장 기회

### 긍정 신호

- 2025년 혼인 건수 24만건, **3년 연속 증가** (18년 만에 처음)
- 모바일 청첩장 이용률 85%+ (사실상 필수)
- MZ세대의 "나만의 결혼식" 트렌드 확산
- 프리미엄 청첩장 시장에 강력한 경쟁자 부재

### 경쟁 분석 요약

| 경쟁사 | 강점 | 약점 | WeddingCraft 대비 |
|--------|------|------|--------------------|
| 보자기카드 | 1위 브랜드, 종이+모바일 | 고정 템플릿, 인터랙션 없음 | 에디터+인터랙션으로 차별화 |
| 데어무드 | 완전 무료 | 수익모델 불명, 기능 제한 | 프리미엄 포지셔닝 |
| 필카드 | 셀프 제작 도구 | 디자인/기능 제한 | 품질 격차 |
| 잇츠카드 | 원스톱 웨딩 | 모바일 혁신 부족 | 기술 차별화 |
| 바른손카드 | 50년 전통 | 디지털 혁신 느림 | 디지털 네이티브 강점 |

**핵심 차별화**: 아케이드 모드(시장 유일), GuestSnap(업로드 무제한, 원본 유지), AfterParty(식 후 주변 추천), 비주얼 에디터, 고급 애니메이션

## 3. 비즈니스 모델

### 가격 티어

| 티어 | 가격 | 핵심 포함 기능 |
|------|------|---------------|
| Basic | 19,000원 | 기본 섹션 10개 + 에디터 + 카카오 공유 + GuestSnap |
| Premium | 39,000원 | Basic + 인터뷰/타임라인/3D/BGM + GuestSnap |
| Arcade | 49,000원 | Premium + RPG 게임 청첩장 + GuestSnap |

예상 ARPU: **~41,000원** (티어 믹스 + 애드온, 디코이 전략으로 Arcade 전환 유도)

### GuestSnap 스토리지 정책 (전 티어 공통)
- 업로드: **무제한** (원본 품질 유지, 당일 끊김 없음)
- 다운로드: **200GB까지 무료** (2주 이내)
- 초과분: +5,000원/100GB (다운로드 시 정산)
- 지연 다운로드(2주+): +5,000원
- 보관: 최대 8주 (연장 +3,000원/4주, 최대 24주)

### 단위 경제학

```
ARPU:       41,000원
변동비:      4,845원 (블렌디드 결제수수료 4.5% + 서버 + 스토리지 2,500원)
공헌이익:   36,155원 (마진 88.2%)
목표 CAC:    8,000원 (보수적, 바이럴 안착 시 하락)
LTV:CAC:    5.1x (건강)
```

## 4. 채널 전략: 네이버 스마트스토어 vs 자체 사이트

### 결론: 멀티 채널 (자체 사이트 중심 + 마켓플레이스 보조)

| | 자체 사이트 (메인) | 스마트스토어 | 추가 채널 |
|-|-------------------|-------------|-----------|
| 역할 | 에디터 + 결제 + 발급 통합 | 검색 트래픽 유입 | 추가 노출 |
| 수수료 | 3.5% (결제만) | 5.5-13% | 플랫폼별 상이 |
| 비중 | 70-80% | 15-20% | 5-10% |

**추가 채널 (Phase 2~)**:
- 카카오톡 스토어 — 카카오 생태계 시너지 (공유→구매 원스톱)
- 아이디어스 — MZ 여성 타겟, "수제/커스텀" 포지셔닝
- 크몽 — "청첩장 제작" 카테고리 입점
- ※ 쿠팡은 디지털 상품 판매 불가로 제외

## 5. MVP 범위

### 13주 개발 + 2주 버퍼, Must Have 13개

1. 에디터 - 섹션 on/off
2. 에디터 - 내용 입력 폼
3. 에디터 - 실시간 미리보기
4. 템플릿 렌더링 엔진
5. 결제 시스템 (토스페이먼츠)
6. 고유 URL 발급
7. 카카오톡 공유
8. 기본 섹션 10개 (기존 코드 활용)
9. 동적 OG 이미지 (카카오 공유 바이럴 핵심)
10. GuestSnap 기본 (전 티어 포함, 업로드 무제한)
11. 관리자 대시보드
12. 랜딩페이지
13. 법적 준비 (통신판매업, 약관, 개인정보처리방침)

### 기술 전환

```
현재 (하드코딩 단일 청첩장)
  ↓ 데이터 분리 (JSON 스키마)
  ↓ DB 연동 (Supabase PostgreSQL)
  ↓ 에디터 UI (Zustand + 폼)
  ↓ 결제 (토스페이먼츠)
  ↓ 동적 라우팅 (/invitation/[slug])
목표 (멀티테넌트 SaaS)
```

## 6. GTM 전략

### 제품 = 마케팅

```
1명 구매 → 200-300명 하객 노출 → 3-5명 문의 → 1-2명 전환
바이럴 계수 > 1 가능 (특히 아케이드 모드)
```

### 런칭 5단계

| 단계 | 기간 | 목표 |
|------|------|------|
| Internal | Week 1-2 | 지인 5쌍 무료 테스트 |
| Alpha | Week 3-4 | 사전예약 100명 |
| Beta | Week 5-8 | 50% 할인 유료 20건 |
| Early Access | Week 9-12 | 정가 판매 월 50건 |
| Full Launch | Week 13+ | 월 100건+ |

### 채널 우선순위

1. 자체 사이트 SEO ("모바일 청첩장" 키워드)
2. 인스타그램 (청첩장 쇼케이스 + 릴스)
3. 네이버 스마트스토어 (검색 트래픽)
4. 바이럴 (청첩장 자체가 광고, CAC 0원)
5. 웨딩 커뮤니티 (후기 마케팅)
6. 카카오톡 스토어 (카카오 생태계 시너지)
7. 아이디어스 (MZ 여성, 커스텀 포지셔닝)

## 8. 리스크 & 킬 스위치

### 주요 리스크

| 리스크 | 대응 |
|--------|------|
| 유료 전환 실패 | 가격/기능 실험, 무료 티어 도입 검토 |
| 에디터 UX 실패 | 반복적 사용성 테스트 (5명씩) |
| 경쟁사 빠른 추격 | 아케이드/GuestSnap 기술 해자 유지 |
| 시장 축소 (혼인 감소) | 인접 시장 확장 (돌잔치, 기업 이벤트) |

### 킬 스위치

- 3개월간 누적 30건 미만 판매 (성장 추세 시 1개월 연장)
- 에디터 완성률 20% 미만
- 환불률 20% 이상
- CAC가 12,000원 초과 (ARPU의 30%)
