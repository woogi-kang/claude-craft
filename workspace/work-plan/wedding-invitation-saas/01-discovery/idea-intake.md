# 모바일 청첩장 SaaS - Idea Intake

## 1. 아이디어 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | (가칭) WeddingCraft |
| 한줄 설명 | 개발자가 만든 프리미엄 모바일 청첩장 에디터 플랫폼 |
| 카테고리 | 웨딩테크 / SaaS / 에디터 플랫폼 |
| 플랫폼 | 웹 (모바일 최적화) |

## 2. 문제 정의

### 고객이 겪는 문제

1. **획일화된 디자인**: 기존 청첩장 서비스는 템플릿이 비슷하고, 커스터마이징이 제한적
2. **기능 제한**: 대부분 사진 + 지도 + 계좌번호 수준. 인터랙티브 요소(게임, 애니메이션, 3D)가 없음
3. **낮은 기술 품질**: 성능 최적화 부족, 느린 로딩, 구식 UI/UX
4. **커플의 개성 표현 불가**: "우리만의 이야기"를 담을 수 있는 섹션이 부족

### 현재 대안의 한계

| 대안 | 한계 |
|------|------|
| 기존 청첩장 서비스 (보자기카드, 데어무드 등) | 템플릿 고정, 커스터마이징 제한, 차별화 어려움 |
| 직접 개발 | 개발 역량 필요, 시간/비용 높음 |
| Canva/Wix 등 범용 도구 | 청첩장 특화 기능 부족 (RSVP, 축의금 등) |
| 프리랜서 외주 | 비용 높음 (50만원+), 소통 비용 |

## 3. 솔루션

### 핵심 가치

"**코드 한 줄 없이, 개발자가 만든 것 같은 프리미엄 모바일 청첩장**"

### 솔루션 상세

1. **섹션 기반 에디터**: 18개+ 섹션을 드래그앤드롭으로 on/off, 재배치
2. **프리미엄 인터랙션**: 3D 효과, Framer Motion 애니메이션, 꽃잎 효과 등
3. **유니크 컨텐츠**: 아케이드 모드(RPG 게임 청첩장), 인터뷰 Q&A, 타임라인
4. **실용 기능**: GuestSnap(사진/영상 업로드), RSVP, 축의금 계좌, 카카오 공유

## 4. 기존 프로젝트 자산

### 이미 구현된 섹션 (18개)

| # | 섹션 | 난이도 | 차별화 |
|---|------|--------|--------|
| 1 | Hero | 기본 | 커플 사진 + 결혼 날짜 |
| 2 | Greeting | 기본 | 인사말 + 양가 부모님 |
| 3 | CoupleIntro | 기본 | 커플 소개 |
| 4 | Gallery | 중간 | Embla carousel + lightbox |
| 5 | Interview | 중간 | Q&A 인터뷰 (4쌍) |
| 6 | Timeline | 중간 | 연애 타임라인 마일스톤 |
| 7 | WeddingInfo | 기본 | 날짜/시간/장소/식사 |
| 8 | Location | 중간 | 지도 + 네이버/카카오/티맵 |
| 9 | Account | 기본 | 축의금 계좌 (신랑/신부/양가) |
| 10 | RSVP | 중간 | 참석 여부 (Google Forms) |
| 11 | Guestbook | 중간 | 방명록 (Giscus) |
| 12 | GuestSnap | 높음 | 사진/영상 업로드 (Google Drive, 청크 업로드, IndexedDB) |
| 13 | Share | 기본 | 카카오톡 공유 |
| 14 | Video | 기본 | YouTube 영상 |
| 15 | Footer | 기본 | 푸터 |
| 16 | TerminalIntro | 높음 | 레트로 터미널 인트로 |
| 17 | MusicPlayer | 중간 | 배경 음악 |
| 18 | MouseTrail | 중간 | 꽃잎 커서 효과 |

### 아케이드 모드 (최대 차별화 포인트)

- RPG 게임 형식 청첩장 (7단계)
- 포켓몬 스타일 배틀
- 세이브/로드, 캐릭터 스탯
- 시장에 동일한 제품 없음

### 기술 스택

- Next.js 16.1.1 (App Router, React 19)
- TypeScript, Tailwind CSS v4, Framer Motion
- Three.js (3D), Firebase, Google Drive API
- Playwright E2E 테스트
