# 모바일 청첩장 SaaS - 비즈니스 모델 & 가격 전략

## 1. 비즈니스 모델

### 수익 모델: 티어드 원타임 + 애드온

1회성 구매 상품이므로 SaaS 구독 모델이 아닌 **티어드 일회 결제 + 옵션 애드온** 모델이 적합하다.

### 가격 티어 설계

| 티어 | 가격 | 타겟 | 핵심 기능 | 포지셔닝 |
|------|------|------|----------|---------|
| **Basic** | 19,000원 | 깔끔한 청첩장 원하는 커플 | 기본 섹션 10개 + 에디터 + 카카오 공유 + GuestSnap | 2만원 미만, 진입 장벽 최소화 |
| **Premium** | 39,000원 | 차별화 원하는 커플 | Basic + 인터뷰/타임라인/3D효과/BGM + GuestSnap | 가장 인기 티어 (앵커) |
| **Arcade** | 49,000원 | 최고의 유니크함 원하는 커플 | Premium + 아케이드 모드 (RPG 게임 청첩장) + GuestSnap | 시장 유일, 5만원 미만 |

### GuestSnap 스토리지 정책 (전 티어 공통)

| 항목 | 정책 |
|------|------|
| 업로드 | **무제한** (원본 품질 유지) |
| 무료 다운로드 | **200GB까지**, 업로드 후 2주 이내 |
| 초과분 다운로드 | 200GB 초과분에 대해 **+5,000원/100GB** (다운로드 시점에 결제) |
| 지연 다운로드 | 2주 경과 후 다운로드 시 **+5,000원** |
| 보관 기간 | 최대 8주 (연장: +3,000원/4주, 최대 24주) |
| 자동 삭제 | 8주 후 자동 삭제 (W6, W7에 사전 알림 발송) |

> 결혼식 당일 업로드가 끊기지 않도록 업로드는 무제한으로 허용하고, 초과분은 다운로드 시점에 정산한다.

**다운로드 과금 기준 명확화**:
- 200GB는 **업로드된 총 용량** 기준 (다운로드 트래픽 아님)
- 신랑신부가 전체 데이터를 한 번에 다운로드하는 것이 기본 시나리오
- 하객 개별 다운로드는 제공하지 않음 (신랑신부가 공유)

**악용 방지**:
- 결제된 청첩장에 연결된 GuestSnap만 업로드 가능 (청첩장 URL 검증)
- 업로드는 결혼식 전후 2주로 제한 (총 4주 업로드 윈도우)
- 1인당 업로드 횟수/용량 제한 (사진 50장, 영상 5개)

**환불 정책**:
- 청첩장 URL 발급 전: 100% 환불
- URL 발급 후, 결혼식 7일 전까지: 70% 환불
- 결혼식 7일 전 이후: 환불 불가
- 예상 환불률: 5-10% (디지털 상품 평균 수준)
- 환불 매출 영향: ARPU x 환불률 = 41,000 x 7.5% ≈ 3,075원/건 손실 → 실효 ARPU 약 37,925원

**평생소장 vs GuestSnap 보관 구분**:
- "평생소장" 애드온: 청첩장 웹페이지의 영구 접근 (기본은 예식일+60일 후 비활성화)
- GuestSnap 보관: 업로드된 사진/영상 파일의 서버 보관 기간 (별도 정책)

### 애드온 옵션

| 옵션 | 가격 | 설명 |
|------|------|------|
| 평생소장 | +10,000원 | 사용 기간 무제한 (기본: 예식일+60일) |
| GuestSnap 초과 다운로드 | +5,000원/100GB | 200GB 초과분 다운로드 시 |
| GuestSnap 지연 다운로드 | +5,000원 | 2주 경과 후 다운로드 |
| GuestSnap 보관 연장 | +3,000원/4주 | 8주 이후 추가 보관 (최대 24주) |
| 프리미엄 BGM 팩 | +5,000원 | 저작권 해결된 10곡 추가 |
| 추가 갤러리 사진 | +3,000원 | 20장 -> 50장 |
| 커스텀 도메인 | +8,000원 | yourname.weddingcraft.kr |

### 예상 객단가

```
Basic:    19,000원 x 20% = 3,800원
Premium:  39,000원 x 40% = 15,600원
Arcade:   49,000원 x 40% = 19,600원 (Premium과 1만원 차이 → 디코이 효과로 Arcade 전환 유도)

가중 평균 객단가: 39,000원

애드온 평균 추가 매출: ~2,000원 (25%가 평균 8,000원 구매 → 전체 평균 2,000원)

실효 ARPU: ~41,000원

※ Premium과 Arcade 가격 차가 1만원으로 의도적으로 좁게 설계.
  "1만원 더 내면 게임 모드까지" → Arcade 전환을 유도하는 디코이(Decoy) 전략.
  Premium은 실질적으로 Arcade의 앵커 역할.
```

## 2. 단위 경제학 (Unit Economics)

### 고객당 수익/비용

| 항목 | 금액 | 비고 |
|------|------|------|
| ARPU | 41,000원 | 가중 평균 |
| 결제 수수료 (블렌디드) | -1,845원 | 자체 3.5% x 75% + 스마트스토어 8% x 25% = 4.5% |
| 서버/인프라 (per customer) | -500원 | Vercel + Supabase (스케일링 시) |
| GuestSnap 스토리지 (per customer) | -2,500원 | 평균 실사용 50GB x 2개월, Cloudflare R2 기준 |
| **매출총이익** | **36,155원** | **마진 88.2%** |

> **스토리지 비용 산정 근거**: 업로드 무제한이지만 실제 평균 사용량은 50GB 내외 예상 (대부분 하객이 맥스로 올리지 않음). Cloudflare R2 기준 $0.015/GB/월 x 50GB x 2개월 = $1.5 ≈ 2,000원. 안전 마진 포함 2,500원 산정. 대용량 사용자(200GB+)는 초과 다운로드 과금으로 커버.

### 고객 획득 비용 (CAC) 시나리오

| 채널 | CAC 추정 | 월 획득 | 비고 |
|------|---------|---------|------|
| 오가닉 (SEO/블로그) | 0원 | 15-30건 | 시간 투자 필요, 6개월+ 소요 |
| 인스타그램 광고 | 20,000원 | 10-15건 | 웨딩 CPC 800-1,500원, 전환율 2-3% |
| 네이버 검색광고 | 10,000원 | 15-25건 | "모바일 청첩장" 키워드 경쟁 |
| 바이럴 (공유) | 0원 | 10-30건 | 청첩장 자체가 마케팅 |
| 웨딩 커뮤니티/블로그 | 5,000원 | 5-15건 | 후기 작성 제휴 |
| 카카오톡 스토어/아이디어스 | 0원 (수수료만) | 5-10건 | 플랫폼 수수료로 대체 |

**목표 블렌디드 CAC: 8,000원 이하** (바이럴 비중이 높아질수록 하락)

### LTV:CAC 비율

```
LTV = ARPU = 41,000원 (1회성 상품이므로 LTV = ARPU)
CAC = 8,000원 (목표)
LTV:CAC = 5.1x (건강)

(일반적으로 3x 이상이면 건강한 비즈니스. 바이럴이 안착하면 CAC 하락 → 비율 개선)

(일반적으로 3x 이상이면 건강한 비즈니스)
```

## 4. 손익분기점 분석

### 월간 고정비

| 항목 | 금액 |

| 기타 (도구/서비스) | 10만원 |
| **월 고정비 합계** | **80만원** |

### BEP 계산

```
객단가: 41,000원
변동비 (결제수수료+서버+스토리지): 4,845원
공헌이익: 36,155원

BEP = 고정비 / 공헌이익
BEP = 800,000 / 36,155
BEP = 23건/월

→ 월 23건 판매 시 손익분기
→ 1인 운영 기준, 충분히 달성 가능한 목표
→ 단, 창업자 인건비(기회비용)는 미포함. 이를 포함 시(월 300만원 가정) BEP = 106건/월
```

## 5. 네이버 스마트스토어 vs 자체 사이트 전략

### 비교 분석

| 항목 | 네이버 스마트스토어 | 자체 사이트 |
|------|-------------------|-----------|
| 초기 구축 비용 | 0원 (무료 입점) | 100-200만원 (결제/인증) |
| 트래픽 | 네이버 검색 트래픽 활용 | 직접 트래픽 확보 필요 |
| 수수료 | 카테고리별 5.5-13% | 결제 수수료만 3.5% |
| 브랜딩 | 네이버 플랫폼 내 제한 | 완전한 브랜드 경험 |
| 고객 데이터 | 제한적 | 완전한 고객 데이터 확보 |
| 에디터 통합 | 불가 (주문 후 별도 제작) | 완전 통합 가능 |
| 결제 UX | 네이버페이 (높은 전환율) | 토스/카카오페이 등 |

### 권장 전략: 듀얼 채널

```
Phase 1 (MVP, Month 1-6):
┌─────────────────────────────────────────────────────────────────┐
│ 자체 사이트 (weddingcraft.kr)                                    │
│ - 에디터 → 미리보기 → 결제 → 발급                               │
│ - 전환 퍼널 최적화                                               │
│ - 결제: 토스페이먼츠                                              │
└─────────────────────────────────────────────────────────────────┘
    +
┌─────────────────────────────────────────────────────────────────┐
│ 네이버 스마트스토어 (보조 채널)                                    │
│ - 상품 등록 (Basic/Premium/Arcade 각각)                          │
│ - 주문 접수 → 자체 사이트 에디터로 안내                           │
│ - 네이버 검색 트래픽 활용                                         │
│ - "모바일 청첩장" 키워드 SEO                                      │
└─────────────────────────────────────────────────────────────────┘

Phase 2 (성장기, Month 7+):
- 자체 사이트 비중 확대 (마진 + 브랜딩)
- 스마트스토어는 트래픽 유입 채널로 유지
- 크몽/숨고 등 추가 채널 테스트
```

### 스마트스토어 운영 방법

디지털 상품이므로 주의할 점:
1. **카테고리**: 디지털 서비스 / 이벤트/행사 서비스
2. **상품 구성**: 티어별 3개 상품 + 애드온 별도 상품
3. **배송**: "디지털 발송" 설정, 구매 후 에디터 링크 발송
4. **반품/환불**: 디지털 상품 특성 고려한 환불 정책 수립
5. **리뷰 관리**: 완성된 청첩장 스크린샷 리뷰 유도

## 6. AfterParty 섹션

결혼식 종료 후 하객에게 주변 정보를 안내하는 섹션. 신랑신부가 에디터에서 직접 등록한다.

### 콘텐츠 구성 (수동 입력)

| 카테고리 | 입력 항목 | 예시 |
|----------|----------|------|
| 뒤풀이 | 장소명, 시간, 네이버/카카오 플레이스 URL | "2차 OO호프 18시" |
| 맛집 추천 | 장소명, 한줄 코멘트, 플레이스 URL | "우리가 매주 가는 파스타집" |
| 카페 추천 | 장소명, 한줄 코멘트, 플레이스 URL | "디저트 맛집, 포토존 있음" |
| 주변 행사/볼거리 | 행사명, 기간, 장소, URL | "여의도 벚꽃축제 (4/1-4/7)" |
| 포토스팟 | 장소명, 설명, URL | "식장 앞 공원, 사진 찍기 좋아요" |

### 노출 조건

| 항목 | 정책 |
|------|------|
| 활성화 시점 | 결혼식 종료 시간 이후 자동 노출 (에디터에서 시간 설정) |
| 식 전 | 섹션 숨김 (청첩장에서 보이지 않음) |
| 티어 | 전 티어 포함 (Premium/Arcade에서 등록 개수 확장 가능) |

> "결혼식 올 때 다들 폼내고 오니까 아까우니까" — 하객 입장에서 실용적이고, 신랑신부 입장에서 센스있는 배려.
