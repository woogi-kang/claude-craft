# WeddingCraft - 신랑신부 대시보드 상세 UI 설계서

## 1. 페이지 개요

| 항목 | 내용 |
|------|------|
| URL | `/dashboard` (홈), `/dashboard/*` (하위 페이지) |
| 렌더링 | SSR (인증 필수, 서버에서 데이터 프리페치) |
| 인증 | Supabase Auth, 미인증 시 `/login`으로 리다이렉트 |
| 목표 | 결제 후 청첩장 관리의 원스톱 허브 |
| 접근 대상 | 결제 완료한 신랑/신부 (draft 상태에서도 제한적 접근 가능) |


### 요약 카드 4개

| 요소 | 스펙 |
|------|------|
| 레이아웃 | 2x2 그리드 (데스크탑), 1열 (모바일) |
| 카드 | `rounded-xl shadow-sm border p-6` |
| 미니 차트 | 7일간 방문자 바 차트 (sparkline), `recharts` mini |
| D-day | 결혼식까지 남은 일수, 결혼식 지나면 "결혼식 +N일" |
| 모바일 | 카드 가로 스크롤 또는 2열 |

| 100%+ | 빨간색 | 초과 (다운로드 시 추가 결제) |

## 14. 데이터 페칭 전략

| 페이지 | 전략 | 캐싱 |
|--------|------|------|
| 홈 (요약) | SSR + TanStack Query | staleTime: 30s |
| 청첩장 관리 | SSR | staleTime: 5min |
| GuestSnap | SSR + 무한 스크롤 | staleTime: 1min |
| 통계 | SSR | staleTime: 5min, 기간 변경 시 CSR |
| RSVP | SSR + 검색/필터 CSR | staleTime: 1min |
| 알림 설정 | SSR | staleTime: 10min |
| 계정/결제 | SSR | staleTime: 10min |

```typescript
// TanStack Query 예시 (홈 요약 데이터)
export function useDashboardSummary(invitationId: string) {
  return useQuery({
    queryKey: ['dashboard', 'summary', invitationId],
    queryFn: () => fetchDashboardSummary(invitationId),
    staleTime: 30 * 1000,  // 30초
    refetchInterval: 60 * 1000,  // 1분마다 자동 리프레시
  });
}
```
