# WeddingCraft - 에디터 상세 UI 설계서

## 1. 페이지 개요

| 항목 | 내용 |
|------|------|
| URL | `/editor` (신규), `/editor/[invitationId]` (기존 편집) |
| 렌더링 | CSR (클라이언트 사이드, 에디터 상태 Zustand) |
| 인증 | 필수 (Supabase Auth, 소셜 로그인) |
| 목표 | 비개발자가 15분 내 청첩장 완성 |
| 자동 저장 | 5초 디바운스, Supabase 실시간 동기화 |

## 4. 왼쪽 편집 패널 (데스크탑)

### 4.1 탭 구조

```
┌────────────────────────────────────────┐
│  [섹션 관리]    [디자인 설정]            │
├────────────────────────────────────────┤
│                                        │
│  (선택된 탭의 콘텐츠)                    │
│                                        │
└────────────────────────────────────────┘
```

| 탭 | 내용 |
|----|------|
| 섹션 관리 | 섹션 목록 (ON/OFF, 순서, 편집 진입) |
| 디자인 설정 | 테마/컬러/폰트 (Phase 2, MVP에서는 기본 테마 고정) |

**섹션 아이템 구조:**

```
┌────────────────────────────────────────┐
│  [⠿ 드래그]  섹션명           [토글]    │
│              섹션 설명                   │
│              상태 표시                   │
└────────────────────────────────────────┘
```

| 요소 | 상세 |
|------|------|
| 드래그 핸들 | `⠿` 6-dot 아이콘, `@dnd-kit/sortable` 사용 |
| 섹션명 | 한글 이름 + 영문 서브텍스트 (e.g. "메인 사진 · Hero") |
| 토글 | ON/OFF 스위치 (활성화/비활성화) |
| 상태 | `● 입력 완료` (초록) / `○ 입력 필요` (노란) / `⊘ 비활성` (회색) |
| 잠금 | 티어 미달 섹션은 잠금 아이콘 + 뱃지 ("Premium" / "Arcade") |
| 클릭 | 섹션 아이템 클릭 시 해당 섹션 편집 폼으로 전환 |
| 프리뷰 스크롤 | 섹션 클릭 시 우측 프리뷰가 해당 섹션으로 자동 스크롤 |

**진행률 바:**
- 활성화된 섹션 중 필수 입력 완료 비율
- 100% 달성 시 "결제하기" 버튼 강조
- 미완성 상태에서도 결제 가능 (결제 후 수정 가능하므로)

### 4.4 섹션별 편집 폼

섹션 목록에서 특정 섹션을 클릭하면, 편집 패널이 해당 섹션의 입력 폼으로 전환된다.

```
┌────────────────────────────────────────┐
│  [← 섹션 목록]  Hero 편집               │
├────────────────────────────────────────┤
│                                        │
│  메인 사진 (필수)                       │
│  ┌──────────────────────────────────┐  │
│  │  [+] 사진 업로드                  │  │
│  │      (드래그 앤 드롭 또는 클릭)    │  │
│  │      권장: 가로 1200px 이상       │  │
│  └──────────────────────────────────┘  │
│                                        │
│  커플 사진 (모바일용)                    │
│  ┌──────────────────────────────────┐  │
│  │  [+] 사진 업로드                  │  │
│  └──────────────────────────────────┘  │
│                                        │
│  표시 문구                              │
│  ┌──────────────────────────────────┐  │
```

## 7. 자동 저장 / 임시 저장 전략

### 저장 메커니즘

```typescript
// Zustand middleware: 자동 저장
const useEditorStore = create(
  persist(
    subscribeWithSelector((set, get) => ({
      // ... store
    })),
    {
      name: 'editor-draft',   // localStorage 키
      storage: createJSONStorage(() => localStorage),
    }
  )
);

// 디바운스 저장 (서버)
const debouncedSave = useDebouncedCallback(
  async (data: InvitationData) => {
    await supabase
      .from('invitations')
      .update({ sections: data.sections, updated_at: new Date() })
      .eq('id', data.id);
    set({ saveStatus: 'saved' });
  },
  5000 // 5초 디바운스
);

// 변경 감지 시 자동 트리거
useEffect(() => {
  const unsub = useEditorStore.subscribe(
    (state) => state.invitation,
    (invitation) => {
      set({ saveStatus: 'unsaved' });
      debouncedSave(invitation);
    }
  );
  return unsub;
}, []);
```

### 저장 전략 계층

| 레벨 | 저장소 | 주기 | 용도 |
|------|--------|------|------|
| L1 | Zustand (메모리) | 즉시 | 실시간 프리뷰 반영 |
| L2 | localStorage | 1초 디바운스 | 브라우저 새로고침 복구 |
| L3 | Supabase DB | 5초 디바운스 | 영구 저장, 기기 간 동기화 |

### 충돌 방지

| 시나리오 | 대응 |
|----------|------|
| 여러 탭에서 동시 편집 | `updated_at` 타임스탬프 비교, 최신 우선 |
| 오프라인 편집 | localStorage에 큐잉, 온라인 복귀 시 동기화 |
| 세션 만료 | 30분 비활동 시 경고, 자동 저장 후 로그아웃 |

### 저장 상태 표시

```

┌──────────────────────────────┐
│  ✓ 저장됨 (3초 전)           │  ← 초록색, 평상시
│  ○ 저장 중...                │  ← 노란색, 저장 진행
│  ● 변경사항 있음              │  ← 주황색, 아직 저장 안 됨
│  ✕ 저장 실패 (재시도)        │  ← 빨간색, 에러
└──────────────────────────────┘
```

## 9. 에디터 상태 관리 (Zustand Store)

```typescript
interface EditorStore {
  // 청첩장 데이터
  invitation: InvitationData;

  // UI 상태
  activeTab: 'sections' | 'design';
  activeSection: string | null;       // 현재 편집 중인 섹션 ID
  selectedTier: 'basic' | 'premium' | 'arcade';
  saveStatus: 'saved' | 'saving' | 'unsaved' | 'error';
  lastSavedAt: Date | null;
  previewZoom: number;                // 0.5 | 0.75 | 1
  mobileView: 'edit' | 'preview';    // 모바일 탭 상태

  // 편집 진행률
  completionRate: number;             // 0-100
  incompleteSections: string[];       // 미완성 섹션 ID 목록

  // 히스토리 (되돌리기/다시하기)
  history: InvitationData[];
  historyIndex: number;

  // Actions
  setActiveSection: (sectionId: string | null) => void;
  toggleSection: (sectionId: string) => void;
  reorderSections: (from: number, to: number) => void;
  updateSectionData: (sectionId: string, field: string, value: unknown) => void;
  setTier: (tier: 'basic' | 'premium' | 'arcade') => void;
  undo: () => void;
  redo: () => void;
  save: () => Promise<void>;
  calculateCompletion: () => void;

  sections: Record<string, SectionData>;

  theme: ThemeConfig;

  data: Record<string, unknown>;
```

## 11. 에러 / 엣지 케이스 처리

| 시나리오 | 대응 |
|----------|------|
| 이미지 업로드 실패 | 재시도 버튼 + 에러 메시지, 3회 자동 재시도 |
| 네트워크 끊김 | "오프라인 모드 — 변경사항은 로컬에 저장됩니다" 배너 |
| 세션 만료 | 모달 "세션이 만료되었습니다. 다시 로그인해주세요." + 현재 상태 localStorage 보존 |
| 브라우저 뒤로가기 | "편집 중인 내용이 있습니다. 나가시겠습니까?" (`beforeunload`) |
| 필수 필드 미입력 후 결제 | 미완성 섹션 하이라이트 + "아직 입력하지 않은 항목이 있어요. 결제 후에도 수정할 수 있습니다." |
| 동시 편집 (여러 기기) | 마지막 저장 우선, 충돌 시 토스트 알림 |
