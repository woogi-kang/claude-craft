## 6. 사용자 리서치 계획

### 6.1 검증해야 할 가설

| # | 가설 | 검증 방법 | 샘플 수 |
|---|------|-----------|---------|
| H1 | 개발자들은 문서화에 주 2시간 이상 소비한다 | 설문조사 | 100명 |
| H2 | Swagger 설정이 가장 큰 진입 장벽이다 | 심층 인터뷰 | 10명 |
| H3 | 코드 주석 기반 문서화를 선호한다 | A/B 테스트 | 50명 |
| H4 | 월 $10-30 가격대가 적정하다 | 가격 민감도 테스트 | 30명 |

### 6.2 인터뷰 가이드 (요약)

**스크리닝 질문**
1. 현재 API 개발을 하고 계신가요?
2. API 문서화를 직접 하시나요?
3. 어떤 문서화 도구를 사용하시나요?

**핵심 질문**
1. 가장 최근에 API 문서를 작성/수정한 게 언제인가요?
2. 그때 어떤 어려움이 있었나요?
3. 이상적인 문서화 도구는 어떤 모습일까요?
4. 문서화에 한 달에 얼마까지 지불할 의향이 있나요?

### 6.3 리서치 일정

| 주차 | 활동 | 목표 |
|------|------|------|
| 1주차 | 설문조사 배포 | 100명 응답 |
| 2주차 | 심층 인터뷰 | 10명 완료 |
| 3주차 | 분석 및 페르소나 검증 | 인사이트 도출 |
