# DocuFlow - Pricing Strategy

> 작성일: 2026-01-16
> 버전: 1.0
> 상태: Draft

---

## Executive Summary

DocuFlow는 **Value-Based Pricing** 접근법을 채택하여, 개발자가 절약하는 시간 가치를 기준으로 가격을 책정합니다. 핵심 전략은 **Freemium으로 진입 장벽 제거 + Pro/Team으로 수익화**입니다.

---

## 1. 가격 책정 철학

### 1.1 가치 기반 가격 (Value-Based Pricing)

```
┌─────────────────────────────────────────────────────────────────┐
│                     가치 = 가격 책정 기준                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   고객이 느끼는 가치                                              │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                          │   │
│   │   시간 절감: 월 8시간 x $50/시간 = $400 가치              │   │
│   │   품질 향상: 전문 문서 → 개발자 경험 향상                  │   │
│   │   동기화: 항상 최신 문서 → 지원 요청 감소                  │   │
│   │                                                          │   │
│   │   총 가치: ~$500+/월                                     │   │
│   │                                                          │   │
│   └─────────────────────────────────────────────────────────┘   │
│                            ↓                                     │
│                     가격: $15-99/월                              │
│                  (가치의 6-20% 수준)                             │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 가격 책정 원칙

| 원칙 | 설명 | 적용 |
|-----|------|-----|
| **접근성** | 개인 개발자도 사용 가능 | Free 플랜 제공 |
| **확장성** | 팀 규모에 따라 자연스럽게 확장 | 시트 기반 과금 |
| **투명성** | 숨은 비용 없음 | 명확한 기능 구분 |
| **유연성** | 필요에 맞는 선택 | 3+1 플랜 구조 |

## 6. 가격 테스트 계획

### 6.1 A/B 테스트

| 테스트 | 가설 | 지표 |
|-------|------|------|
| Pro $15 vs $39 | 높은 가격도 전환율 유지 | 전환율, MRR |
| Team $99 vs $79 | 낮은 가격이 팀 확대 촉진 | 팀 규모, 확장율 |
| 연간 할인 17% vs 25% | 높은 할인이 연간 전환 증가 | 연간 비율 |

### 6.2 가격 민감도 조사

```
Van Westendorp 모델 적용:
Q1: 너무 싸서 품질 의심? → $10 이하
Q2: 저렴하다고 느끼는 가격? → $20-30
Q3: 비싸지만 수용 가능? → $40-50
Q4: 너무 비싸서 고려 안 함? → $60 이상

최적 가격대: $25-40 → $15 선택
```

## 7. 수익 최적화

### 7.1 Expansion Revenue

```
┌─────────────────────────────────────────────────────────────────┐
│                    Expansion Revenue 전략                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   시트 확장                                                      │
│   ├─ 신규 팀원 온보딩 시 자동 추가                               │
│   └─ 볼륨 할인: 10+ 시트 시 10% 할인                             │
│                                                                  │
│   사용량 확장                                                    │
│   ├─ AI 크레딧 추가 구매                                         │
│   └─ 스토리지 확장                                               │
│                                                                  │
│   플랜 업그레이드                                                │
│   ├─ Free → Pro: 5% 전환율 목표                                 │
│   ├─ Pro → Team: 20% 전환율 목표                                │
│   └─ Team → Enterprise: 5% 전환율 목표                          │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 7.2 Net Revenue Retention (NRR)

```
목표 NRR: 120%

계산:
시작 MRR: $100K
- Churn: -$5K (5%)
+ Expansion: +$25K (25%)
= 종료 MRR: $120K

NRR = 120%
```

## 9. 실행 로드맵

### Phase 1: 베타 (Month 1-3)
- Free 플랜만 제공
- 가격 민감도 설문 진행
- 핵심 기능 검증

### Phase 2: 소프트 런칭 (Month 4-6)
- Pro 플랜 도입 ($15)
- 얼리버드 30% 할인
- 전환율 모니터링

### Phase 3: 풀 런칭 (Month 7+)
- Team 플랜 도입 ($99)
- Enterprise 상담 오픈
- A/B 테스트 시작
