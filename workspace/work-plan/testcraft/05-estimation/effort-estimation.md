## 1. Estimation Methodology

### 산정 방식: T-Shirt Sizing + Story Points

```
+------------------------------------------------------------------+
|                    Estimation Framework                           |
+------------------------------------------------------------------+
|                                                                   |
|   T-Shirt Size    Story Points    실제 시간 (1인 기준)            |
|   ------------    ------------    -------------------------       |
|   XS              1-2             0.5일 (4시간)                   |
|   S               3-5             1일 (8시간)                     |
|   M               8-13            2-3일 (16-24시간)               |
|   L               21-34           1주 (40시간)                    |
|   XL              55+             2주+ (80시간+)                  |
|                                                                   |
|   Buffer: +20% (예상치 못한 이슈, 학습 시간)                       |
|                                                                   |
+------------------------------------------------------------------+
```

### 가정 사항

| 항목 | 가정 |
|-----|------|
| **개발자 수준** | 중급 (3-5년 경험) |
| **풀타임 여부** | 주 40시간 |
| **기술 스택 숙련도** | Next.js 중급, Supabase 초급 |
| **AI/LLM 경험** | 기본 API 사용 경험 |
| **외부 의존성** | 3rd party 서비스 안정적 |

### Weekly Capacity (3인 팀)

```
주당 가용 시간:
  - 개발자 A (Full-stack): 40시간 × 0.8 = 32시간 (80% 집중)
  - 개발자 B (Frontend): 40시간 × 0.8 = 32시간
  - 개발자 C (Part-time/AI): 20시간 × 0.8 = 16시간

  주당 총: 80시간 = 10 Man-days

8주 총: 80시간 × 8 = 640시간 = 80 Man-days

필요 공수: 100 Man-days (버퍼 포함)
→ 약 20% 부족, 일부 기능 축소 또는 추가 인력 필요
```
