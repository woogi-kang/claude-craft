## 1. Unique Value Proposition (UVP)

### 1.1 한 줄 UVP

> **"기획서만 넣으면, 플랫폼별 엣지케이스까지 포함된 테스트케이스가 5분 만에 완성됩니다"**

### 1.2 UVP 구성 요소

| 요소 | 내용 |
|-----|------|
| **대상** | IT 기획자, QA 팀, 외주 개발팀 |
| **문제** | 테스트케이스 수동 작성의 시간 소모 + 엣지케이스 누락 |
| **솔루션** | AI 기반 자동 생성 |
| **차별점** | 플랫폼별 특화 엣지케이스 DB |
| **결과** | 90% 시간 단축 + 품질 향상 |

## 4. Why Now? (지금인 이유)

### 4.1 시장 환경 변화

| 트렌드 | 영향 | 적시성 |
|-------|------|--------|
| **AI/LLM 기술 성숙** | 기획서 파싱 정확도 향상 | 2024년 GPT-4o 이후 실용적 수준 도달 |
| **멀티플랫폼 필수화** | 테스트 복잡도 증가 | 모바일+웹 동시 지원이 기본 |
| **외주 개발 증가** | QA 기준 전달 니즈 | 스타트업의 외주 의존도 증가 |
| **QA 자동화 트렌드** | 시장 인식 형성 | 테스트 자동화에 대한 수용성 증가 |

### 4.2 기술 실현 가능성

```
2020년: NLP 기술 부족 → 기획서 파싱 불가능
2022년: GPT-3 등장 → 가능하나 정확도 부족
2024년: GPT-4/Claude → 실용적 정확도 달성
2026년: 현재 → 비용 효율적 구현 가능
```

### 4.3 경쟁 상황

| 상황 | 분석 |
|-----|------|
| **직접 경쟁자 부재** | 플랫폼별 엣지케이스 특화 서비스 없음 |
| **기존 도구 한계** | TestRail, Zephyr는 관리 도구 (생성 X) |
| **AI 도구 초기** | 범용 AI 테스트 도구는 엣지케이스 약함 |
