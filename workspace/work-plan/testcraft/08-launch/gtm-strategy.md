## 1. GTM Overview

### 전략 요약

```
┌─────────────────────────────────────────────────────────────────┐
│                    GTM Strategy Overview                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   Launch Type: Product-Led Growth (PLG)                         │
│   Primary Market: 한국 스타트업 QA/PM                           │
│   Secondary Market: 글로벌 QA 커뮤니티                          │
│                                                                  │
│   ─────────────────────────────────────────────────────────     │
│                                                                  │
│   Phase 1: Soft Launch (Week 1-2)                               │
│   └─ 베타 테스터 피드백, 버그 수정                              │
│                                                                  │
│   Phase 2: Community Launch (Week 3-4)                          │
│   └─ QA Korea, 디스콰이엇, GeekNews                             │
│                                                                  │
│   Phase 3: Product Hunt Launch (Week 5-6)                       │
│   └─ 글로벌 노출, 초기 사용자 확보                              │
│                                                                  │
│   Phase 4: Growth Mode (Week 7+)                                │
│   └─ 콘텐츠 마케팅, 레퍼럴 프로그램                             │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### GTM Principles

| 원칙 | 설명 | 실행 |
|-----|------|------|
| **Free First** | 무료로 가치 증명 | Free Tier로 진입 장벽 제거 |
| **Community Trust** | 커뮤니티에서 신뢰 구축 | 먼저 가치 제공, 후 제품 소개 |
| **Content Authority** | 콘텐츠로 전문성 어필 | 블로그, 뉴스레터, 무료 리소스 |
| **Word of Mouth** | 입소문 중심 성장 | 추천 프로그램, 바이럴 루프 |

## 3. Launch Checklist

### Pre-Launch Checklist (Week -4 to -1)

#### Product Ready

- [ ] MVP 기능 완성 (7개 핵심 기능)
- [ ] 버그 수정 완료 (Critical 0개)
- [ ] 성능 최적화 (TC 생성 < 2분)
- [ ] 모바일 반응형 확인
- [ ] 결제 시스템 테스트 (Stripe)

#### Marketing Ready

- [ ] 랜딩 페이지 완성
- [ ] 브랜드 가이드라인 확정
- [ ] 소셜 미디어 계정 생성
- [ ] 이메일 템플릿 준비
- [ ] 프레스킷 준비

#### Content Ready

- [ ] 제품 소개 영상 (30초)
- [ ] 스크린샷 세트 (10장)
- [ ] 데모 GIF (3개)
- [ ] 런칭 블로그 포스트 초안
- [ ] FAQ 작성

#### Community Ready

- [ ] QA Korea 슬랙 멤버십
- [ ] 디스콰이엇 프로필 완성
- [ ] GeekNews 계정 생성
- [ ] LinkedIn 프로필 최적화

#### Operations Ready

- [ ] 고객 지원 채널 (이메일, 챗)

## 4. Channel Strategy

### 4.1 Primary Channels (집중 채널)

#### 디스콰이엇

```yaml
채널_역할: 국내 스타트업/PM 타겟

런칭_전략:
  Pre-launch:
    - 프로필 완성 (팀, 스토리)
    - 빌딩 로그 2-3개 선게시

  Launch:
    - 프로덕트 등록
    - 런칭 스토리 공유
    - Q&A 적극 대응

  Post-launch:
    - 주간 빌딩 로그 업데이트
    - 다른 메이커와 네트워킹
    - 피드백 반영 공유

콘텐츠_예시:
  런칭_포스트: |
    "QA 3년차가 만든 테스트케이스 자동 생성 서비스

    [문제]
    기획서 받고 TC 뽑는 데 일주일,
    그러다 기획 바뀌면 또 처음부터...

    [솔루션]
    PRD 업로드하면 30분 안에 TC 생성
    플랫폼별 엣지케이스 65개 자동 포함

    [런칭 특별 혜택]
    첫 100명 Early Bird 40% 할인"

KPI:
  - 북마크: 100+
  - 팔로워: 200+
  - 가입 전환: 50명/월
```

#### QA Korea 커뮤니티

```yaml
채널_역할: 핵심 타겟 (QA 엔지니어) 직접 접근

접근_전략:
  Phase 1 (신뢰 구축):
    - 질문에 적극 답변
    - 무료 리소스 공유 (엣지케이스 체크리스트)
    - 자연스러운 대화 참여

  Phase 2 (런칭):
    - "직접 만든 도구" 소개
    - Early Bird 특별 혜택
    - 피드백 요청

  Phase 3 (성장):
    - 성공 사례 공유
    - 커뮤니티 전용 할인
    - AMA (Ask Me Anything) 세션

콘텐츠_예시:
  티저: |
    "요즘 TC 작성 시간 줄이려고 AI 도구 만들고 있는데,
    혹시 가장 자주 놓치는 엣지케이스가 뭔가요?

    저는 iOS 앱 전환 시 상태 유지 테스트를 자주 놓쳤어요 ㅠㅠ"

  런칭: |
    "[공유] PRD → TC 자동 생성 도구 만들었습니다

    QA 하면서 가장 시간 많이 들었던 게 TC 작성이라
    직접 만들어봤습니다.

    - PRD 업로드하면 30분 내 TC 생성
    - Android/iOS/Web 플랫폼별 엣지케이스 자동 포함

    QA Korea 멤버분들께 첫 달 무료 + 평생 30% 할인 드려요!
    피드백 주시면 적극 반영하겠습니다."

KPI:
  - 반응 (좋아요/댓글): 50+
  - 가입 전환: 30명
  - 유료 전환: 5명
```

#### Product Hunt

```yaml
채널_역할: 글로벌 노출, 초기 사용자 대량 확보

런칭_준비:
  - Hunter 섭외 (팔로워 1,000+ 한국 IT 인플루언서)
  - 런칭 날짜: 화요일 00:01 PST (한국 시간 화요일 17:01)
  - 제품 이미지/영상 준비
  - 첫 댓글 준비 (Maker 소개)

런칭_콘텐츠:
  Tagline: "Turn PRDs into Test Cases in 30 Minutes with AI"

  Description: |
    TestCraft automatically generates test cases from your
    PRD documents using AI.

    Key Features:
    - PRD upload (PDF, MD, DOCX)
    - Platform-specific edge cases (Android, iOS, Web)
    - CSV export for TestRail, Jira

    Save 80% of your QA time!

  First_Comment: |
    "Hi Product Hunt!

    I'm [Name], founder of TestCraft.

    As a QA engineer for 5 years, I spent 70% of my time
    writing test cases manually. So I built TestCraft.

    Would love your feedback!"

런칭_당일_전략:
  - 커뮤니티 동원 (upvote 요청)
  - 실시간 댓글 응답
  - 소셜 미디어 동시 홍보
  - 목표: Top 10 (Daily)

KPI:
  - Upvotes: 300+
  - Daily Rank: Top 10
  - 가입 전환: 100명
  - 글로벌 비율: 50%+
```
