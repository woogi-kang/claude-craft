## 2. 초기 100명 확보 전략 (First 100 Users)

### Phase 1: Pre-Launch (Week -4 to 0)

| 주차 | 활동 | 목표 | 채널 |
|-----|------|------|------|
| **-4주** | 대기 명단 페이지 제작 | 50명 대기자 | 개인 네트워크 |
| **-3주** | QA 커뮤니티 티저 | 100명 대기자 | QA Korea, 디스콰이엇 |
| **-2주** | Early Bird 발표 | 150명 대기자 | LinkedIn, SNS |
| **-1주** | 베타 테스터 모집 | 20명 베타 | 대기자 중 선별 |

### Phase 2: Launch (Week 1-4)

```
Week 1: Soft Launch
├── 베타 테스터 피드백 수집
├── 버그 수정 및 개선
└── 사례 연구 콘텐츠 준비

Week 2: Community Launch
├── QA Korea 슬랙 채널 공유
├── 디스콰이엇 프로덕트 등록
├── GeekNews 게시
└── 목표: +30명

Week 3: Product Hunt Launch
├── Product Hunt 런칭
├── 헌터 섭외 (한국 QA 인플루언서)
├── 런칭 데이 커뮤니티 동원
└── 목표: +40명

Week 4: Content Push
├── "테스트케이스 작성 자동화" 블로그
├── "놓치기 쉬운 모바일 엣지케이스 50선" 콘텐츠
├── LinkedIn QA/PM 그룹 공유
└── 목표: +30명
```

## 3. 채널별 전략

### 3.1 커뮤니티 채널

#### QA Korea (슬랙/오픈카톡)

```yaml
채널_특성:
  규모: 약 2,000명
  활성도: 높음
  의사결정권: QA 실무자 + 일부 PM

접근_전략:
  - 먼저 가치 제공 (무료 엣지케이스 체크리스트 공유)
  - 질문에 적극적으로 답변
  - 제품 홍보는 최소화, 자연스러운 언급

콘텐츠_예시:
  - "[공유] 플랫폼별 놓치기 쉬운 엣지케이스 체크리스트"
  - "[질문] TC 작성 시간 어떻게 줄이세요?"
  - "[후기] AI로 TC 생성해봤는데, 생각보다 괜찮네요"

KPI:
  - 게시물 반응: 좋아요 20+ / 댓글 10+
  - 웹사이트 유입: 100명/월
  - 가입 전환: 20명/월
```

## 4. AARRR 퍼널

### 전체 퍼널 구조

```
┌─────────────────────────────────────────────────────────────────┐
│                        AARRR Funnel                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   ACQUISITION (획득)                                             │
│   ├── 웹사이트 방문: 10,000/월                                  │
│   └── 전환율: 5%                                                │
│            ▼                                                     │
│   ACTIVATION (활성화)                                            │
│   ├── 가입 완료: 500/월                                         │
│   ├── 첫 TC 생성: 60%                                           │
│   └── 가입 → 활성화: 300/월                                     │
│            ▼                                                     │
│   RETENTION (유지)                                               │
│   ├── 2주 재방문: 40%                                           │
│   ├── 월 활성 사용자: 30%                                       │
│   └── 활성화 → 유지: 120/월                                     │
│            ▼                                                     │
│   REVENUE (수익)                                                 │
│   ├── Free → Paid 전환: 8%                                      │
│   ├── 유료 사용자: 24/월                                        │
│   └── MRR: $360/월                                              │
│            ▼                                                     │
│   REFERRAL (추천)                                                │
│   ├── 추천 프로그램 참여: 10%                                   │
│   ├── 바이럴 계수: 0.3                                          │
│   └── 추천 유입: 36/월                                          │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 단계별 상세 전략

#### Acquisition (획득)

| 채널 | 월 유입 | CAC | 전환율 |
|-----|--------|-----|-------|
| **Organic/SEO** | 4,000 | $0 | 6% |
| **커뮤니티** | 2,000 | $0 | 8% |
| **SNS** | 2,000 | $0 | 4% |
| **Product Hunt** | 1,500 | $0 | 5% |
| **Paid (Phase 2)** | 500 | $50 | 3% |

**Acquisition 최적화:**
- 랜딩 페이지 A/B 테스트
- CTA 문구 최적화
- Social Proof 강화 (사용자 수, 후기)

#### Activation (활성화)

```yaml
활성화_정의: 첫 TC 생성 완료

온보딩_흐름:
  Step 1: 가입 (Email/Google)
  Step 2: 플랫폼 선택 (Android/iOS/Web)
  Step 3: 샘플 PRD로 데모 (30초)
  Step 4: 직접 PRD 업로드
  Step 5: TC 생성 완료 → "Wow Moment"

활성화_개선_전략:
  - 온보딩 단계 최소화 (5단계 이내)
  - 샘플 PRD 자동 제공
  - 진행률 표시
  - 첫 TC 생성 시 축하 메시지

드롭오프_분석:
  - 가입 후 미업로드: 이메일 리마인더 (24h, 72h)
  - 업로드 후 미생성: 인앱 가이드 강화
  - 생성 실패: 에러 메시지 개선
```

#### Retention (유지)

```yaml
유지_정의: 14일 내 재방문 + TC 생성

유지_전략:
  - 신규 엣지케이스 추가 알림
  - 주간 QA 인사이트 뉴스레터
  - 기획 변경 시 TC 업데이트 유도

리인게이지먼트:
  Day 3: "첫 TC는 만족스러우셨나요?" 이메일
  Day 7: "이번 주 새로 추가된 엣지케이스" 알림
  Day 14: "무료 한도 리셋 임박" 알림
  Day 30: "놓치고 있는 기능" 이메일

Churn_예방:
  - 사용량 감소 감지 → 개인화 이메일
  - 유료 해지 시도 → 할인 오퍼
  - NPS 낮은 사용자 → 1:1 피드백 수집
```

#### Revenue (수익)

```yaml
전환_트리거:
  - 월 5 TC 한도 도달
  - 추가 플랫폼 필요
  - 팀 멤버 초대 필요
  - Export 형식 다양화 필요

전환_최적화:
  - 한도 도달 시 자연스러운 업그레이드 모달
  - "Pro 기능 미리보기" 티저
  - 연간 결제 20% 할인 강조
  - Early Bird / 레퍼럴 할인 활용

업셀_전략:
  Pro → Team:
    - 팀 멤버 3명 초과 시 유도
    - TestRail/Jira 연동 필요 시 유도
    - API 필요 시 유도
```

#### Referral (추천)

```yaml
추천_프로그램:
  보상_구조:
    - 추천인: 1개월 무료 (Pro 기준 $12)
    - 피추천인: 첫 달 50% 할인

  추천_채널:
    - 고유 추천 링크
    - 이메일 초대
    - SNS 공유 버튼

바이럴_루프:
  1. 사용자 A가 TC 생성
  2. TC에 "Powered by TestCraft" 워터마크
  3. TC 공유 시 TestCraft 링크 포함
  4. 사용자 B가 링크 클릭 → 가입
  5. 사용자 A에게 보상

바이럴_계수_목표:
  초기: 0.3 (10명 중 3명 추천)
  목표: 0.5+ (10명 중 5명 추천)
```
