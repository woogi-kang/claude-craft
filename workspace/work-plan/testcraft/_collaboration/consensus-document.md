# Claude-Gemini 합의 문서: TestCraft TC 개선

> 합의일: 2026-01-16

> 상태: **합의 완료 - 실행 준비**

## Executive Summary

Claude와 Gemini의 협업을 통해 TestCraft의 TC 생성 프로세스 개선 방향에 대한 합의를 도출했습니다. 핵심 합의 사항은 다음과 같습니다:

1. **대화형 TC 세분화**: Wizard UI 기반 단계별 질문 시스템 도입
2. **Excel 템플릿 표준화**: 통합 시트 구조 + 추가 필수 컬럼
3. **UX 개선**: PRD 분석 시각화, 실시간 미리보기, 사전 설정 템플릿
4. **TC 품질 향상**: 부정적 TC, 경계값 분석, 테스트 데이터 자동 생성

---

## 1. 대화형 TC 세분화 - 합의 사항

### 1.1 최종 플로우 (7단계)

```
Step 1: PRD 업로드
    ↓
Step 2: AI 기능 추출 → 사용자 편집 (드래그앤드롭 UI)
    - Epic > Feature > User Story 위계 표현
    ↓
Step 3: 테스트 범위 선택 (기능별)
    - 정상 시나리오 / 에러 케이스 / 성능 / 보안
    ↓
Step 4: 데이터 관점 질문 (NEW)
    - 정상 데이터 / 비정상 데이터 / 경계값 데이터
    ↓
Step 5: 사용자 페르소나 선택 (NEW)
    - 신규 / 일반 / 파워 / 관리자
    ↓
Step 6: 플랫폼별 상세 옵션
    - Android/iOS/Web 특화 옵션
    ↓
Step 7: TC 생성 (실시간 미리보기 제공)
    ↓
Step 8: 생성 결과 리뷰 → 추가 요청 반영
```

### 1.2 UI/UX 합의 사항

| 항목 | 합의 내용 | 우선순위 |
|-----|----------|---------|
| **Wizard UI** | 한 번에 한 단계씩 집중하는 마법사 형태 | P0 (MVP) |
| **실시간 미리보기** | 옵션 선택 시 예상 TC 개수/목록 실시간 업데이트 | P0 (MVP) |
| **사전 설정 템플릿** | 스모크 테스트, 회귀 테스트, 보안 테스트 등 프리셋 | P1 |
| **기능 편집 UI** | 드래그앤드롭으로 추가/수정/삭제/순서변경 | P0 (MVP) |

### 1.3 질문 유형 확장

**기존 (Claude 제안)**:
- 테스트 범위 질문
- 플랫폼별 옵션 질문

**추가 (Gemini 제안)**:
- 데이터 관점 질문 (정상/비정상/경계값)
- 사용자 페르소나 질문 (신규/일반/파워/관리자)

## 2. Excel 템플릿 - 합의 사항

| **테스트 데이터** | 테스트에 필요한 구체적 데이터 | **신규** |

| 비고 | 추가 참고사항 | 기존 |

## 3. UX 개선 - 합의 사항

### 3.1 PRD 분석 시각화

**합의**: PRD 분석 후 원본 문서에 하이라이팅으로 추출 근거 표시
- AI의 분석 과정 투명화
- 사용자 신뢰도 향상
- **우선순위**: P1

### 3.2 대시보드 기능

**합의**: 프로젝트별 TC 현황 대시보드 제공
- TC 생성 현황
- 커버리지 시각화
- 테스트 실행 결과 (통과/실패/진행률)
- **우선순위**: P2 (이미 Feature Spec에 포함)

### 3.3 TC 버전 관리

**합의**: TC 변경 이력 및 diff 제공
- 이전 버전과의 차이 표시
- 롤백 기능
- **우선순위**: P2 (이미 Feature Spec에 포함)

### 3.4 협업 기능

**합의**: TC에 대한 댓글 및 @mention 기능
- 팀 협업 플랫폼으로 발전
- **우선순위**: P1 (이미 Feature Spec에 포함)

## 4. TC 품질 향상 - 합의 사항

### 4.1 신규 기능 추가

| 기능 | 설명 | 우선순위 |
|-----|------|---------|
| **부정적 TC 자동 생성** | "만약 ~가 아니라면?" 시나리오 자동 제안 | P0 (MVP) |
| **경계값 분석 제안** | 입력 필드의 경계값 TC 자동 제안 | P0 (MVP) |
| **테스트 데이터 자동 생성** | 더미 데이터 자동 채움 | P1 |
| **접근성 체크리스트** | 웹/앱 기본 접근성 테스트 항목 자동 추가 | P2 |

### 4.2 부정적 TC 예시

```
원본 TC: "유효한 이메일로 회원가입 성공"

자동 생성되는 부정적 TC:
- "유효하지 않은 이메일 형식 입력 시 오류 메시지 표시"
- "이미 가입된 이메일 입력 시 중복 안내"
- "비밀번호 불일치 시 가입 버튼 비활성화"
- "필수 항목 미입력 시 경고 표시"
```

### 4.3 경계값 분석 예시

```
입력 필드: 비밀번호 (8-20자)

자동 생성되는 경계값 TC:
- "7자 입력 시 오류 (최소-1)"
- "8자 입력 시 성공 (최소)"
- "20자 입력 시 성공 (최대)"
- "21자 입력 시 오류 (최대+1)"
```
