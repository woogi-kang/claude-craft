## 1. 현재 상황

TestCraft는 PRD/기획서를 업로드하면 플랫폼별 테스트케이스를 자동 생성하는 AI 기반 SaaS입니다.

### 현재 TC 생성 플로우
```
PRD 업로드 → 플랫폼 선택 → AI 분석 → TC 일괄 생성 → Export
```

### 개선이 필요한 영역
1. **TC 세분화 부족**: 현재는 일괄 생성 방식으로, 사용자의 세부 요구사항 반영이 어려움
2. **대화형 상호작용 부재**: 사용자와의 티키타카 없이 일방적 결과 제공
3. **Excel 템플릿 미비**: 업계 표준 TC 관리 형식(Excel)에 대한 템플릿 부재

## 2. 개선 제안

### 제안 A: 대화형 TC 세분화 (Interactive TC Refinement)

**개념**: TC 생성 과정에서 사용자와 단계별 대화를 통해 요구사항을 구체화

**제안 플로우**:
```
Step 1: PRD 업로드
    ↓
Step 2: AI가 핵심 기능 목록 추출 → 사용자 확인/수정
    ↓
Step 3: 기능별 테스트 범위 질문 → 사용자 선택
    (예: "로그인 기능의 테스트 범위를 선택해주세요"
         □ 정상 시나리오만
         □ 에러 케이스 포함
         □ 성능 테스트 포함
         □ 보안 테스트 포함)
    ↓
Step 4: 플랫폼별 상세 옵션 질문
    (예: "Android 테스트에서 중점적으로 확인할 항목은?"
         □ 다양한 해상도 대응
         □ 권한 거부 시나리오
         □ 백그라운드 전환)
    ↓
Step 5: TC 생성 (사용자 선택 기반)
    ↓
Step 6: 생성된 TC 리뷰 → 추가 요청 반영
```

**예상 이점**:
- 사용자 요구사항에 맞춘 맞춤형 TC
- 불필요한 TC 생성 방지
- 사용자 학습 효과 (어떤 테스트가 필요한지 인지)

### 제안 B: Excel TC 템플릿 표준화

**개념**: 업계에서 널리 사용되는 TC 관리 형식을 템플릿으로 제공

**제안 템플릿 구조**:

| 시트명 | 용도 | 주요 컬럼 |
|-------|------|----------|
| **Overview** | 프로젝트 요약 | 프로젝트명, 버전, 작성일, 플랫폼 |
| **Test Cases** | TC 목록 | TC-ID, 기능, 시나리오, 전제조건, 테스트단계, 예상결과, 우선순위, 상태 |
| **Edge Cases** | 엣지케이스 | TC-ID, 유형, 플랫폼, 시나리오, 테스트단계, 예상결과 |
| **Execution Log** | 실행 기록 | TC-ID, 실행일, 실행자, 결과, 비고, 스크린샷링크 |
| **Defect Mapping** | 결함 매핑 | TC-ID, 결함ID, 심각도, 상태, 재현단계 |
| **Coverage Matrix** | 커버리지 | 요구사항ID, 관련TC, 커버리지상태 |

**컬럼 상세 (Test Cases 시트)**:

| 컬럼 | 설명 | 예시 |
|-----|------|------|
| TC-ID | 고유 식별자 | TC-001, TC-002 |
| 기능 | 테스트 대상 기능 | 회원가입, 로그인 |
| 시나리오 | 테스트 시나리오명 | 이메일 회원가입 성공 |
| 전제조건 | 테스트 전 필요 상태 | 로그아웃 상태 |
| 테스트단계 | 상세 실행 절차 | 1. 회원가입 페이지 접속\n2. 이메일 입력... |
| 예상결과 | 기대 동작 | 인증 이메일 발송, 완료 메시지 표시 |
| 플랫폼 | 대상 플랫폼 | Android / iOS / Web / All |
| 카테고리 | TC 유형 | 기능/엣지케이스/경계값/성능/보안 |
| 우선순위 | 중요도 | P0(필수) / P1(높음) / P2(중간) / P3(낮음) |
| 상태 | 실행 상태 | 미실행/통과/실패/차단/건너뜀 |
| 담당자 | 실행 담당 | - |
| 비고 | 추가 참고사항 | - |
