## 1. 데이터 전략 개요

### 1.1 목적

TestCraft의 데이터 전략은 다음 목표를 달성하기 위해 설계되었습니다:

| 목적 | 설명 | 활용 |
|-----|------|------|
| **제품 개선** | 사용자 행동 기반 UX 최적화 | 플로우 개선, 기능 우선순위 |
| **비즈니스 성장** | 전환율, 리텐션 분석 | 마케팅, 가격 전략 |
| **AI 품질 향상** | TC 생성 품질 피드백 | 모델 튜닝, 프롬프트 개선 |
| **운영 효율화** | 시스템 모니터링 | 인프라 최적화, 비용 관리 |

### 1.2 데이터 원칙

```
┌─────────────────────────────────────────────────────────────────┐
│                      데이터 원칙                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   1. 최소 수집 (Minimal Collection)                             │
│      필요한 데이터만 수집, 과도한 추적 지양                      │
│                                                                  │
│   2. 투명성 (Transparency)                                      │
│      수집 데이터 및 용도 명확히 고지                            │
│                                                                  │
│   3. 보안 (Security)                                            │
│      암호화, 접근 제어, 감사 로그                               │
│                                                                  │
│   4. 활용 가능성 (Actionability)                                │
│      수집 데이터는 반드시 의사결정에 활용                       │
│                                                                  │
│   5. 개인정보 보호 (Privacy)                                    │
│      GDPR/개인정보보호법 준수, 동의 기반                        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 데이터 분류

### 2.1 데이터 유형

| 유형 | 설명 | 예시 | 보관 기간 |
|-----|------|------|----------|
| **사용자 데이터** | 계정 및 프로필 | 이메일, 이름 | 계정 삭제 시까지 |
| **행동 데이터** | 서비스 내 활동 | 클릭, 페이지뷰 | 2년 |
| **비즈니스 데이터** | 프로젝트, TC | PRD, 테스트케이스 | 사용자 삭제 시까지 |
| **시스템 데이터** | 로그, 메트릭 | 에러 로그, 응답 시간 | 90일 |
| **분석 데이터** | 집계/익명화 | DAU, 전환율 | 영구 |

### 2.2 데이터 민감도

| 민감도 | 데이터 | 보호 조치 |
|-------|-------|----------|
| **High** | 이메일, 비밀번호, API 키 | 암호화 저장, 접근 로그 |
| **Medium** | PRD 내용, TC 내용 | 암호화 전송, 권한 제어 |
| **Low** | 페이지뷰, 클릭 이벤트 | 익명화 가능 |

## 3. 이벤트 트래킹 설계

### 3.1 이벤트 네이밍 컨벤션

```
{object}_{action}

예시:
- user_signup
- project_create
- prd_upload
- tc_generate
- tc_export
```

### 3.2 핵심 이벤트 목록

#### 인증 이벤트

| 이벤트 | 설명 | 속성 |
|-------|------|------|
| `user_signup` | 회원가입 완료 | method (email/google/github) |
| `user_login` | 로그인 | method, success |
| `user_logout` | 로그아웃 | - |
| `user_password_reset` | 비밀번호 재설정 | - |

#### 프로젝트 이벤트

| 이벤트 | 설명 | 속성 |
|-------|------|------|
| `project_create` | 프로젝트 생성 | project_id |
| `project_view` | 프로젝트 조회 | project_id |
| `project_delete` | 프로젝트 삭제 | project_id, tc_count |
| `project_settings_update` | 설정 변경 | project_id, changed_fields |

#### PRD 업로드 이벤트

| 이벤트 | 설명 | 속성 |
|-------|------|------|
| `prd_upload_start` | 업로드 시작 | project_id, file_type, file_size |
| `prd_upload_complete` | 업로드 완료 | project_id, file_type, page_count |
| `prd_upload_fail` | 업로드 실패 | project_id, error_type |
| `prd_notion_import` | Notion 가져오기 | project_id, page_count |
| `prd_figma_import` | Figma 가져오기 | project_id, frame_count |

#### TC 생성 이벤트

| 이벤트 | 설명 | 속성 |
|-------|------|------|
| `tc_generate_start` | TC 생성 시작 | project_id, platforms[], options |
| `tc_generate_complete` | TC 생성 완료 | project_id, tc_count, edge_case_count, duration_sec |
| `tc_generate_fail` | TC 생성 실패 | project_id, error_type, stage |
| `tc_generate_cancel` | TC 생성 취소 | project_id, stage, progress_pct |

#### TC 관리 이벤트

| 이벤트 | 설명 | 속성 |
|-------|------|------|
| `tc_view` | TC 상세 조회 | project_id, tc_id |
| `tc_edit` | TC 수정 | project_id, tc_id, edited_fields[] |
| `tc_delete` | TC 삭제 | project_id, tc_id, is_bulk |
| `tc_filter` | 필터 적용 | project_id, filter_type, filter_value |
| `tc_search` | 검색 | project_id, query, result_count |

#### Export 이벤트

| 이벤트 | 설명 | 속성 |
|-------|------|------|
| `export_start` | Export 시작 | project_id, format, tc_count, scope |
| `export_complete` | Export 완료 | project_id, format, tc_count, file_size |
| `export_fail` | Export 실패 | project_id, format, error_type |

#### 협업 이벤트

| 이벤트 | 설명 | 속성 |
|-------|------|------|
| `team_invite` | 팀원 초대 | project_id, role |

| `team_leave` | 팀 탈퇴 | project_id |

#### 결제 이벤트 (P1)

| 이벤트 | 설명 | 속성 |
|-------|------|------|
| `subscription_start` | 구독 시작 | plan, billing_cycle |

| `subscription_downgrade` | 다운그레이드 | from_plan, to_plan |

### 3.3 이벤트 속성 공통 스키마

```typescript
interface BaseEvent {
  // 자동 수집
  event_id: string;          // 이벤트 고유 ID
  timestamp: string;         // ISO 8601
  user_id: string;           // 사용자 ID (로그인 시)
  anonymous_id: string;      // 익명 ID (미로그인 시)
  session_id: string;        // 세션 ID

  // 컨텍스트
  page_url: string;
  page_title: string;
  referrer: string;

  // 디바이스
  device_type: 'desktop' | 'tablet' | 'mobile';
  browser: string;
  os: string;
  screen_resolution: string;

  // 사용자 속성 (식별 시)
  user_plan?: 'free' | 'pro' | 'enterprise';
  user_created_at?: string;
  user_project_count?: number;
}

interface TCGenerateCompleteEvent extends BaseEvent {
  event_name: 'tc_generate_complete';

  // 이벤트별 속성
  project_id: string;
  tc_count: number;
  edge_case_count: number;
  platforms: string[];
  duration_sec: number;
  prd_page_count: number;
}
```

### 4.2 핵심 KPI

| 카테고리 | 지표 | 정의 | 목표 (MVP) | 목표 (6개월) |
|---------|------|------|-----------|-------------|
| **활성화** | DAU | 일간 활성 사용자 | 30 | 300 |
| | WAU | 주간 활성 사용자 | 70 | 700 |
| | MAU | 월간 활성 사용자 | 100 | 1,000 |
| **참여도** | TC/User/Month | 사용자당 월 TC 생성 수 | 50 | 100 |
| | Projects/User | 사용자당 프로젝트 수 | 2 | 3 |
| | Session Duration | 평균 세션 시간 | 10분 | 15분 |
| **리텐션** | D1 Retention | 1일 재방문율 | 30% | 50% |
| | D7 Retention | 7일 재방문율 | 20% | 40% |
| | D30 Retention | 30일 재방문율 | 10% | 30% |
| **전환** | Signup Rate | 방문자 가입 전환율 | 5% | 10% |
| | Activation Rate | 가입 후 TC 생성 비율 | 50% | 70% |
| | Paid Conversion | 유료 전환율 | 5% | 15% |
| **품질** | TC Accuracy | TC 정확도 (편집률 기반) | 70% | 85% |
| | Generation Success | TC 생성 성공률 | 90% | 98% |
| | NPS | 순추천지수 | 30 | 50 |

### 4.3 지표 계산 공식

```
활성 사용자 (Active User)
= 해당 기간 내 1회 이상 의미있는 액션(TC 조회/생성/편집) 수행한 사용자

TC 정확도 (TC Accuracy)
= 1 - (편집된 TC 수 / 전체 생성 TC 수)
* 사용자가 편집하지 않은 TC = AI 생성 품질 만족

활성화율 (Activation Rate)
= (가입 후 7일 내 TC 1개 이상 생성한 사용자) / (신규 가입자)

유료 전환율 (Paid Conversion)
= (유료 구독 시작) / (Free 플랜 사용자)
```

### 7.3 이벤트 구현 예시 (Next.js)

```typescript
// lib/analytics.ts
import mixpanel from 'mixpanel-browser';

// 초기화
export const initAnalytics = () => {
  if (process.env.NEXT_PUBLIC_MIXPANEL_TOKEN) {
    mixpanel.init(process.env.NEXT_PUBLIC_MIXPANEL_TOKEN, {
      debug: process.env.NODE_ENV === 'development',
      track_pageview: true,
    });
  }
};

// 이벤트 트래킹
export const track = (
  eventName: string,
  properties?: Record<string, unknown>
) => {
  mixpanel.track(eventName, {
    ...properties,
    timestamp: new Date().toISOString(),
  });
};

// 사용자 식별
export const identify = (userId: string, traits?: Record<string, unknown>) => {
  mixpanel.identify(userId);
  if (traits) {
    mixpanel.people.set(traits);
  }
};

// 사용 예시
// TC 생성 완료 이벤트
track('tc_generate_complete', {
  project_id: 'proj_123',
  tc_count: 120,
  edge_case_count: 45,
  platforms: ['android', 'ios'],
  duration_sec: 42,
  prd_page_count: 12,
});
```
