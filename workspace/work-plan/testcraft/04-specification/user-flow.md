### 2.2 단계별 상세

| 단계 | 화면 | 사용자 액션 | 시스템 응답 | 예외 처리 |
|-----|------|-----------|-----------|----------|
| 1 | 랜딩 | "시작하기" 클릭 | 회원가입 페이지 이동 | - |
| 2a | 회원가입 | 이메일/비밀번호 입력 | 실시간 유효성 검사 | 형식 오류 표시 |
| 2b | 회원가입 | 소셜 로그인 버튼 클릭 | OAuth 화면 이동 | 팝업 차단 안내 |
| 3a | 회원가입 | 약관 동의 체크 | - | 미동의 시 버튼 비활성화 |
| 3b | OAuth | 권한 승인 | 계정 자동 생성 | 취소 시 회원가입 페이지 |
| 4a | 이메일 | 인증 링크 클릭 | 계정 활성화 | 만료 시 재발송 안내 |
| 5 | - | - | 대시보드 자동 이동 | - |

### 2.3 예외 플로우

```
[E1: 이메일 중복]
이메일 입력 → 중복 체크 → "이미 가입된 이메일입니다" → 로그인 링크 제공

[E2: 인증 이메일 만료]
링크 클릭 → 만료 확인 → "링크가 만료되었습니다" → 재발송 버튼

[E3: 소셜 로그인 취소]
OAuth 화면 → 취소 클릭 → 회원가입 페이지 복귀

[E4: 비밀번호 조건 미충족]
비밀번호 입력 → 조건 미충족 → 요구사항 표시 (8자 이상, 영문+숫자)
```
