> TC Excel 템플릿 상세 명세서
> 버전: 1.0
> 작성일: 2026-01-16
> 상태: Draft
> 협업: Claude-Gemini 합의 기반

---

## 1. 개요

### 1.1 목적

업계 표준에 부합하는 TC 관리용 Excel 템플릿을 제공하여, 기존 워크플로우와의 호환성을 보장하고 효율적인 TC 관리를 지원합니다.

### 1.2 설계 원칙

| 원칙 | 설명 |
|-----|------|
| **ISTQB 표준 준수** | 국제 소프트웨어 테스팅 자격 위원회 표준 참조 |
| **통합 관리** | Edge Cases를 별도 시트가 아닌 카테고리로 통합 |
| **역추적성** | 요구사항 ID 연결로 커버리지 추적 가능 |
| **확장성** | Jira, TestRail 등과 연동 가능한 구조 |

## 3. 시트별 상세 명세

### 3.1 Overview 시트

#### 용도

#### 레이아웃

```
┌─────────────────────────────────────────────────────────────────┐
│                        TestCraft TC Report                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  프로젝트 정보                                                   │
│  ┌───────────────────┬────────────────────────────────────────┐ │
│  │ 프로젝트명        │ [프로젝트 이름]                         │ │
│  │ 버전             │ v1.0.0                                  │ │
│  │ 작성일           │ 2026-01-16                              │ │
│  │ 작성자           │ TestCraft AI + [사용자명]               │ │
│  │ 대상 플랫폼      │ Android, iOS, Web                       │ │
│  │ PRD 문서         │ [원본 PRD 파일명]                        │ │
│  └───────────────────┴────────────────────────────────────────┘ │
│                                                                  │
│  TC 요약 통계                                                    │
│  ┌───────────────────┬────────────────────────────────────────┐ │
│  │ 총 TC 수         │ 150                                     │ │
│  │ 기능 테스트      │ 80 (53%)                                │ │
│  │ 엣지케이스       │ 45 (30%)                                │ │
│  │ 경계값 테스트    │ 15 (10%)                                │ │
│  │ 성능/보안        │ 10 (7%)                                 │ │
│  └───────────────────┴────────────────────────────────────────┘ │
│                                                                  │
│  우선순위별 분포                                                 │
│  ┌───────────────────┬────────────────────────────────────────┐ │
│  │ P0 (필수)        │ 30 (20%)                                │ │
│  │ P1 (높음)        │ 50 (33%)                                │ │
│  │ P2 (중간)        │ 45 (30%)                                │ │
│  │ P3 (낮음)        │ 25 (17%)                                │ │
│  └───────────────────┴────────────────────────────────────────┘ │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

#### 컬럼 정의

| 필드 | 타입 | 설명 |

| 프로젝트명 | Text | 프로젝트 이름 |
| 버전 | Text | TC 문서 버전 |

| 작성자 | Text | 작성자 정보 |
| 대상 플랫폼 | Text | 테스트 대상 플랫폼 목록 |
| PRD 문서 | Text | 원본 PRD 파일명 |

### 3.2 Test Cases 시트 (핵심)

#### 용도
모든 TC를 통합 관리하는 메인 시트입니다.

#### 컬럼 정의

| 컬럼 | 필수 | 타입 | 설명 | 예시 |
|-----|-----|------|------|------|
| **TC-ID** | ✅ | Text | 고유 식별자 | TC-001 |
| **기능** | ✅ | Text | 테스트 대상 기능 | 회원가입 |
| **시나리오** | ✅ | Text | 테스트 시나리오명 | 이메일 회원가입 성공 |
| **전제조건** | ✅ | Text | 테스트 전 필요 상태 | 로그아웃 상태, 앱 최초 실행 |
| **테스트단계** | ✅ | Text | 상세 실행 절차 (줄바꿈 포함) | 1. 회원가입 페이지 접속\n2. 이메일 입력... |
| **예상결과** | ✅ | Text | 기대 동작 | 인증 이메일 발송, 완료 메시지 표시 |
| **테스트 데이터** | ✅ | Text | 테스트에 필요한 구체적 데이터 | 이메일: test@example.com, PW: Test1234! |
| **실행 후 조건** | | Text | 테스트 완료 후 시스템 상태 | 사용자 계정 생성됨, 인증 대기 상태 |
| **플랫폼** | ✅ | Enum | 대상 플랫폼 | Android / iOS / Web / All |
| **카테고리** | ✅ | Enum | TC 유형 | 기능 / 엣지케이스 / 경계값 / 성능 / 보안 / 접근성 |
| **우선순위** | ✅ | Enum | 중요도 | P0 / P1 / P2 / P3 |
| **상태** | | Enum | 실행 상태 | 미실행 / 통과 / 실패 / 차단 / 건너뜀 |
| **연관 요구사항 ID** | | Text | 역추적용 요구사항 ID | REQ-001, REQ-002 |
| **테스트 환경** | | Text | 실행 환경 정보 | Android 13, Galaxy S23 |
| **담당자** | | Text | 실행 담당자 | - |
| **비고** | | Text | 추가 참고사항 | - |

#### 카테고리 값 정의

| 값 | 설명 | 색상 코드 |
|---|------|----------|
| 기능 | 정상 기능 테스트 (Happy Path) | #4CAF50 (녹색) |
| 엣지케이스 | 예외 상황 테스트 | #FF9800 (주황) |
| 경계값 | 경계값 분석 테스트 | #2196F3 (파랑) |
| 성능 | 성능/부하 테스트 | #9C27B0 (보라) |
| 보안 | 보안 취약점 테스트 | #F44336 (빨강) |
| 접근성 | 접근성 테스트 | #00BCD4 (청록) |

#### 우선순위 값 정의

| 값 | 설명 | 기준 |
|---|------|------|
| P0 | 필수 (Critical) | 핵심 기능, 배포 차단 기준 |
| P1 | 높음 (High) | 주요 기능, 심각한 영향 |
| P2 | 중간 (Medium) | 일반 기능, 중간 영향 |
| P3 | 낮음 (Low) | 부가 기능, 경미한 영향 |

#### 상태 값 정의

| 값 | 설명 | 색상 코드 |
|---|------|----------|
| 미실행 | 아직 실행하지 않음 | #9E9E9E (회색) |
| 통과 | 예상 결과와 일치 | #4CAF50 (녹색) |
| 실패 | 예상 결과와 불일치 | #F44336 (빨강) |
| 차단 | 선행 조건으로 실행 불가 | #FF9800 (주황) |
| 건너뜀 | 의도적으로 생략 | #9E9E9E (회색) |

### 3.5 Coverage Matrix 시트

#### 용도
요구사항과 TC의 매핑 관계를 시각화합니다.

#### 레이아웃

```
┌─────────────────────────────────────────────────────────────────┐
│                     Requirements Coverage Matrix                 │
├──────────┬──────────┬──────────┬──────────┬──────────┬─────────┤
│ 요구사항  │ TC-001   │ TC-002   │ TC-003   │ TC-004   │ 커버리지 │
├──────────┼──────────┼──────────┼──────────┼──────────┼─────────┤
│ REQ-001  │    ●     │    ●     │          │          │  50%    │
│ REQ-002  │          │    ●     │    ●     │    ●     │  75%    │
│ REQ-003  │    ●     │          │          │    ●     │  50%    │
│ REQ-004  │          │          │          │          │   0%    │ ← 커버 안됨
└──────────┴──────────┴──────────┴──────────┴──────────┴─────────┘
```

#### 컬럼 정의

| 컬럼 | 타입 | 설명 |
|-----|------|------|
| **요구사항 ID** | Text | 요구사항 식별자 |
| **요구사항 설명** | Text | 요구사항 내용 요약 |
| **관련 TC** | Text | 연결된 TC ID 목록 |
| **TC 수** | Number | 연결된 TC 개수 |
| **커버리지 상태** | Enum | 완전 / 부분 / 미커버 |

### 3.6 Glossary 시트 (신규)

#### 용도
프로젝트에서 사용되는 용어와 약어를 정의합니다.

#### 컬럼 정의

| 컬럼 | 필수 | 타입 | 설명 |
|-----|-----|------|------|
| **용어** | ✅ | Text | 용어 (정식 명칭) |
| **약어** | | Text | 약어 표기 |
| **정의** | ✅ | Text | 용어의 정의 |
| **예시** | | Text | 사용 예시 |
| **관련 문서** | | URL | 참고 문서 링크 |

#### 예시 데이터

| 용어 | 약어 | 정의 | 예시 |
|-----|-----|------|------|
| Test Case | TC | 특정 조건에서 시스템의 예상 동작을 검증하는 테스트 항목 | TC-001: 로그인 성공 |
| Product Requirements Document | PRD | 제품 요구사항을 정의한 문서 | feature-spec.md |
| Edge Case | - | 정상 범위를 벗어난 예외적인 입력이나 상황 | 네트워크 끊김 상태에서 저장 |
| Boundary Value | BV | 입력 범위의 경계에 해당하는 값 | 비밀번호 최소 8자 |
