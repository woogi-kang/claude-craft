
### 1.2 브랜드 키워드

| 카테고리 | Primary Keywords | Secondary Keywords |
|---------|-----------------|-------------------|
| **기능적** | 자동화, 정확성, 효율 | 빠름, 간편, 완전 |
| **감정적** | 안심, 자신감, 만족 | 놀람, 감탄, 편안 |
| **인지적** | 전문적, 똑똑한, 믿음직한 | 현대적, 기술적, 세련된 |

### 1.3 브랜드 포지셔닝 선언

> **For** QA 엔지니어와 IT 기획자
> **Who** 테스트케이스 작성에 너무 많은 시간을 쓰고 있는
> **TestCraft is** AI 기반 테스트케이스 자동 생성 서비스
> **That** 기획서만 업로드하면 5분 만에 완전한 TC를 제공합니다
> **Unlike** 수동 작성이나 기존 테스트 관리 도구
> **Our product** 플랫폼별 엣지케이스까지 자동으로 포함합니다

### 2.2 퍼스낼리티 요약

```yaml
primary_traits:
  - 믿음직한 (Reliable): "제대로 해내는"
  - 똑똑한 (Smart): "알아서 챙기는"
  - 효율적인 (Efficient): "시간 아껴주는"

secondary_traits:
  - 접근하기 쉬운 (Approachable): "어렵지 않은"
  - 현대적인 (Modern): "최신 기술의"
  - 세심한 (Thorough): "꼼꼼한"

not_this:
  - 과시적인 (Flashy): 기술 과시 X
  - 차갑거나 로봇 같은 (Cold/Robotic): 무미건조 X
  - 복잡한 (Complicated): 어려워 보이는 X
  - 유치한 (Childish): 지나치게 가벼운 X
```

### 3.2 상황별 톤

| 상황 | 톤 | 예시 |
|-----|---|-----|
| **마케팅/랜딩** | 자신감 + 호기심 유발 | "놓치는 엣지케이스, 이제 없습니다" |
| **온보딩** | 친절 + 안내 | "첫 번째 테스트케이스를 만들어볼까요?" |
| **작업 중** | 담백 + 정보 전달 | "PRD 분석 중... 12개 기능 발견" |
| **성공** | 축하 + 다음 단계 | "120개 TC가 생성되었습니다. 확인해보세요" |
| **에러** | 공감 + 해결책 | "문제가 발생했습니다. 이렇게 해보세요" |
| **도움말** | 친절 + 명확 | "여기서 플랫폼을 선택하면 해당 엣지케이스가 추가됩니다" |

## 10. 모션 & 애니메이션 가이드

### 10.1 애니메이션 원칙

```yaml
principles:
  purposeful:
    - 의미 있는 애니메이션만 사용
    - 장식적 애니메이션 지양

  subtle:
    - 과하지 않은 미묘한 효과
    - 전문 도구의 신뢰감 유지

  responsive:
    - 즉각적인 피드백
    - 사용자 액션에 반응

timing:
  micro: 100-150ms   # 버튼 피드백, 토글
  short: 200-250ms   # 페이드, 슬라이드
  medium: 300-400ms  # 모달, 패널
  long: 500-800ms    # 페이지 전환, 완료 효과

easing:
  default: "cubic-bezier(0.4, 0, 0.2, 1)"   # ease-in-out
  enter: "cubic-bezier(0, 0, 0.2, 1)"       # ease-out
  exit: "cubic-bezier(0.4, 0, 1, 1)"        # ease-in
  bounce: "cubic-bezier(0.175, 0.885, 0.32, 1.275)"  # spring
```
