# TestCraft Market Research

## Executive Summary

QA 자동화 및 테스트 관리 시장은 2024년 기준 약 346억 달러 규모로, 연평균 19% 성장률을 기록하며 빠르게 확대되고 있습니다. 특히 AI 기반 테스트케이스 자동 생성은 신흥 세그먼트로, 기존 테스트 관리 도구들의 공백 영역입니다.

---

## 1. 시장 규모 분석 (TAM/SAM/SOM)

### TAM (Total Addressable Market) - 전체 시장

| 시장 세그먼트 | 2024년 | 2025년 | 2029년 | CAGR |
|--------------|--------|--------|--------|------|
| **소프트웨어 테스팅 전체** | $55.8B | $60.0B | - | 7.2% |
| **자동화 테스팅** | $34.6B | $35.3B | $59.9B | 19.0% |
| **테스트 관리 도구** | $2.5B* | $2.9B* | $5.0B* | 15.0%* |

> *테스트 관리 도구 시장은 자동화 테스팅 시장의 약 7-8% 비중으로 추정

**시장 동향:**
- 북미가 2024년 최대 시장 점유율 보유 (기술 공급업체 밀집)
- BFSI(금융) 섹터가 27.42%로 최대 수요 산업
- Functional Testing이 52.7%로 테스트 유형 중 최대 비중
- 솔루션 부문이 62.2%, 서비스 부문이 19.4% CAGR로 성장 중

### SAM (Serviceable Addressable Market) - 접근 가능 시장

**TestCraft SAM 정의:**
- PRD/요구사항 기반 테스트케이스 관리가 필요한 조직
- QA팀 또는 기획팀이 있는 IT 서비스 기업
- 플랫폼별(Web/Mobile/PC) 서비스 개발 기업

| 세그먼트 | 추정 규모 | 근거 |
|---------|----------|------|
| **테스트케이스 관리 도구** | $1.2B | 테스트 관리 도구 시장의 ~45% |
| **AI 테스트 생성 도구** | $300M | 신흥 시장, 급성장 중 |
| **아시아-태평양 시장** | $400M | 전체의 ~15%, 한국 포함 |

**SAM 총계: ~$1.9B (약 2.5조원)**

### SOM (Serviceable Obtainable Market) - 획득 가능 시장

**1차 타겟: 한국 시장**

| 카테고리 | 추정 기업 수 | 연간 지출 추정 | 시장 규모 |
|---------|-------------|---------------|----------|
| IT 서비스 기업 (대기업) | 50개 | $50K/년 | $2.5M |
| IT 서비스 기업 (중견) | 200개 | $15K/년 | $3.0M |
| 스타트업/스케일업 | 500개 | $5K/년 | $2.5M |
| SI/외주개발사 | 300개 | $10K/년 | $3.0M |

**한국 SOM: ~$11M (약 145억원)**

**2차 타겟: 아시아 영어권 확장**
- 싱가포르, 홍콩, 인도 영어권 기업
- 추가 SOM: ~$30M

**총 SOM (3년 목표): ~$41M (약 540억원)**

## 3. 시장 트렌드 분석

### 3.1 테스트 관리 도구 진화

```
[과거]                    [현재]                    [미래]
스프레드시트 기반    →    전문 테스트 관리 도구   →    AI 통합 테스트 플랫폼
- Excel/Sheets           - TestRail, Zephyr         - 자동 생성
- 수동 추적              - Jira 연동                - 지능형 우선순위
- 협업 어려움            - 실행 추적                - 자가 치유
                         - 리포팅                   - 예측 분석
```

### 3.2 주요 트렌드

**1. Shift-Left Testing**
- 테스트를 개발 초기 단계로 이동
- 요구사항 단계에서 테스트케이스 정의
- **TestCraft 기회**: PRD 작성 시점에 테스트케이스 자동 생성

**2. AI/ML 통합 가속화**
- 테스트케이스 자동 생성
- 결함 예측 및 우선순위 지정
- 자가 치유(Self-healing) 테스트
- **TestCraft 기회**: AI 기반 핵심 차별화 가능

**3. 코드리스(Codeless) 테스팅**
- 비개발자도 테스트 자동화 가능
- 자연어 기반 테스트 정의
- **TestCraft 기회**: IT 기획자/QA가 직접 사용 가능

**4. 플랫폼 특화 테스팅**
- 모바일, 웹, API 각각 다른 테스트 패턴
- 디바이스/브라우저 파편화 대응
- **TestCraft 기회**: 플랫폼별 엣지케이스 자동 생성

### 3.3 기술 채택 트렌드

| 기술 | 2024 채택률 | 2027 예상 | 영향 |
|-----|------------|----------|------|
| AI 테스트 자동화 | 25% | 60% | TestCraft 핵심 기술 |
| 클라우드 테스팅 | 55% | 80% | SaaS 모델 적합 |
| CI/CD 통합 | 65% | 85% | 연동 필수 |
| Jira 연동 | 70% | 85% | 최우선 통합 대상 |

## 4. 지역별 시장 분석

### 4.1 글로벌 분포

| 지역 | 시장 점유율 | 특징 |
|-----|-----------|------|
| **북미** | 42% | 기술 선도, 대기업 중심, 높은 예산 |
| **유럽** | 28% | 규제 중심, 품질 중시, GDPR 영향 |
| **아시아-태평양** | 22% | 빠른 성장, 아웃소싱 중심, 가격 민감 |
| **기타** | 8% | 신흥 시장 |

### 4.2 한국 시장 특수성

**시장 특성:**
- 대기업 SI 중심의 테스트 외주 문화
- 스타트업 생태계 급성장
- 영어 도구 사용 장벽 (로컬라이제이션 니즈)
- 카카오, 네이버 등 대형 플랫폼 기업의 자체 도구 사용

**기회 영역:**
- 한국어 지원 테스트 관리 도구 부재
- 중소 IT 서비스 기업의 도구 도입 니즈
- 외주 개발팀과 고객사 간 협업 도구 필요

## 7. 시장 기회 결론

### 7.1 핵심 인사이트

1. **타이밍이 적절함**: AI/LLM 기술 성숙으로 PRD→테스트케이스 자동화가 기술적으로 가능해짐
2. **명확한 시장 공백**: 기존 도구들은 "관리"에 집중, "생성"은 미흡
3. **성장하는 시장**: 자동화 테스팅 시장 CAGR 19%
4. **로컬 기회**: 한국어 지원 도구 부재로 진입 기회

### 7.2 Go/No-Go 권고

| 항목 | 평가 | 근거 |
|-----|------|------|
| 시장 규모 | Go | SOM $41M, 충분한 규모 |
| 성장성 | Go | CAGR 15-19% |
| 경쟁 강도 | Caution | 대형 벤더 존재, 니치 집중 필요 |
| 기술 실현성 | Go | LLM 기술로 구현 가능 |
| 차별화 가능성 | Go | PRD→TC 자동화 공백 |

**종합 권고: Go (조건부)**
- 조건: 플랫폼별 엣지케이스 생성의 명확한 차별화 필요
- 전략: 한국 시장 우선 → 아시아 확장
