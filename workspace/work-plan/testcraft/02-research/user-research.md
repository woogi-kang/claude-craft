# TestCraft User Research Guide

## Executive Summary

TestCraft의 핵심 타겟 사용자인 QA 팀, IT 기획자, 외주개발팀을 대상으로 한 사용자 리서치 계획입니다. 인터뷰 가이드, 설문 설계, 관찰 조사 방법론을 포함합니다.

---

## 1. 리서치 목표

### 1.1 핵심 질문 (Research Questions)

| # | 핵심 질문 | 검증 목표 |
|---|----------|----------|
| RQ1 | 테스트케이스 작성에 얼마나 많은 시간을 소요하는가? | Pain Point 크기 |
| RQ2 | PRD/기획서에서 테스트케이스로 변환하는 현재 프로세스는? | 현재 워크플로우 |
| RQ3 | 플랫폼별(Android/iOS/Web/PC) 테스트케이스 작성 시 어려움은? | 차별화 기회 |
| RQ4 | 엣지케이스/예외 상황은 어떻게 발견하고 관리하는가? | 핵심 기능 검증 |
| RQ5 | 현재 사용 중인 도구와 불만족 요인은? | 경쟁 포지셔닝 |
| RQ6 | AI가 테스트케이스를 자동 생성한다면 신뢰할 수 있는가? | AI 수용도 |

### 1.2 가설 (Hypotheses)

```
H1: QA/기획자는 테스트케이스 작성에 전체 업무 시간의 30% 이상을 소요한다
    → 검증 방법: 인터뷰 + 설문

H2: PRD에서 테스트케이스로 변환 시 누락되는 케이스가 30% 이상이다
    → 검증 방법: 인터뷰 + 문서 분석

H3: 플랫폼별 특수 케이스(Android 권한, iOS 가이드라인 등)는 경험에 의존한다
    → 검증 방법: 인터뷰 + 관찰

H4: 기존 테스트 관리 도구는 "관리"에 집중하고 "생성"을 지원하지 않는다
    → 검증 방법: 인터뷰 + 설문

H5: AI 생성 테스트케이스는 검토 후 사용하겠다는 의향이 70% 이상이다
    → 검증 방법: 설문 + 프로토타입 테스트
```

│  ├─ 규모: 1-3명                                                              │
│  ├─ 환경: 초기 스타트업                                                      │

## 5. 관찰 조사 (Contextual Inquiry)

### 5.1 목적

실제 업무 환경에서 테스트케이스 작성 과정을 관찰하여 인터뷰/설문에서 파악하지 못한 암묵적 행동과 워크어라운드를 발견합니다.

### 5.2 관찰 대상

| 대상 | 인원 | 시간 | 장소 |
|-----|------|------|------|
| QA 엔지니어 | 3명 | 2시간/명 | 현장 or 화면 공유 |
| IT 기획자 | 2명 | 1시간/명 | 현장 or 화면 공유 |

### 5.3 관찰 체크리스트

```
□ PRD/기획서를 어떻게 열람하는가? (Notion, Confluence, Word 등)
□ 테스트케이스 작성 시 어떤 도구를 사용하는가?
□ 기획서와 TC 작성 화면을 동시에 열어두는가?
□ 복사-붙여넣기를 많이 하는가?
□ 템플릿이나 이전 TC를 참고하는가?
□ 엣지케이스를 어떻게 떠올리는가? (경험? 체크리스트?)
□ 플랫폼별 차이를 어떻게 처리하는가?
□ 작성 중 다른 사람에게 질문하는가?
□ 작성 완료 후 검토 프로세스가 있는가?
□ 워크어라운드나 불편한 점이 드러나는 순간은?
```

### 7.1 인터뷰 분석: Affinity Diagram

```
[Pain Points]              [Needs]                [Opportunities]
├─ TC 작성 시간           ├─ 자동화              ├─ PRD→TC 변환
├─ 엣지케이스 누락        ├─ 플랫폼별 가이드     ├─ 엣지케이스 제안
├─ 요구사항 모호성        ├─ 추적성              ├─ 요구사항 명확화
├─ 플랫폼별 차이          ├─ 협업                ├─ 플랫폼별 TC
└─ ...                    └─ ...                 └─ ...
```

### 7.2 설문 분석: Key Metrics

| 지표 | 목표 | 의미 |
|-----|------|------|
| TC 작성 시간 비중 | >30% | H1 검증 |
| 엣지케이스 누락 경험 | >50% | H2 검증 |
| AI 생성 유용성 | >70% 긍정 | H5 검증 |
| WTP (지불의향) | >$10/월 | 가격 설정 |
| 플랫폼별 TC 니즈 | >60% | 차별화 검증 |
