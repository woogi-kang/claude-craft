# Couple App Positioning Strategy

## Executive Summary

> **"수동 기록 시대는 끝났다. AI가 알아서 해주는 2025 커플앱"**

비트윈을 포함한 기존 커플앱들이 "수동 기록" 방식에 머물러 있는 동안, 우리는 AI 자동화로 새로운 카테고리를 창출한다.

---

## 1. Segmentation (시장 세분화)

### 세분화 기준

| 기준 | 변수 | 세그먼트 |
|------|------|----------|
| 인구통계 | 나이, 연애 기간 | 대학생, 사회초년생, 장기연애 |
| 행동 | 앱 사용 빈도, 정리 습관 | 활발한 기록러, 귀차니스트, 방치형 |
| 심리 | 기술 수용도 | 얼리어답터, 다수, 보수적 |

### 세그먼트 정의

#### Segment A: 얼리어답터 대학생 커플

| 항목 | 내용 |
|------|------|
| 규모 | 약 50만 쌍 (활발한 앱 사용자) |
| 특성 | 새로운 앱 좋아함, SNS 활발, 트렌드 선도 |
| 니즈 | 무료, 예쁜 UI, 친구에게 자랑, 최신 기술 |
| 현재 행동 | TikTok 헤비 유저, 비트윈 쓰지만 질림 |

#### Segment B: 효율 추구 사회초년생 커플

| 항목 | 내용 |
|------|------|
| 규모 | 약 80만 쌍 (25-30세 직장인) |
| 특성 | 시간 없음, 자동화 선호, 결제 가능 |
| 니즈 | 시간 절약, 자동 기록, 기념일 관리 |
| 현재 행동 | 비트윈 유지 중, 더 나은 대안 탐색 |

#### Segment C: 변화 저항형 장기연애 커플

| 항목 | 내용 |
|------|------|
| 규모 | 약 100만 쌍 |
| 특성 | 현상 유지 선호, 전환 비용 높음 |
| 니즈 | 데이터 안전, 쉬운 전환 |
| 현재 행동 | 비트윈 고착, 새 앱 관심 낮음 |

### 세그먼트 비교

| 세그먼트 | 규모 | 전환 용이성 | 바이럴 잠재력 | 접근성 | 총점 |
|---------|------|------------|--------------|--------|------|
| A. 대학생 | 3점 | 5점 | 5점 | 5점 | 18점 |
| B. 사회초년생 | 4점 | 3점 | 3점 | 3점 | 13점 |
| C. 장기연애 | 5점 | 1점 | 2점 | 2점 | 10점 |

## 2. Targeting (타겟팅)

### 타겟팅 전략

**선택**: Concentrated (집중) + 확장

**1차 타겟 → 2차 타겟 순차 공략**

```
Phase 1 (M1-M3): 대학생 커플 집중 (60%)
         ↓
Phase 2 (M4-M6): 사회초년생 확장 (30%)
         ↓
Phase 3 (M7+): 장기연애 진입 (10%)
```

**근거**:
- 대학생: 전환 장벽 낮음, 바이럴 잠재력 최고, TikTok 접근 용이
- 무예산 상황에서 바이럴이 유일한 성장 엔진
- 대학생이 쓰면 → 사회초년생에게 확산

### 타겟 우선순위

| 순위 | 세그먼트 | 이유 | 마케팅 비중 |
|------|---------|------|------------|
| 1 | 대학생 커플 | 바이럴, 전환 용이, TikTok 매치 | 60% |
| 2 | 사회초년생 | 유료 전환 가능, 입소문 확산 | 30% |
| 3 | 장기연애 | 규모 크지만 전환 어려움 | 10% |

### Primary Target 상세

```yaml
who: "대학생 커플 (20-24세), SNS 활발, 새 앱 좋아함"
what_they_need: "사진 자동 정리, 귀찮음 해결, 예쁜 UI"
where_they_are: "TikTok, Instagram, 에브리타임"
when_they_buy: "친구 추천 or 바이럴 영상 발견 시"
why_they_choose_us: "AI가 알아서 해줌, 비트윈보다 신선함"
```

## 3. Positioning (포지셔닝)

### 포지셔닝 공식

```
For [타겟 고객]
-----------------------------------------------
사진 정리와 데이트 기록이 귀찮은 2030 커플

[브랜드]는 [카테고리]로서
-----------------------------------------------
[앱 이름]은 AI 자동화 커플앱으로서

[핵심 편익]을 제공합니다
-----------------------------------------------
수동 기록 없이 추억이 자동으로 정리되는 편리함을 제공합니다

왜냐하면 [차별화 이유]이기 때문입니다
-----------------------------------------------
왜냐하면 AI가 사진을 분류하고 데이트를 기록하는
2025년에 맞는 기술을 적용했기 때문입니다
```

### 한 줄 포지셔닝

> **"AI가 알아서 해주는 2025 커플앱"**

### 대안 포지셔닝 문구

1. "수동 기록 시대 끝. AI가 추억을 정리해드립니다"
2. "비트윈 졸업하고, 2025년 커플앱으로"
3. "사진 정리? AI한테 시키세요"

### 포지셔닝 맵

```
             High Tech (AI/자동화)
                    |
       +------------+------------+
       |            |            |
       |            |    ★       |  <-- 우리 위치
       |            |  [Couple   |
       |            |    App]    |
-------+------------+------------+-------
Low    |            |            |  High
기능   | [커플]     |            |  기능
       |            |  [비트윈]  |
       |            |            |
       |  [러블]    |[데이트팝]  |
       +------------+------------+
                    |
             Low Tech (수동)
```

**축 설명**:
- X축: 기능 다양성 (Low ↔ High)
- Y축: 기술 수준 (수동 ↔ AI 자동화)

**우리 위치**: 높은 기술(AI) + 핵심 기능 집중 = **스마트 미니멀**

### 경쟁사 대비 포지셔닝

| 브랜드 | 포지셔닝 | 핵심 메시지 |
|--------|----------|------------|
| **우리** | AI 자동화 커플앱 | "AI가 알아서 해줌" |
| 비트윈 | 커플 소통 대명사 | "둘만의 공간" |
| 러블 | 귀여운 일상 기록 | "소소한 일상" |
| 커플 | 심플 캘린더 | "D-day 관리" |

## 4. 차별화 전략

### 핵심 차별화 포인트 (3개)

#### 1. AI 자동 분류

- **설명**: 사진을 AI가 자동으로 장소/날짜/인물별 분류
- **증거**: [AI 기술 적용 - 개발 중 데모 영상]
- **고객 가치**: "내가 안 해도 정리가 되네?"

#### 2. 데이트 자동 기록

- **설명**: 함께 간 장소, 날짜가 자동으로 캘린더에 저장
- **증거**: [위치 기반 + AI 인식 기술]
- **고객 가치**: "언제 어디 갔는지 자동 기록"

#### 3. 2025 감성 UX

- **설명**: 미니멀하고 현대적인 UI/UX, MZ 감성
- **증거**: [디자인 프리뷰]
- **고객 가치**: "비트윈보다 예쁘고 신선함"

### 차별화 유형

| 유형 | 내용 | 강도 |
|------|------|------|
| 기술 | AI 자동화 (유일) | 5점 |
| 디자인 | 2025 감성 UI | 4점 |
| 가격 | 핵심 기능 무료 | 4점 |
| 서비스 | 간편한 온보딩 | 3점 |

### 메시지 프레임워크

| 타겟 | 핵심 메시지 | 지원 포인트 | 톤 |
|------|------------|------------|-----|

| 사용할 단어 | 피할 단어 |

## 6. 포지셔닝 검증 체크리스트

```

[x] 일관되게 전달 가능한가?

[x] 확장 가능한가?
    → AI 기반으로 추가 기능 확장 가능
```
