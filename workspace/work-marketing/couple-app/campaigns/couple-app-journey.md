## Stage 1: 인지 (Discovery)

### 고객 상태

| 항목 | 내용 |
|------|------|
| 목표 | 재미있는 콘텐츠 소비 (숏폼 스크롤) |
| 질문 | "이게 뭐지? 커플앱?" |
| 감정 | 호기심, 약간의 의심 |
| 행동 | TikTok 스크롤 중 숏폼 발견 |

### 터치포인트

| 채널 | 유형 | 우선순위 |
|------|------|---------|
| TikTok 숏폼 | 바이럴 콘텐츠 | High |
| Instagram Reels | 바이럴 콘텐츠 | High |
| 에브리타임 | 커뮤니티 포스트 | Medium |
| 친구 추천 | 입소문 | Medium |

### 고객이 묻는 질문

1. "이거 뭐야? 비트윈 같은 건가?"
2. "AI가 사진 정리해준다고? 진짜?"
3. "무료야?"

### 필요한 콘텐츠

| 콘텐츠 | 형식 | 목적 |
|--------|------|------|
| 훅 영상 | 15초 숏폼 | 스크롤 멈추게 |
| 공감 영상 | 15초 숏폼 | "나도 그래" 느낌 |
| Before/After | 30초 숏폼 | 제품 이해 |

### Pain Points & Opportunities

| Pain Point | Opportunity |
|------------|-------------|
| 광고 같으면 스킵 | 밈/공감 콘텐츠로 광고 느낌 제거 |
| 3초 안에 안 끌리면 패스 | 강력한 훅으로 주목 |
| "또 커플앱?" 피로감 | "AI가 달라" 차별화 |

### 성공 지표

| 지표 | 목표 |
|------|------|
| 조회수 | 영상당 5,000+ |
| 완시율 | 40%+ |
| 프로필 클릭 | 2%+ |

## Stage 2: 고려 (Consideration)

### 고객 상태

| 항목 | 내용 |
|------|------|
| 목표 | "이 앱이 나한테 맞는지" 확인 |
| 질문 | "비트윈보다 뭐가 나아?" |
| 감정 | 관심, 비교 심리 |
| 행동 | 프로필 방문, 다른 영상 탐색 |

### 비교 요소

고객이 비교하는 것들:
1. 비트윈과 뭐가 다른지
2. 진짜 AI가 정리해주는지 (데모)
3. 무료인지, 얼마인지
4. 다른 사람들 반응 (댓글, UGC)

### 터치포인트

| 채널 | 유형 | 우선순위 |
|------|------|---------|
| TikTok 프로필 | 추가 영상 탐색 | High |
| 댓글 | 다른 사람 반응 | High |
| 인스타 프로필 | 브랜드 이미지 | Medium |

### 고객이 묻는 질문

1. "비트윈보다 뭐가 좋아?"
2. "진짜 자동으로 되는 거 맞아?"
3. "다른 사람들 반응은 어때?"
4. "개인정보 괜찮아?"

### 필요한 콘텐츠

| 콘텐츠 | 형식 | 목적 |
|--------|------|------|
| 기능 데모 | 30초 숏폼 | 실제 작동 증명 |
| 비교 영상 | 30초 숏폼 | 비트윈 vs 우리 |
| 사용자 반응 | UGC | 소셜 프루프 |
| FAQ 영상 | 30초 숏폼 | 의문 해소 |

### 경쟁사 대응

| 경쟁사 접점 | 우리의 대응 |
|------------|------------|
| "비트윈도 충분해" | AI 자동화는 비트윈에 없음 |
| "새 앱 깔기 귀찮아" | "5분이면 OK" 강조 |
| "데이터 옮기기 귀찮아" | "새로 시작" 프레이밍 |

### Pain Points & Opportunities

| Pain Point | Opportunity |
|------------|-------------|
| 비교할 정보가 부족 | 명확한 비교 콘텐츠 제공 |
| 다른 사람 반응 궁금 | UGC, 댓글 활용 |
| "어차피 비슷할 거야" | 데모로 차별점 증명 |

## Stage 3: 결정 (Decision)

### 고객 상태

| 항목 | 내용 |
|------|------|
| 목표 | 사전예약 할지 말지 결정 |
| 질문 | "지금 예약해야 하나? 나중에 해도 되지 않아?" |
| 감정 | 관심↑, 망설임 |
| 행동 | 랜딩페이지 방문, 폼 확인 |

### 결정 장벽 (Objections)

| 장벽 | 대응 방법 |
|------|----------|
| "나중에 해도 되겠지" | 선착순 1,000명 평생 무료 (긴급성) |
| "개인정보 괜찮아?" | 온디바이스 AI, 보안 강조 |
| "남친/여친도 깔아야 하는데" | 초대 링크로 같이 예약 혜택 |
| "어차피 안 쓰게 될 듯" | 무료니까 일단 예약 (리스크 없음) |

### 결정 촉진 요소

1. **긴급성**: "선착순 1,000명만 평생 무료"
2. **희소성**: "남은 자리 237명" 카운터
3. **소셜 프루프**: "이미 763명이 예약했어요"
4. **리스크 제거**: "무료, 이메일만 입력"

### 터치포인트

| 채널 | 유형 | 우선순위 |
|------|------|---------|
| 랜딩페이지 | 전환 페이지 | High |
| 바이오 링크 | 연결 | High |
| 영상 내 CTA | 행동 유도 | High |

### 필요한 콘텐츠

| 콘텐츠 | 형식 | 목적 |
|--------|------|------|
| 랜딩페이지 | 웹페이지 | 전환 |
| CTA 영상 | 15초 숏폼 | 긴급성 전달 |
| "마지막 기회" | 스토리 | 긴급성 |

## Stage 4: 예약 (Sign-up)

### 고객 상태

| 항목 | 내용 |
|------|------|
| 목표 | 빠르게 예약 완료 |
| 기대 | "간단하겠지?" |
| 감정 | 기대, 약간의 설렘 |
| 행동 | 이메일 입력, 제출 |

### 예약 경험 체크리스트

- [x] 폼 필드 최소화 (이메일만)
- [x] 1클릭 완료 가능
- [x] 확인 메시지 즉시 표시
- [x] 확인 이메일 자동 발송
- [x] 다음 단계 안내 (친구 초대)

### 터치포인트

| 채널 | 유형 | 우선순위 |
|------|------|---------|
| 예약 폼 | Tally | High |
| 확인 페이지 | Thank You | High |
| 확인 이메일 | Mailchimp | High |

### 즉각적인 가치 전달

1. "축하해요! 선착순 안에 들었어요" 메시지
2. 추천 코드 즉시 발급
3. 친구 초대 시 앰배서더 자격 안내

### Pain Points & Opportunities

| Pain Point | Opportunity |
|------------|-------------|
| 폼이 복잡하면 이탈 | 이메일 1개만 |
| "예약하고 뭐 하지?" | 즉시 다음 액션 제시 (공유) |
| 확인이 안 오면 불안 | 즉시 확인 이메일 |

## Stage 5: 대기 (Waiting)

### 고객 상태

| 항목 | 내용 |
|------|------|
| 목표 | 런칭까지 관심 유지 |
| 기대 | "언제 나오지?" |
| 감정 | 기대 → (시간 지나면) 잊힘 |
| 행동 | 이메일 오픈, 커뮤니티 참여 |

### 관심 유지 전략

1. **이메일 시퀀스**: 주 1회 개발 소식
2. **커뮤니티**: 오픈채팅 초대
3. **Building in Public**: SNS로 과정 공유
4. **앰배서더 미션**: 친구 초대 챌린지

### 터치포인트

| 채널 | 유형 | 우선순위 |
|------|------|---------|
| 이메일 | 뉴스레터 | High |
| 카카오 오픈채팅 | 커뮤니티 | High |
| 인스타 스토리 | Building in Public | Medium |
| TikTok | 개발 비하인드 | Medium |

### Pain Points & Opportunities

| Pain Point | Opportunity |
|------------|-------------|
| 시간 지나면 잊힘 | 주기적 이메일 리마인드 |
| "아직도 안 나와?" | 개발 진행 상황 공유 |
| 관심 떨어짐 | 커뮤니티로 참여감 유지 |

## Touchpoint Summary

### 전체 터치포인트 맵

| 단계 | 온라인 | 오프라인 | 우선순위 |
|------|--------|---------|---------|
| 인지 | TikTok, Reels, 에브리타임 | 친구 추천 | TikTok |
| 고려 | 프로필, 댓글, UGC | - | 프로필 |
| 결정 | 랜딩페이지 | - | LP |
| 예약 | Tally 폼, 이메일 | - | 폼 |
| 대기 | 이메일, 커뮤니티, SNS | - | 이메일 |
| 런칭 | 앱스토어, 이메일 | - | 앱스토어 |

### 핵심 터치포인트 (Top 5)

1. **TikTok 숏폼** - 인지 단계
   - 역할: 첫 발견, 스크롤 멈추기
   - 최적화 방향: 훅 강화, 완시율 최적화

2. **랜딩페이지** - 결정 단계
   - 역할: 전환, 사전예약 완료
   - 최적화 방향: CRO, 긴급성/희소성

3. **확인 이메일** - 예약 단계
   - 역할: 신뢰 구축, 다음 액션 유도
   - 최적화 방향: 즉시 발송, 추천 코드

4. **이메일 시퀀스** - 대기 단계
   - 역할: 관심 유지, 런칭 대기
   - 최적화 방향: 오픈율, 가치 제공

5. **커뮤니티** - 대기 단계
   - 역할: 팬덤 빌딩, 바이럴 확산
   - 최적화 방향: 활성화율, 참여
