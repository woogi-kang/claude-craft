# Couple App Pre-Registration Email Sequence

## Sequence Overview

| 항목 | 내용 |
|------|------|
| 유형 | Pre-Registration Welcome + Waiting |
| 목적 | 사전예약자 관계 구축 & 런칭 시 설치 유도 |
| 이메일 수 | 5통 |
| 기간 | 사전예약 ~ 런칭 (약 12주) |
| 트리거 | 사전예약 완료 |
| 타겟 | 20-30대 커플 (사전예약자) |
| 톤 | 친근, MZ 감성, 공감, Building in Public |

### Success Metrics

| 지표 | 목표 |
|------|------|
| 오픈율 | 40%+ |
| 클릭률 | 10%+ |
| 추천 전환 | 20%+ (공유 → 신규 예약) |
| 런칭 시 설치율 | 60%+ |

---

## AI Era (2025) 적용 원칙

```
1. 짧은 이메일 (모바일 3스크롤 이내)
2. Building in Public 콘텐츠 (개발 비하인드)
3. 참여형 콘텐츠 (투표, 질문)
4. 커뮤니티 유도 (오픈채팅/Discord)
5. 추천 프로그램 강화 (바이럴)
```

## Sequence Logic

### Branching Rules

```
Email 1 발송 (Day 0)
     │
     ├─ 오픈 O + 공유 O ──▶ Email 3으로 스킵 (Week 2)
     │
     ├─ 오픈 O + 공유 X ──▶ Email 2 (Day 3)
     │
     └─ 오픈 X ──▶ Email 1 리마인더 (Day 1, 다른 제목)
                        │
                        └─▶ Email 2 (Day 4)
```

### Exit Conditions

- [ ] 수신 거부 (Unsubscribe)
- [ ] 하드 바운스 (잘못된 이메일)
- [ ] 런칭 후 앱 설치 완료 → Post-Install 시퀀스로 전환

### Segment Rules

| 세그먼트 | 조건 | 다음 액션 |
|---------|------|----------|
| Super Engaged | 모든 이메일 오픈 + 공유 완료 | VIP 대우, 런칭 알림 우선 |
| Engaged | 오픈율 50%+ | 일반 시퀀스 |
| Cold | 오픈 1회 이하 | 런칭 당일만 발송 |
