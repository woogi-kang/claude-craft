## AI Era (2025) 테스트 원칙

```
1. 빠른 반복: 1-2주 단위 테스트 사이클
2. 숏폼 중심: 조회수/완시율이 1차 지표
3. 낮은 표본도 OK: 트래픽 적어도 방향성 파악
4. 직감 + 데이터: 완벽한 통계보다 빠른 의사결정
5. 한 번에 하나: 변수 하나씩만 테스트
```

## Test 1: 숏폼 훅 유형 테스트

### Test Overview

| 항목 | 내용 |
|------|------|
| 테스트명 | Hook-Type-Test |
| 테스트 대상 | TikTok/Instagram 숏폼 |
| 테스트 변수 | 훅 유형 (질문형 vs 공감형 vs 도발형) |
| 시작일 | Week 1 |
| 예상 종료일 | Week 2 |
| 상태 | 계획 |

### Hypothesis (가설)

> 만약 **공감형 훅**을 사용하면,
> **완시율과 프로필 방문율**이 **증가**할 것이다.
> 왜냐하면 **타겟(대학생 커플)이 "이거 나야" 반응에 더 공감**하기 때문이다.

### Test Design

| 훅 유형 | 예시 | 콘텐츠 수 |
|--------|------|----------|
| A. 질문형 | "커플 사진 몇 장이야? 정리한 건 0개?" | 5개 |
| B. 공감형 | "사진 정리 포기한 사람 손" | 5개 |
| C. 도발형 | "비트윈 쓰는 사람 보세요" | 5개 |

### Success Metrics

| 지표 | 현재(추정) | 목표 | 측정 방법 |
|------|----------|------|----------|
| 완시율 | 20% | 35%+ | 플랫폼 인사이트 |
| 프로필 방문율 | 2% | 4%+ | 플랫폼 인사이트 |
| 댓글/저장 | 1% | 3%+ | 플랫폼 인사이트 |

### Implementation

```
Week 1:
- Day 1-2: A타입 5개 업로드
- Day 3-4: B타입 5개 업로드
- Day 5-7: C타입 5개 업로드

Week 2:
- 각 유형별 성과 집계
- 우승 유형 선정
- 해당 유형으로 집중 제작
```

### Decision Framework

| 결과 | 액션 |
|------|------|
| B(공감형) 완시율 가장 높음 | 공감형 훅 집중 제작 |
| A(질문형) 프로필 방문율 가장 높음 | 질문형 훅 집중 제작 |
| 차이 없음 | 3가지 믹스 유지, 다른 변수 테스트 |

## Test 2: LP 헤드라인 테스트

### Test Overview

| 항목 | 내용 |
|------|------|
| 테스트명 | LP-Headline-Test |
| 테스트 대상 | 사전예약 랜딩페이지 |
| 테스트 변수 | 헤드라인 (AI 강조 vs 문제 해결 vs 비교) |
| 시작일 | Week 2 |
| 예상 종료일 | Week 4 |
| 상태 | 계획 |

### Hypothesis (가설)

> 만약 **문제 해결 헤드라인**을 사용하면,
> **전환율**이 **증가**할 것이다.
> 왜냐하면 **타겟의 Pain Point를 직접 건드려서 공감을 유발**하기 때문이다.

### Control vs Variant

| 버전 | 헤드라인 | 관점 |
|------|---------|------|
| A (Control) | AI가 알아서 정리해주는 커플앱 | AI/솔루션 강조 |
| B (Variant) | 사진 정리? 이제 AI한테 시키세요 | 문제 해결 |
| C (Variant) | 비트윈 졸업, 2025 커플앱으로 | 비교/대체 |

### Sample Size Calculation

| 항목 | 값 |
|------|-----|
| 현재 전환율 (추정) | 20% |
| 최소 탐지 효과 | 25% (상대적) |
| 필요 표본 (변형당) | ~400명 |
| 예상 일일 LP 방문 | 30-50명 |
| **예상 테스트 기간** | **8-14일** |

### Success Metrics

| 지표 | 현재(추정) | 목표 (최소) | 측정 방법 |
|------|----------|------------|----------|
| LP 전환율 | 20% | 25%+ | GA4 |
| 폼 시작율 | 30% | 40%+ | 폼 이벤트 |
| 이탈률 | 60% | 50% 이하 | GA4 |

### Implementation (Carrd A/B)

```
옵션 1: 수동 A/B (무료)
- 짝수 날짜: A 버전
- 홀수 날짜: B 버전
- 수동 트래픽 분리

옵션 2: URL 파라미터
- coupleapp.co?v=a (A 버전)
- coupleapp.co?v=b (B 버전)
- 숏폼에서 번갈아 링크

옵션 3: 시간대 분리
- 오전: A 버전
- 오후: B 버전
```

## Notes & Best Practices

### 0원 예산에서 A/B 테스트

```
1. 수동 분리: 짝수/홀수 날짜, 오전/오후
2. URL 파라미터: ?v=a, ?v=b 로 트래킹
3. 시간 기반: 일주일씩 교대
4. 플랫폼별: TikTok A버전, Reels B버전
```

### 적은 트래픽에서 의사결정

```
- 통계적 유의성보다 "방향성" 중시
- 100명 정도면 큰 차이는 보임
- 20% 이상 차이나면 방향 선택
- 비슷하면 더 오래 테스트
```

### 테스트 실패 시

```
1. 둘 다 안 좋음 → 완전히 새로운 방향 테스트
2. 차이 없음 → 더 큰 변화 테스트 (미세 조정 X)
3. 외부 요인 → 기록해두고 다시 테스트
```
