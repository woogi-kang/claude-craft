## Pre-Launch Checklist

### 1. Hero Section

- [ ] **헤드라인 3초 테스트**: 처음 보는 사람이 3초 내 이해하는가?
  - 테스트 방법: 친구에게 3초 보여주고 "뭐하는 앱이야?" 질문
  - 기준: "AI가 사진 정리해주는 커플앱" 정도로 대답하면 통과

- [ ] **CTA 버튼 대비 색상**: 배경과 확실히 구분되는가?
  - 권장: 밝은 배경 + 진한 버튼 (예: 흰 배경 + 보라색 버튼)
  - 체크: 5m 떨어져서 봐도 버튼 위치가 보이는가?

- [ ] **Above the Fold 완성**: 스크롤 없이 핵심 요소 다 보이는가?
  - 필수 요소: 헤드라인, 서브헤드라인, CTA 버튼, 폼
  - 모바일 기준: 상단 500px 내 모든 요소 배치

- [ ] **모바일 Hero 최적화**: 모바일에서 텍스트 잘 읽히는가?
  - 헤드라인 폰트: 최소 24px
  - CTA 버튼: 터치 영역 44px 이상

- [ ] **데모 GIF/영상**: AI 기능 시각적으로 보여주는가?
  - 권장: 3-5초 루프 GIF
  - 용량: 1MB 이하 (로딩 속도)

- [ ] **긴급성 요소**: 선착순/카운터 표시되는가?
  - 형식: "선착순 1,000명 중 237명 남음"
  - 위치: CTA 버튼 바로 아래

---

### 2. Trust Elements (신뢰 요소)

- [ ] **숫자 표시**: 구체적인 숫자로 신뢰 구축
  - "763명 사전예약 완료" (실시간 또는 정기 업데이트)
  - 초기: 목업 숫자 사용 가능 (100명부터 시작)

- [ ] **사전예약자 반응**: 기대평/댓글 형태 후기
  - 최소 3개 이상
  - 실명+간단한 정보 (수진, 대학생 커플)

- [ ] **스팸 없음 보장**: 이메일 안심 문구
  - "스팸 없음. 런칭 알림만 보내드려요."
  - 위치: 폼 아래

- [ ] **개인정보 링크**: 개인정보처리방침 링크 배치
  - Footer에 최소 배치

---

### 3. Form Optimization (폼 최적화)

- [ ] **필드 수 최소화**: 이메일 1개만
  - 황금률: 필드 1개 추가 시 전환율 ~10% 감소
  - 사전예약 단계: 이메일만 수집

- [ ] **Placeholder 명확**: "이메일 주소 입력"
  - 라벨보다 placeholder 권장 (깔끔함)

- [ ] **버튼 텍스트 명확**: 행동 지향적 문구
  - Good: "평생 무료로 시작하기"
  - Bad: "제출", "확인"

- [ ] **에러 메시지 친절**: 무슨 문제인지 명확히
  - "이메일 형식이 올바르지 않아요"
  - "이미 예약된 이메일이에요"

- [ ] **Thank You 페이지**: 예약 완료 후 페이지 준비
  - 확인 메시지
  - 공유 유도 (추천 프로그램)
  - 다음 단계 안내

- [ ] **확인 이메일**: 자동 발송 설정
  - 예약 확인
  - 추천 링크 포함

---

### 4. Technical Requirements (기술)

- [ ] **로딩 속도 3초 이하**
  - 도구: Google PageSpeed Insights
  - 목표: 모바일 점수 70+
  - 이미지 최적화 필수

- [ ] **모바일 반응형**
  - 375px (iPhone SE) ~ 428px (iPhone 14 Pro Max) 테스트
  - Carrd는 기본 반응형이나 미리보기 확인

- [ ] **GA4 설치**
  - 이벤트: page_view, form_start, form_submit
  - Goal 설정: 사전예약 완료

- [ ] **Meta Pixel 설치** (선택)
  - 나중에 리타게팅 광고용
  - 0원 예산이면 일단 스킵 가능

- [ ] **UTM 파라미터 지원**
  - 트래픽 소스 추적용
  - 예: ?utm_source=tiktok&utm_campaign=launch

---

### 5. Copy Quality (카피)

- [ ] **타겟 언어 사용**: MZ 감성, 친근한 톤
  - Good: "이거 혹시 나...?"
  - Bad: "귀사의 니즈에 맞춤형 솔루션"

- [ ] **혜택 중심 표현**: Feature → Benefit 변환
  - Feature: AI 자동 분류
  - Benefit: 5시간 정리 → 5초로 끝

- [ ] **CTA 명확**: 클릭하면 뭐가 되는지 명확
  - "평생 무료로 시작하기" (혜택 명시)

- [ ] **Objection 처리**: FAQ로 반론 해소
  - "정말 무료인가요?"
  - "스팸 오나요?"
  - "개인정보 안전한가요?"

- [ ] **스캔 가능 구조**: 불릿, 소제목, 강조
  - 긴 문단 X
  - 짧은 문장 + 불릿 포인트

---

### 6. Mobile Specific (모바일 전용)

- [ ] **Sticky CTA**: 하단 고정 버튼
  - 스크롤해도 항상 보이는 CTA
  - Carrd에서 설정 가능

- [ ] **터치 영역**: 버튼/링크 44px 이상
  - 손가락으로 쉽게 탭 가능

- [ ] **폼 키보드 최적화**: 이메일 입력 시 @ 키보드
  - input type="email" 설정

- [ ] **세로 스크롤만**: 가로 스크롤 발생 X
  - 모든 요소가 화면 너비 내

## Post-Launch Monitoring

### Week 1

- [ ] **전환율 확인**
  - 목표: 25%+ (LP 방문 → 예약)
  - 측정: GA4 Events / Carrd Analytics

- [ ] **히트맵 분석** (선택)
  - 도구: Hotjar 무료 티어
  - 확인: 어디서 스크롤 멈추는지

- [ ] **이탈 지점 파악**
  - 스크롤 깊이: 어디까지 보는가?
  - 폼 이탈: 입력 시작 vs 완료

- [ ] **첫 A/B 테스트 시작**
  - 우선순위 1: 헤드라인 테스트
  - 도구: Google Optimize (무료) 또는 Carrd A/B

---

### Week 2-4

- [ ] **A/B 테스트 결과 분석**
  - 통계적 유의성: 최소 100 전환 이상
  - 승자 버전 확정

- [ ] **우승 버전 적용**
  - 더 나은 버전으로 교체

- [ ] **다음 테스트 계획**
  - 헤드라인 → CTA → 긴급성 순서

- [ ] **전환율 추이 모니터링**
  - 주간 리포트
  - 이상 징후 체크

## Common Issues & Fixes

| 문제 | 가능한 원인 | 해결 방법 |
|------|------------|----------|
| 높은 이탈률 | Hero가 약함, 로딩 느림 | 헤드라인 테스트, 이미지 최적화 |
| 낮은 폼 시작율 | CTA 눈에 안 띔 | 버튼 색상/크기 변경, 위치 조정 |
| 폼 이탈 높음 | 필드 많음, 에러 불친절 | 필드 줄이기, 에러 메시지 개선 |
| 낮은 스크롤 깊이 | 콘텐츠 지루함 | Problem 섹션 강화, 비주얼 추가 |
| 모바일 전환 낮음 | 터치 영역 작음 | 버튼 크기 키우기, Sticky CTA |

## Launch Checklist Summary

### D-3 (3일 전)

- [ ] 모든 섹션 카피 확정
