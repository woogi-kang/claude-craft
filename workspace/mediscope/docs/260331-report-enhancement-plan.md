# MediScope 리포트 고도화 계획

> 작성일: 2026-03-31
> 목적: 기술 SEO 점수 중심 → **실제 마케팅 성과 + 실행 가능한 인사이트** 중심으로 리포트 전환

---

## 현재 상태

- 21항목 기술 SEO 진단 (Technical 15 + GEO/AEO 5 + 다국어 1)
- 단일 종합점수 (0-100, A~F 등급)
- 지역 벤치마크 (경쟁 병원 비교)
- Before/After 프로젝트 비교
- PDF 리포트 생성

---

## Phase 1: 데이터 재구성 (기존 데이터 활용)

기존 크롤링/체크 데이터를 재분류하여 새로운 관점의 리포트 생성.
신규 API 연동 없이 즉시 구현 가능.

### 1-1. 포털별 SEO 스코어카드

**개요**: 단일 종합점수 → 포털/플랫폼별 개별 점수로 분해

**구조**:
```
┌────────────┬───────┬──────────────────────────┐
│ 플랫폼      │ 점수   │ 핵심 이슈                  │
├────────────┼───────┼──────────────────────────┤
│ Google     │ 78/100│ 구조화 데이터 누락           │
│ Naver      │ 45/100│ 네이버 웹마스터 미등록        │
│ Baidu      │ 12/100│ 중국어 페이지 없음           │
│ Yahoo JP   │ 31/100│ 일본어 hreflang 미설정       │
├────────────┼───────┼──────────────────────────┤
│ ChatGPT    │ 노출됨 │ 3개 질문 중 2개 언급         │
│ Perplexity │ 미노출 │ 0개 질문 중 0개 언급         │
└────────────┴───────┴──────────────────────────┘
```

**점수 산출 기준**:

| 포털 | 반영 체크항목 |
|------|-------------|
| Google | sitemap, robots.txt, 구조화 데이터, Core Web Vitals, 모바일, HTTPS, canonical, OG |
| Naver | OG 태그, 한국어 콘텐츠 품질, 네이버 웹마스터 meta 태그, 사이트맵 |
| Baidu | 중국어 페이지 존재, ICP 비안 meta, 중국 CDN 힌트, baidu-site-verification |
| Yahoo JP | 일본어 hreflang, 일본어 콘텐츠, 모바일 최적화 |
| AI 검색 | ChatGPT/Perplexity 언급 여부 (기존 GEO/AEO 체크) |

**구현 범위**:
- [ ] 기존 21개 체크항목을 포털별로 매핑하는 스코어링 로직
- [ ] 포털별 점수 계산 서비스 (`services/portal_scorer.py`)
- [ ] 리포트 UI 섹션 추가
- [ ] PDF 템플릿에 포털별 스코어카드 추가

**난이도**: 낮음 | **예상 임팩트**: 높음

### 1-2. 다국어 준비도 대시보드

**개요**: 병원 웹사이트가 각 언어권 환자를 받을 준비가 되어있는지 페이지별 진단

**구조**:
```
┌───────────┬──────┬──────┬──────┬──────┐
│ 페이지      │  KO  │  EN  │  JA  │  ZH  │
├───────────┼──────┼──────┼──────┼──────┤
│ 메인       │  ✅  │  ✅  │  ❌  │  ❌  │
│ 시술 소개   │  ✅  │  ⚠️  │  ❌  │  ❌  │
│ 의사 소개   │  ✅  │  ❌  │  ❌  │  ❌  │
│ 가격 안내   │  ✅  │  ❌  │  ❌  │  ❌  │
│ 예약 폼    │  ✅  │  ✅  │  ❌  │  ❌  │
│ 후기/리뷰   │  ✅  │  ❌  │  ❌  │  ❌  │
├───────────┼──────┼──────┼──────┼──────┤
│ 준비도     │ 100% │ 33%  │  0%  │  0%  │
└───────────┴──────┴──────┴──────┴──────┘
```

**탐지 로직**:
- 크롤링된 URL에서 `/en/`, `/ja/`, `/zh/` 경로 또는 서브도메인 패턴
- hreflang 태그에서 지원 언어 추출
- 페이지 본문 언어 감지 (charset, lang 속성, 문자 빈도)
- 페이지 유형 분류 (메인/시술/의사/가격/예약/후기)

**구현 범위**:
- [ ] 크롤링 시 다국어 페이지 매핑 로직 추가
- [ ] 페이지 유형 자동 분류기 (URL 패턴 + 콘텐츠 기반)
- [ ] 다국어 준비도 점수 계산
- [ ] 리포트 UI + PDF 섹션

**난이도**: 낮음 | **예상 임팩트**: 높음 (의료관광 특화)

### 1-3. 환자 여정(Patient Journey) 퍼널 리포트

**개요**: 검색 → 신뢰 → 비교 → 예약, 4단계 환자 여정별 점수

**구조**:
```
🔍 발견(72) → 🤝 신뢰(45) → ⚖️ 비교(28) → 📅 예약(61)

[발견] 검색 노출은 양호하나
[신뢰] 의사 프로필/자격증 미노출 ← 가장 취약
[비교] 가격/후기 정보 부족
[예약] 예약 폼은 존재하나 다국어 미지원
```

**단계별 매핑**:

| 단계 | 반영 항목 |
|------|----------|
| 발견 | SEO 기술 점수, 사이트맵, robots, 검색 노출 |
| 신뢰 | E-E-A-T (의사 자격증), HTTPS, 구조화 데이터, 리뷰 존재 |
| 비교 | 가격 페이지 존재, 시술 상세, Before/After 갤러리, FAQ |
| 예약 | 예약 폼, 다국어 폼, 연락처 접근성, 모바일 최적화 |

**구현 범위**:
- [ ] 환자 여정 단계별 체크항목 매핑 정의
- [ ] 단계별 가중 점수 계산
- [ ] 퍼널 시각화 UI 컴포넌트
- [ ] "가장 취약한 단계" 하이라이트 + 추천 액션

**난이도**: 낮음 | **예상 임팩트**: 높음 (비개발자도 이해하기 쉬움)

### 1-4. 시간 흐름 트렌드 리포트 (구독 고객용)

**개요**: 구독 고객의 점수 변화를 시각화하여 개선 효과를 증명

**구조**:
```
종합 점수: 45 → 62 → 78 (+73% 개선)

포털별 변화:
  Google:  62 → 71 → 78 ↑
  Naver:   30 → 42 → 45 ↑
  AI 검색:  0 →  0 →  1 (첫 노출!)

개선된 항목: sitemap 추가(+8), HTTPS 적용(+5), 구조화 데이터(+7)
아직 미개선: 다국어 페이지, 네이버 웹마스터
```

**구현 범위**:
- [ ] `score_history` 테이블 기반 트렌드 API
- [ ] 항목별 변화 diff 계산
- [ ] Recharts 기반 시계열 차트 컴포넌트
- [ ] 구독 리포트 이메일에 트렌드 요약 포함

**난이도**: 낮음 (데이터 이미 존재) | **예상 임팩트**: 중간

## Phase 2: 검색 순위 분석 (신규 API 연동)

실제 검색 엔진에서의 노출/순위를 측정하는 핵심 기능.

### 2-1. 키워드 검색 순위 체크

**개요**: 시술 키워드 × 포털 × 국가 매트릭스로 실제 검색 순위 측정

**구조**:
```
┌──────────────┬────────┬────────┬────────┬──────────┐
│ 키워드        │ Google │ Naver  │ Baidu  │ Yahoo JP │
├──────────────┼────────┼────────┼────────┼──────────┤
│ 포텐자 강남   │  3위   │  미노출  │   -    │    -     │
│ 눈성형 강남   │  7위   │  12위   │   -    │    -     │
│ rhinoplasty  │ 15위   │   -    │  미노출 │   8위    │
│ gangnam      │        │        │        │          │
│ 鼻整形 江南   │   -    │   -    │  22위  │    -     │
└──────────────┴────────┴────────┴────────┴──────────┘

● 1-3위  ● 4-10위  ● 11-20위  ● 20+/미노출
```

**키워드 자동 생성 로직**:
1. 크롤링에서 시술명/진료과목 추출 (메뉴, 페이지 타이틀)
2. 지역명 + 시술명 조합 (예: "포텐자 강남", "리프팅 논현")
3. 타겟 언어별 자동 번역 (EN/JA/ZH)
4. 의료관광 특화 키워드 추가 ("korea plastic surgery", "韩国整形" 등)

**API 옵션**:

| 포털 | API/방식 | 비용 |
|------|---------|------|
| Google | Google Custom Search API / SerpAPI / DataForSEO | $5-50/1000쿼리 |
| Naver | Naver Search API (광고 API로 순위 추정) | 무료~저가 |
| Baidu | DataForSEO Baidu SERP | $10-30/1000쿼리 |
| Yahoo JP | SerpAPI (Yahoo Japan) | $5-30/1000쿼리 |

**구현 범위**:
- [ ] 키워드 자동 추출 서비스 (`services/keyword_extractor.py`)
- [ ] 키워드 번역 매핑 (시술명 한/영/일/중 사전)
- [ ] SERP API 통합 서비스 (`services/serp_checker.py`)
- [ ] DB: `keyword_rankings` 테이블 (hospital_id, keyword, portal, rank, checked_at)
- [ ] 리포트 UI: 키워드 × 포털 매트릭스 테이블
- [ ] PDF 템플릿 업데이트

**난이도**: 중간 | **예상 임팩트**: 매우 높음

### 2-2. 경쟁사 키워드 갭 분석

**개요**: 동일 키워드에서 경쟁사는 노출되는데 우리는 안 되는 키워드 식별

**구조**:
```
📊 키워드 갭 분석: "포텐자 강남"

  1위: B피부과         ← 경쟁사
  2위: C클리닉         ← 경쟁사
  3위: 당신의 병원 ✓

⚠️ 당신이 없고 경쟁사가 있는 키워드:
  ┌──────────────┬───────────┬──────────┐
  │ 키워드        │ 경쟁사 최고순위 │ 기회 점수  │
  ├──────────────┼───────────┼──────────┤
  │ 보톡스 강남   │ B피부과 1위  │ ★★★★★  │
  │ 리프팅 논현   │ C클리닉 3위  │ ★★★★☆  │
  │ 쥬베룩 강남   │ B피부과 5위  │ ★★★☆☆  │
  └──────────────┴───────────┴──────────┘
```

**기회 점수 계산**: 검색량 × (1 - 경쟁 강도) × 시술 관련성

**구현 범위**:
- [ ] 벤치마크 병원의 키워드 순위도 함께 수집
- [ ] 키워드 갭 계산 로직
- [ ] 기회 점수 산출 알고리즘
- [ ] 리포트 UI: 갭 분석 테이블 + 기회 하이라이트

**난이도**: 중간 (2-1 완료 후 확장) | **예상 임팩트**: 매우 높음

## Phase 3: 유니크 인사이트 (차별화 기능)

경쟁 SEO 도구에 없는 의료관광 특화 분석.

### 3-1. 소셜 프루프 스코어 (리뷰/평판 통합 분석)

**개요**: 여러 플랫폼에 흩어진 병원 리뷰/평점을 통합 분석

**구조**:
```
📱 소셜 프루프 종합: 4.2/5.0 (리뷰 847건)

┌──────────────┬──────┬───────┬──────────────┐
│ 플랫폼        │ 평점  │ 리뷰수 │ 최근 30일     │
├──────────────┼──────┼───────┼──────────────┤
│ Google Maps  │ 4.3  │  312  │ +12건, 4.5   │
│ 네이버 플레이스│ 4.1  │  256  │ +8건, 3.9 ↓  │
│ 강남언니      │ 4.4  │  189  │ +5건, 4.6    │
│ 바비톡       │ 3.8  │   67  │ +2건, 4.0    │
│ 小红书       │ 4.5  │   23  │ +1건         │
└──────────────┴──────┴───────┴──────────────┘

💬 리뷰 키워드 분석:
  긍정: "친절" (34%), "자연스러운" (28%), "만족" (22%)
  부정: "대기시간" (15%), "가격" (12%)

⚠️ 경고: 네이버 평점 최근 하락 추세 (4.3 → 3.9)
```

**데이터 수집 방법**:
- Google Maps: Places API (공식)
- 네이버 플레이스: 네이버 검색 API + 크롤링
- 강남언니/바비톡: 구조화 데이터 또는 크롤링
- 小红书: 키워드 검색 기반 언급 수

**구현 범위**:
- [ ] 플랫폼별 리뷰 수집 서비스 (`services/review_aggregator.py`)
- [ ] 리뷰 감성 분석 (키워드 빈도 기반, 간단한 NLP)
- [ ] DB: `review_snapshots` 테이블
- [ ] 리포트 UI: 리뷰 통합 대시보드
- [ ] 경고 알림: 평점 급락 시 구독자 알림

**난이도**: 중간~높음 | **예상 임팩트**: 매우 높음

### 3-2. 예약 전환 퍼널 분석

**개요**: 병원 웹사이트의 예약 전환 경로를 분석하여 이탈 지점 식별

**구조**:
```
🔄 예약 전환 퍼널 (크롤링 기반 추정)

  메인 페이지 ──→ 시술 페이지 ──→ 의사 소개 ──→ 예약 폼
     100%           ???           ???          ???

  📋 전환 요소 체크:
  ┌──────────────────────┬────────┬──────────────────┐
  │ 요소                  │ 상태    │ 권장사항            │
  ├──────────────────────┼────────┼──────────────────┤
  │ CTA 버튼 (메인)       │ ✅ 있음 │                    │
  │ CTA 버튼 (시술 페이지) │ ❌ 없음 │ 각 시술 페이지에 추가 │
  │ 전화번호 클릭 가능     │ ⚠️ 텍스트│ tel: 링크로 변환    │
  │ 카카오톡 상담         │ ✅ 있음 │                    │
  │ LINE 상담            │ ❌ 없음 │ 일본 환자용 추가     │
  │ WeChat 상담          │ ❌ 없음 │ 중국 환자용 추가     │
  │ 예약 폼 필드 수       │ ⚠️ 12개 │ 5개 이하로 줄이기    │
  │ 예약 폼 다국어        │ ❌ KO만 │ EN/JA/ZH 추가      │
  │ 실시간 채팅           │ ❌ 없음 │ 도입 권장           │
  │ 가격 표시            │ ❌ 없음 │ 시술별 가격대 표시    │
  └──────────────────────┴────────┴──────────────────┘

  💡 전환율 예상: 현재 ~1.2% → 개선 시 ~3.5% (업계 상위)
```

**탐지 로직**:
- CTA 버튼: `<a>`, `<button>` 태그에서 "예약", "상담", "문의", "book", "contact" 텍스트
- 전화번호: `tel:` 링크 vs 일반 텍스트
- 메신저: 카카오/LINE/WeChat 링크 또는 위젯 스크립트
- 폼 분석: `<form>` 내 `<input>` 필드 수
- 가격: "가격", "비용", "price", "料金" 등 키워드 포함 페이지

**구현 범위**:
- [ ] CTA/전환 요소 탐지 체크 모듈 (`checks/conversion_elements.py`)
- [ ] 폼 분석 로직 (필드 수, 다국어 여부)
- [ ] 메신저 위젯 탐지
- [ ] 전환율 벤치마크 데이터 (업계 평균)
- [ ] 리포트 UI: 퍼널 시각화 + 체크리스트

**난이도**: 중간 | **예상 임팩트**: 높음

### 3-3. AI 검색 노출 심층 분석 (GEO 확장)

**개요**: ChatGPT, Perplexity, Google AI Overview에서 병원이 어떻게 언급되는지 심층 분석

## Phase 5: AI 콘텐츠 엔진 + 이미지 생성

피부과 wiki 수준 데이터를 활용한 AI 콘텐츠/이미지 생성 서비스.
진단(문제 발견) → 해결(콘텐츠 생성)으로 서비스 범위 확장.

### 5-1. 피부과 전문 AI 콘텐츠 에이전트

**개요**: 보유 데이터(시술 원리/방법/부작용/사후관리 등)를 학습한 AI가
병원 맞춤형 SEO 콘텐츠를 생성

**활용 데이터**:
```
현재 보유 (procedure_details):
├── 시술 원리 (principle)
├── 시술 방법 (method)
├── 소요 시간 (duration)
├── 부작용 (side_effects)
├── 통증 수준 (pain_level)
├── 다운타임 (downtime)
├── 사후 관리 (post_care)
├── 평균 가격 (average_price)
├── 4개 언어 번역 (procedure_intl)
└── 6개국 가격 비교 (price_comparison)
```

**생성 가능한 콘텐츠 유형**:

```
A. 블로그 포스트 (SEO 최적화)
   ─────────────────────────
   Input:  병원명 + 시술명 + 타겟 키워드
   Output: SEO 최적화된 블로그 초안 (2000자+)

   예시: "강남 A피부과의 포텐자 시술 가이드"
   - 포텐자란? (시술 원리 → DB에서 가져옴)
   - 시술 과정 (방법 → DB에서 가져옴)
   - 소요 시간/다운타임 (DB 데이터)
   - 부작용과 주의사항 (DB 데이터)
   - 사후 관리 팁 (DB 데이터)
   - A피부과만의 차별점 (병원 크롤링 데이터에서 추출)
   - 가격 안내 (DB + 해외 비교)
   + SEO 메타 태그, 구조화 데이터 코드 포함

B. FAQ 콘텐츠 (음성검색 + AI 검색 대응)
   ─────────────────────────
   Input:  시술명
   Output: 시술별 FAQ 20개 + Schema.org FAQ 마크업

   예시: "포텐자 FAQ"
   - 포텐자 시술 시간은 얼마나 걸리나요? → DB: duration
   - 포텐자 부작용이 있나요? → DB: side_effects
   - 포텐자 후 관리는 어떻게 하나요? → DB: post_care
   - 포텐자 가격은 얼마인가요? → DB: average_price
   - 포텐자와 프락셀 차이는? → DB: 시술 간 비교 생성

C. 다국어 시술 소개 페이지
   ─────────────────────────
   Input:  시술명 + 타겟 언어
   Output: 해당 언어로 완성된 시술 소개 페이지 HTML

   - 이미 4개 언어 번역 데이터 보유
   - 언어별 SEO 키워드 자동 삽입
   - 해당 국가 대비 한국 가격 장점 강조 (price_comparison 활용)

   예시 (일본어):
   "ポテンツァ施術ガイド - 韓国での費用は日本の約60%"

D. SNS 콘텐츠 세트
   ─────────────────────────
   Input:  시술명 + 플랫폼(인스타/블로그/小红书)
   Output: 플랫폼별 최적화된 포스트 세트 (5~10개)

   - 인스타그램: 카드뉴스 텍스트 + 해시태그
   - 네이버 블로그: SEO 최적화 글
   - 小红书: 중국어 체험형 콘텐츠
   - TikTok/YouTube: 영상 스크립트 + 썸네일 카피

E. 의료관광 가이드 콘텐츠
   ─────────────────────────
   Input:  타겟 국가 + 시술 카테고리
   Output: 해당 국가 환자를 위한 종합 가이드

   - 한국에서 {시술} 받기: 비용 비교 (price_comparison 활용)
   - 시술 전 준비사항
   - 한국 입국 → 시술 → 회복 타임라인
   - 사후 관리 및 귀국 후 주의사항
   - 한국어 기본 회화 (병원 방문용)

F. 경쟁 분석 기반 콘텐츠 전략
   ─────────────────────────
   Input:  병원 스캔 결과 + 경쟁사 스캔 결과
   Output: "경쟁사는 있고 우리는 없는 콘텐츠" 리스트 + 초안

   - 시술별 콘텐츠 완성도 매트릭스(3-9)와 연동
   - 부족한 콘텐츠를 AI가 자동으로 초안 생성
   - 우선순위: 검색량 높은 키워드 관련 콘텐츠부터
```

**기술 구현**:
```
┌──────────────────────────────────────────┐
│ AI 콘텐츠 에이전트 아키텍처                  │
│                                          │
│  MediScope DB (피부과 데이터)               │
│       ↓                                  │
│  프롬프트 엔진                              │
│  ├── 시술 데이터 주입 (RAG 방식)             │
│  ├── 병원 크롤링 데이터 주입                  │
│  ├── SEO 키워드/규칙 주입                   │
│  └── 의료광고법 가이드라인 주입               │
│       ↓                                  │
│  LLM (Claude API / GPT-4o)               │
│       ↓                                  │
│  후처리                                    │
│  ├── 의료광고법 금지 표현 필터링              │
│  ├── SEO 메타태그 자동 삽입                 │
│  ├── 구조화 데이터 코드 생성                 │
│  └── 플랫폼별 포맷 변환                     │
│       ↓                                  │
│  콘텐츠 초안 (사람 검수 필요)                 │
└──────────────────────────────────────────┘
```

**의료광고법 안전장치**:
- 금지 표현 자동 필터 ("최고", "최상", "100%", "보장", "완벽" 등)
- "개인에 따라 결과가 다를 수 있습니다" 자동 삽입
- 부작용/합병증 고지 자동 포함
- 생성 후 규정 준수 체크 자동 실행

**구현 범위**:
- [ ] 콘텐츠 생성 API 엔드포인트 (`/api/content/generate`)
- [ ] 프롬프트 템플릿 엔진 (시술별 × 콘텐츠유형별 × 언어별)
- [ ] DB 데이터 → 프롬프트 컨텍스트 변환 서비스
- [ ] 의료광고법 필터링 모듈
- [ ] 생성 콘텐츠 저장/관리 테이블 (`generated_contents`)
- [ ] 프론트엔드 UI: 콘텐츠 생성 대시보드

**난이도**: 중간 | **예상 임팩트**: 매우 높음 (핵심 수익 기능)

### 5-2. AI 의료 이미지 생성

**개요**: 시술 설명용 일러스트, 인포그래픽, SNS 카드를 AI로 생성

**생성 가능한 이미지 유형**:

```
A. 시술 과정 인포그래픽
   ─────────────────
   Step 1: 상담 → Step 2: 마취 → Step 3: 시술 → Step 4: 관리
   (DB의 method 데이터를 단계별로 시각화)

B. Before/After 일러스트 (실제 사진 대체)
   ─────────────────
   의료광고법상 실제 환자 사진 규제가 강함
   → AI 일러스트로 대체하면 규제 회피 + 시각적 효과

   예: 피부 톤 개선, 주름 감소, 모공 축소 등
       실사가 아닌 의학 일러스트 스타일

C. SNS 카드 이미지
   ─────────────────
   시술명 + 핵심 정보 요약 카드
   인스타그램 슬라이드용 (1080×1080)

   "포텐자 시술이란?"
   "시술 시간: 30분 | 다운타임: 3-5일 | 통증: ●●○○○"

D. 다국어 배너
   ─────────────────
   언어별 마케팅 배너 자동 생성
   "韓国でポテンツァ - 日本の60%の費用で" (일본어)
   "Korea Potenza - 60% less than US prices" (영어)

E. 비교 차트 이미지
   ─────────────────
   시술 간 비교, 국가 간 가격 비교를
   시각적 차트 이미지로 생성
```

**기술 옵션**:
```
┌──────────────┬──────────┬──────────────────────────┐
│ 도구          │ 비용      │ 적합한 이미지 유형           │
├──────────────┼──────────┼──────────────────────────┤
│ HTML → 이미지  │ $0       │ 인포그래픽, 카드, 차트, 배너 │
│ (Playwright)  │ (이미 있음)│ → 가장 현실적인 1순위        │
│               │          │                          │
│ Gemini 이미지  │ 무료~저가 │ 일러스트, 시술 과정 시각화   │
│ DALL-E 3      │ $0.04/장  │ 일러스트, 의학 다이어그램    │
│ Flux/SD       │ 셀프호스팅│ 대량 생성 시 비용 절감       │
│ Canva API     │ $월정액   │ 템플릿 기반 배너/카드        │
└──────────────┴──────────┴──────────────────────────┘

추천: HTML → Playwright 스크린샷 (1순위, 비용 0)
     + Gemini 이미지 생성 (2순위, 일러스트용)
```

**구현 범위**:
- [ ] HTML 템플릿 기반 이미지 생성 서비스 (Playwright 활용)
  - 인포그래픽 템플릿 (시술 과정, 비교 차트, 가격 비교)
  - SNS 카드 템플릿 (인스타 슬라이드, 블로그 썸네일)
  - 다국어 배너 템플릿
- [ ] AI 일러스트 생성 연동 (Gemini / DALL-E)
  - 시술 과정 일러스트
  - 피부 상태 시각화 (모공, 주름, 톤 등)
- [ ] 이미지 생성 API (`/api/content/image`)
- [ ] 생성 이미지 Supabase Storage 저장
- [ ] 프론트엔드: 이미지 생성/미리보기/다운로드 UI

**난이도**: 중간 | **예상 임팩트**: 높음

### 6-1. 성형외과 확장

**데이터 수집 범위**:
```
시술 카테고리 (예상):
├── 눈 성형 (쌍꺼풀, 눈매교정, 하안검, 상안검)
├── 코 성형 (코끝, 콧대, 매부리, 들창코)
├── 안면윤곽 (광대, 사각턱, 턱끝, V라인)
├── 가슴 성형 (확대, 축소, 재건)
├── 지방 성형 (지방흡입, 지방이식)
├── 리프팅 (실리프팅, 절개리프팅)
├── 눈썹/이마 (이마거상, 눈썹리프팅)
├── 입술/입 (입술필러, 입꼬리)
├── 귀 성형 (돌출귀, 부이어)
├── 바디 (복부성형, 팔리프팅)
├── 모발 이식
└── 흉터 치료
```

**성형외과 특화 SEO 체크 추가**:
- Before/After 갤러리 (필수, 전환율 핵심)
- 의사 자격/경력 페이지 (E-E-A-T 핵심)
- 수술 영상/리얼 후기
- 의료사고 보험 안내
- 무료 상담 예약 시스템

**데이터 수집 방법**:
- [ ] 대한성형외과학회 시술 정보 스크래핑
- [ ] 강남언니/바비톡 시술 정보 구조화
- [ ] 글로벌 성형외과 가이드 (ISAPS, ASPS) 참조
- [ ] 각국 성형외과 가격 조사 (의료관광 비교)
- [ ] 전문의 자문을 통한 데이터 검증

### 6-2. 치과 확장

**데이터 수집 범위**:
```
시술 카테고리 (예상):
├── 임플란트 (단일, 다중, 전악, All-on-4/6)
├── 라미네이트/비니어
├── 치아 미백
├── 치열 교정 (교정, 투명교정, 설측교정)
├── 충치 치료 (레진, 인레이, 크라운)
├── 잇몸 치료 (스케일링, 잇몸수술)
├── 발치 (사랑니, 매복)
├── 보철 (틀니, 브릿지)
└── 소아 치과
```

**치과 특화 SEO 체크 추가**:
- 임플란트 브랜드/재료 표기
- 보증 기간 안내
- 응급 치료 안내
- 치료 과정 단계별 안내
- 치과 보험/결제 안내

**치과가 유망한 이유**:
- 임플란트 의료관광 규모가 성형 다음으로 큼
- 가격 차이가 매우 큼 (미국 대비 한국 60~70% 저렴)
- 치료 기간이 길어 콘텐츠 수요 높음
- 동남아(베트남, 태국) 환자도 한국으로 유입 증가

---

### 6-3. 건강검진 확장

**데이터 수집 범위**:
```
검진 패키지:
├── 기본 건강검진
├── 종합 건강검진
├── 프리미엄 건강검진
├── 암 특화 검진 (위, 대장, 간, 폐, 유방, 자궁)
├── 심혈관 검진
├── 뇌 검진 (MRI, MRA)
├── 소화기 검진
├── 갑상선 검진
└── 기업 단체 검진
```

**건강검진이 유망한 이유**:
- 외국인 건강검진 시장 급성장 (특히 중국, 동남아)
- 병원 입장에서 객단가 높음 (100만원~500만원)
- 패키지 비교가 핵심 → 구조화된 데이터 가치 높음
- 검진 + 관광 결합 상품 수요

**건강검진 특화 SEO 체크**:
- 검진 패키지 비교 페이지
- 검진 항목 상세 설명
- 외국어 검진 결과지 발급
- 통역/코디네이터 서비스
- 공항 픽업 + 숙소 패키지

---

### 확장 시 공통 작업

각 진료과 확장 시 반복되는 작업:

```
1. 데이터 수집/구조화
   - [ ] procedure_categories 추가 (진료과별 카테고리)
   - [ ] procedures 추가 (개별 시술/검진)
   - [ ] procedure_details 추가 (상세 정보)
   - [ ] procedure_intl 추가 (4개 언어 번역)
   - [ ] price_comparison 추가 (6개국 가격)

2. SEO 체크 확장
   - [ ] 진료과 특화 체크 모듈 추가
   - [ ] 스코어링 가중치 조정
   - [ ] 리포트 템플릿 분기

3. AI 콘텐츠 확장
   - [ ] 프롬프트 템플릿 추가 (진료과별)
   - [ ] 의료광고법 필터 확장 (진료과별 규제 차이)
   - [ ] 이미지 템플릿 추가

4. 영업/마케팅
```
