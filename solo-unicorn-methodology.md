## 1. 배경: 왜 지금 1인 유니콘인가?

### 1.1 패러다임 전환

2025년은 **AI 코딩 에이전트가 개발을 해결한 원년**으로 기록된다. 지난 수십 년간 스타트업의 최대 병목이었던 "개발"이 더 이상 병목이 아니게 되었다. 그러나 이것은 시작에 불과하다.

> 💡 **핵심 통찰**: 개발이 병목에서 해방된 것이 아니라, **가장 쉽고 싼 재화**가 된 세상이 도래했다.

### 1.2 새로운 공식

| 과거 | 현재 |
|------|------|
| 아이디어 → 팀 빌딩 → 개발 → 제품 | 아이디어 = 제품 |
| 질문 → 리서치 → 분석 → 해답 | 질문 = 해답 |
| 영감 → 기획 → 실행 → 결과 | 영감 = 결과 |

### 1.3 1인 유니콘의 정의

```
1인 유니콘 = 1명의 창업자 + n개의 AI 에이전트 + 무한 확장 가능한 시스템
```

**구성 요소:**
- **인간(1명)**: 비전, 의사결정, 창의성, 최종 판단
- **AI 에이전트(n개)**: 실행, 자동화, 24/7 운영, 확장
- **시스템**: 복리로 성장하는 지식/프로세스 자산

⚠️ **주의**: 1인 유니콘은 "혼자 모든 것을 한다"가 아니라, "1명이 무한한 실행력을 갖는다"를 의미한다.

## 2. 핵심 개념

### 2.1 Compound Engineering (복리 엔지니어링)

복리 엔지니어링은 1인 유니콘의 핵심 작동 원리다.

```
Day 1: 문제 해결 → 문서화 → 스킬 생성
Day 2: 스킬 재사용 → 새 문제 해결 → 새 스킬 생성
Day N: 스킬 조합 → 복잡한 문제 자동 해결
```

**복리 공식:**
```
가치(t) = 초기_스킬 × (1 + 학습률)^시간
```

| 요소 | 설명 | 예시 |
|------|------|------|
| 초기_스킬 | 첫 번째로 만든 자동화 | 세금계산서 발행 스크립트 |
| 학습률 | 스킬이 다른 스킬을 생성하는 비율 | 세금 → 재무제표 → 투자 실사 |
| 시간 | 축적 기간 | 주 단위로 가속 |

✅ **성공 지표**: 한 번 해결한 문제를 AI가 다시 해결할 수 있다면 복리가 작동 중이다.

### 2.2 컨텍스트 응집 (Context Cohesion)

AI에게 효과적으로 일을 시키는 핵심은 **컨텍스트의 응집**이다.

```
분산된 컨텍스트 (❌)          응집된 컨텍스트 (✅)
─────────────────           ─────────────────
iOS 앱 리포 ───┐              company-monorepo/
Web 앱 리포 ───┼─ AI가        ├── products/
API 서버 리포 ─┤  어디를      ├── operations/
회계 스프레드 ─┤  봐야할지    ├── marketing/
계약서 드라이브┘  모름        └── docs/
                              ↑
                              AI가 전체 맥락을 
                              한눈에 파악
```

### 2.3 디렉토리 = 조직도

전통적 회사에서 조직도가 책임과 권한을 정의했듯이, 1인 유니콘에서는 **디렉토리 구조가 AI 에이전트의 역할을 정의**한다.

| 전통 조직 | 디렉토리 | AI 에이전트 역할 |
|----------|----------|-----------------|
| CEO | `/CLAUDE.md` | 전략적 의사결정 보조 |
| CTO | `/products/` | 제품 개발, 기술 결정 |
| CFO | `/operations/finance/` | 재무 분석, 리포트 |
| COO | `/operations/` | 운영 자동화 |
| CMO | `/marketing/` | 콘텐츠, 분석 |
| Legal | `/operations/legal/` | 계약 검토, 규정 |
| HR | `/operations/hr/` | 채용 프로세스 |

**구현 예시 (슬래시 명령어):**

```markdown
# /financial-report

## 트리거
월말 또는 수동 실행

## 입력
- 대상 월: YYYY-MM (기본값: 이전 월)

## 실행 단계
1. /operations/finance/receipts/{대상월}/ 스캔
2. /operations/finance/invoices/{대상월}/ 집계
3. ledger.json과 교차 검증
4. 손익계산서 템플릿으로 PDF 생성
5. /operations/finance/statements/{대상월}/ 저장
6. Notion 업로드 (MCP)
7. Slack 알림 (MCP)

## 출력
- statements/{대상월}/pnl.pdf
- statements/{대상월}/summary.md
- Notion 페이지 링크

## 5. MCP 통합 전략

### 5.1 멀티 LLM 아키텍처

1인 유니콘의 핵심은 **외부 유료 서비스 대신 AI 모델을 직접 활용**하는 것이다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    Multi-LLM Routing Strategy                    │
└─────────────────────────────────────────────────────────────────┘

                        ┌─────────────┐
                        │   Request   │
                        └──────┬──────┘
                               │
                    ┌──────────┴──────────┐
                    │   Task Classifier   │
                    └──────────┬──────────┘
                               │
         ┌─────────────────────┼─────────────────────┐
         │                     │                     │
         ▼                     ▼                     ▼
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│  Gemini CLI     │  │  Claude Code    │  │  Codex CLI      │
│  ─────────────  │  │  ─────────────  │  │  ─────────────  │
│  • OCR/Vision   │  │  • 추론/분석    │  │  • 코드 생성    │
│  • 이미지 분석  │  │  • 긴 문서 처리 │  │  • 리팩토링     │
│  • 빠른 처리    │  │  • 계약서 검토  │  │  • 코드 리뷰    │
│  • 무료 티어    │  │  • 전략 수립    │  │  • PR 작성      │
└─────────────────┘  └─────────────────┘  └─────────────────┘
```

**모델별 최적 용도**:

| 작업 | 권장 모델 | 이유 |
|------|----------|------|
| 영수증/청구서 OCR | **Gemini CLI** | 네이티브 멀티모달, 무료 1,000건/일 |
| 한국어 OCR | Gemini 3 Pro / Claude | 아시아권 언어 강점 |
| 계약서 검토 | **Claude** | 100만 토큰 컨텍스트, 정확도 |
| 코드 생성 | **Codex** | 코딩 특화, 7시간+ 자율 작업 |
| 콘텐츠 생성 | Claude / Gemini | 품질 + 비용 균형 |
| 빠른 분류 | Gemini Flash-Lite | 저비용, 고속 |

    ;;
  review)
    # 멀티 LLM 리뷰 (Claude + Gemini + Codex 합의)
    claude "코드 리뷰" --file "$2" > /tmp/claude_review.md
    gemini "코드 리뷰" --file "$2" > /tmp/gemini_review.md
    codex "코드 리뷰" --file "$2" > /tmp/codex_review.md
    claude "세 리뷰를 종합해서 합의된 피드백 생성" \
      /tmp/claude_review.md /tmp/gemini_review.md /tmp/codex_review.md
    ;;
  *)
    echo "Usage: smart-ai.sh [ocr|contract|code|review] [file] [prompt]"
    ;;
esac
```

```
현재 보유:

멀티 LLM 통합 (우선 추가):
├── gemini-ocr-mcp     → Gemini 기반 OCR (무료)
│   └── pip install gemini-ocr-mcp
├── gemini-cli 직접 호출 → Bash 통해 활용
└── codex 직접 호출     → 코드 생성 자동화

운영 확장 (필요시 추가):
├── notion-mcp         → 지식베이스 연동
├── google-drive       → 계약서/영수증 원본
├── slack              → 알림/협업
├── gmail              → 영수증 자동 수집
├── google-calendar    → 마일스톤/리마인더
└── stripe             → 결제 데이터 연동
```

```json
// .mcp.json

```
