# PPT Validation Skill

리서치 단계에서 수집한 데이터의 신뢰성과 정확성을 검증하는 Skill입니다.
Structure Skill 이전에 반드시 실행되어 탄탄한 데이터 기반을 확보합니다.

## 검증 프레임워크

### 1. 출처 신뢰도 검증 (Source Credibility Check)

출처의 신뢰도를 5단계로 평가합니다:

| Level | 등급 | 출처 유형 | 설명 |
|-------|------|----------|------|
| **5** | 최고 | 정부기관, 학술논문, 공인기관 보고서 | 공신력 최상위 |
| **4** | 높음 | 주요 언론사, 산업 리서치 기관 | 검증된 전문 기관 |
| **3** | 보통 | 전문 블로그, 기업 공식 자료 | 일반적 신뢰 수준 |
| **2** | 낮음 | 일반 블로그, 커뮤니티 글 | 추가 검증 필요 |
| **1** | 미검증 | 출처 불명, 익명 자료 | 사용 비권장 |

### 2. 팩트체크 프로세스 (5단계 검증)

| 단계 | 검증 항목 | 검증 방법 | 합격 기준 |
|------|----------|----------|----------|
| **1차** | 출처 존재 여부 | URL 접근성 확인, 원문 존재 확인 | 원문 접근 가능 |
| **2차** | 데이터 최신성 | 발행일 확인 | 발행일 기준 6개월 이내 권장 |
| **3차** | 교차 검증 | 최소 2개 이상 독립 출처 확인 | 2+ 출처에서 일치 |
| **4차** | 맥락 검증 | 인용이 원문 맥락과 일치 여부 | 맥락 왜곡 없음 |
| **5차** | 수치 검증 | 통계 데이터 산출 방법 확인 | 방법론 명확 |

### 3. 검증 결과 태깅

검증 결과는 다음 JSON 구조로 저장합니다:

```json
{
  "validation_result": {
    "item_id": "R001",
    "status": "verified|partially_verified|unverified|disputed",
    "credibility_score": 0.92,
    "credibility_level": 4,
    "checks_passed": [
      "source_exists",
      "recent_data",
      "cross_verified"
    ],
    "checks_failed": [],
    "warnings": [
      "Single source - recommend additional verification"
    ],
    "recommendation": "safe_to_use|use_with_caveat|do_not_use",
    "verified_at": "2025-01-04T10:30:00Z",
    "notes": "검증 상세 메모"
  }
}
```

### 4. 검증 상태 정의

| 상태 | 설명 | 사용 권장 |
|-----|------|---------|
| `verified` | 모든 검증 통과 | 자유롭게 사용 |
| `partially_verified` | 일부 검증 통과 | 주의하여 사용 |
| `unverified` | 검증 불가 | 사용 비권장 |
| `disputed` | 상충되는 정보 존재 | 사용 금지 |

## 발표 유형별 검증 기준

발표 목적에 따라 요구되는 검증 수준이 다릅니다:

| 발표 유형 | 최소 신뢰도 | 교차검증 요구 | 최신성 요구 |
|----------|------------|--------------|------------|
| **투자 피치** | Level 4+ | 3개 이상 필수 | 3개월 이내 |
| **기술 세미나** | Level 3+ | 2개 이상 필수 | 6개월 이내 |
| **내부 보고** | Level 3+ | 선택적 | 12개월 이내 |
| **교육 자료** | Level 4+ | 2개 이상 필수 | 24개월 이내 |
| **마케팅 자료** | Level 3+ | 2개 이상 권장 | 6개월 이내 |

## 검증 워크플로우

```
Research Skill에서 데이터 수신
         │
         ▼
   [1차] 출처 존재 확인
         │
         ├─ 실패 → 출처 불명 태그 → 사용 비권장
         │
         ▼
   [2차] 최신성 확인
         │
         ├─ 경고 → 오래된 데이터 주의 태그
         │
         ▼
   [3차] 교차 검증
         │
         ├─ 경고 → 단일 출처 주의 태그
         │
         ▼
   [4차] 맥락 검증
         │
         ├─ 실패 → 맥락 왜곡 태그 → 사용 금지
         │
         ▼
   [5차] 수치 검증
         │
         ├─ 경고 → 방법론 불명확 태그
         │
         ▼
   최종 검증 결과 생성
         │
         ▼
   Structure Skill로 전달
```

## 핵심 기능

### 1. source_verify (출처 검증)
출처의 존재 여부와 접근성을 확인합니다.

**수행 작업:**
- URL 접근성 테스트
- 원문 페이지 존재 확인
- 출처 기관/저자 확인

### 2. freshness_check (최신성 확인)
데이터의 최신성을 발표 유형에 맞게 검증합니다.

**수행 작업:**
- 발행일 추출
- 발표 유형별 기준 적용
- 최신성 점수 산출

### 3. cross_verify (교차 검증)
동일한 정보를 다수의 독립 출처에서 확인합니다.

**수행 작업:**
- 유사 정보 검색
- 독립 출처 개수 확인
- 정보 일치도 분석

### 4. context_verify (맥락 검증)
인용된 내용이 원문 맥락과 일치하는지 확인합니다.

**수행 작업:**
- 원문 전후 문맥 확인
- 인용 정확성 검증
- 의도적 왜곡 여부 확인

### 5. numeric_verify (수치 검증)
통계 데이터의 산출 방법과 정확성을 확인합니다.

**수행 작업:**
- 표본 크기 확인
- 조사 방법론 확인
- 수치 재계산 가능 여부

## 사용 예시

### 예시 1: 데이터 검증 요청
```
사용자: "수집한 자료 팩트체크 해줘"

수행:
1. Research Skill에서 전달받은 모든 research_item 순회
2. 각 항목에 5단계 검증 프로세스 적용
3. validation_result 생성
4. 검증 실패 항목은 경고와 함께 리포트
5. 검증 통과 항목만 Structure Skill로 전달
```

### 예시 2: 특정 출처 검증
```
사용자: "이 통계 데이터 출처 확인해줘"

수행:
1. source_verify로 원문 접근 확인
2. freshness_check로 최신성 확인
3. cross_verify로 교차 검증 시도
4. numeric_verify로 수치 검증
5. 종합 검증 결과 리포트
```

## 다음 단계 연결

검증 완료 후:
1. `verified` 또는 `partially_verified` 항목만 **Structure Skill**로 전달
2. `use_with_caveat` 항목은 사용 시 출처와 함께 주의 문구 포함 권장
3. `do_not_use` 항목은 **Research Skill**에 재조사 요청 가능

## 주의사항

- 검증 실패 항목도 기록 보관 (추후 참조용)
- 검증 결과는 절대적이 아니며, 최종 판단은 사용자 몫
- 빠른 검증이 필요한 경우 1-3차 검증만 수행 가능 (quick_validate 모드)
- 투자 피치 등 중요 발표는 반드시 전체 검증 수행
