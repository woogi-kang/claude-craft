# 2. Validation: 콘텐츠 검증

## 개요

콘텐츠 발행 전 정확성, 브랜드 일관성, 민감도를 검증하여 리스크를 최소화합니다.

## 검증 체크리스트

### 1. 팩트 체크 (Fact Check)

```yaml
fact_check:
  statistics:
    - item: "인용된 통계/수치"
      check:
        - 출처가 명시되어 있는가?
        - 원본 데이터와 일치하는가?
        - 데이터가 최신인가? (6개월 이내 권장)
      severity: critical

  quotes:
    - item: "인용문"
      check:
        - 원문과 정확히 일치하는가?
        - 맥락이 왜곡되지 않았는가?
        - 인용 허가가 필요한가?
      severity: critical

  claims:
    - item: "주장/사실 진술"
      check:
        - 검증 가능한 사실인가?
        - 2개 이상의 출처로 교차 검증되었는가?
        - 과장/허위 표현이 없는가?
      severity: high
```

### 2. 링크 & 멘션 검증

```yaml
link_validation:
  urls:
    - 모든 링크가 유효한가?
    - 올바른 페이지로 연결되는가?
    - 모바일에서도 작동하는가?
    - UTM 파라미터가 올바른가?

  mentions:
    - @멘션이 올바른 계정인가?
    - 멘션 허가를 받았는가?
    - 태그된 사람/브랜드가 적절한가?

  hashtags:
    - 해시태그 철자가 맞는가?
    - 금지/민감한 해시태그가 포함되지 않았는가?
    - 의도와 다른 의미의 해시태그는 없는가?
```

### 3. 브랜드 일관성 체크

```yaml
brand_consistency:
  voice_tone:
    - 정의된 브랜드 보이스와 일치하는가?
    - 톤앤매너가 플랫폼에 적합한가?
    - 이모지 사용이 가이드라인을 따르는가?

  visual_identity:
    - 브랜드 컬러가 올바르게 사용되었는가?
    - 로고 사용이 가이드라인을 따르는가?
    - 폰트/타이포그래피가 일관적인가?

  messaging:
    - 핵심 메시지가 브랜드 전략과 일치하는가?
    - 이전 콘텐츠와 모순되는 내용이 없는가?
    - 경쟁사 비방/비교가 적절한가?
```

### 4. 민감도 체크 (Sensitivity Check)

```yaml
sensitivity_check:
  timing:
    - 현재 발생 중인 재난/사고가 있는가?
    - 추모일/기념일과 충돌하는가?
    - 민감한 시사 이슈와 연관되는가?

  content:
    - 특정 집단을 차별/비하하는 표현이 없는가?
    - 정치적으로 편향된 내용이 아닌가?
    - 종교/문화적으로 민감한 요소가 없는가?

  context:
    - 다른 의미로 해석될 여지가 없는가?
    - 밈/트렌드의 원래 맥락이 적절한가?
    - 이미지/영상에 문제가 될 요소가 없는가?
```

### 5. 중복 & 표절 체크

```yaml
duplication_check:
  internal:
    - 과거 발행 콘텐츠와 유사하지 않은가?
    - 같은 이미지를 너무 자주 사용하지 않았는가?
    - 동일 메시지를 플랫폼별로 다르게 변형했는가?

  external:
    - 타 브랜드 콘텐츠와 유사하지 않은가?
    - 저작권 있는 콘텐츠를 무단 사용하지 않았는가?
    - 밈/트렌드 활용이 적절한가?
```

## 브랜드 세이프티 가이드

### 피해야 할 상황

```yaml
brand_safety_risks:
  high_risk:
    - 논쟁적 정치/사회 이슈 언급
    - 재난/사고 발생 시 일반 마케팅 콘텐츠
    - 검증되지 않은 정보 공유
    - AI 생성 콘텐츠 미표시

  medium_risk:
    - 경쟁사 직접 비교/비방
    - 시사 이슈 밈 활용
    - 논란 있는 인플루언서/브랜드 협업

  low_risk:
    - 브랜드 톤 일시적 변경
    - 새로운 콘텐츠 형식 실험
```

### 위기 징후 모니터링

```yaml
crisis_signals:
  monitor:
    - 부정적 멘션 급증
    - 언론 문의
    - 바이럴 비판 콘텐츠
    - 내부 고발/유출

  response:
    - 즉시 발행 중단
    - 위기관리팀 통보
    - 대응 메시지 준비
    - 상황 모니터링 강화
```

## 자동화 체크 항목

AI/도구로 자동 검증 가능한 항목:

| 항목 | 자동화 가능 | 도구/방법 |
|------|-------------|----------|
| 맞춤법/문법 | ✓ | 맞춤법 검사기 |
| 링크 유효성 | ✓ | URL 체커 |
| 이미지 중복 | ✓ | 역이미지 검색 |
| 해시태그 금지 목록 | ✓ | 키워드 필터 |
| 브랜드 키워드 | ✓ | 텍스트 분석 |
| 감성 분석 | △ | NLP 도구 |
| 팩트 체크 | △ | 수동 검증 필요 |
| 맥락/뉘앙스 | ✗ | 수동 검증 필수 |

## 다음 단계

검증 완료 후:

3. `rejected` → 콘텐츠 폐기 또는 전면 재작성
