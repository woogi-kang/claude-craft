## 콘텐츠 유형별 가이드

### 1. 단일 트윗

```yaml
single_tweet:
  specs:
    max_length: 280자 (한글 기준 약 140자)
    premium_length: 25000자 (Premium+ 구독자)
    media: "이미지 4장 또는 영상 1개"

  structure:
    hook: "첫 문장에 핵심/임팩트"
    body: "간결한 본문"
    cta: "리트윗/답글 유도 (선택)"

  best_practices:
    - 한 트윗 = 한 가지 메시지
    - 첫 단어가 가장 중요
    - 여백 활용 (짧을수록 좋음)
    - 해시태그 1-2개만
```

### 2. 스레드 (Thread)

```yaml
thread:
  specs:
    optimal_length: "5-15 트윗"
    max_length: "25 트윗 이하 권장"

  structure:
    tweet_1: |
      훅 + 스레드 예고
      "🧵 [주제]에 대해 알아야 할 모든 것:"
      또는
      "[충격적 사실/질문]

      스레드 👇"

    tweet_2_n: |
      본문
      - 각 트윗이 독립적으로도 가치 있게
      - 번호 표시 (1/, 2/, 또는 1/10 형식)
      - 한 트윗 = 한 포인트

    last_tweet: |
      CTA + 요약
      - 핵심 요약
      - 리트윗/팔로우 요청
      - 첫 트윗 링크 (셀프 인용)

  formatting:
    - 짧은 문장
    - 줄바꿈으로 가독성
    - 이모지로 포인트 강조
    - 번호 매기기

  example:
    tweet_1: |
      10년간 스타트업을 운영하며 깨달은 것들.

      창업 전 알았으면 좋았을 10가지 진실 🧵
    tweet_2: |
      1/ 첫 번째 아이디어는 거의 항상 틀립니다.

      중요한 건 아이디어가 아니라
      피벗할 수 있는 능력입니다.
    tweet_last: |
      요약:

      • 아이디어보다 실행
      • 팀이 가장 중요
      • 현금흐름에 집착
      • 고객 피드백 > 직감

      도움이 됐다면 RT 부탁드립니다.

      팔로우하시면 매일 창업 인사이트 공유해요.
```

### 3. 인용 트윗 (Quote Tweet)

```yaml
quote_tweet:
  use_cases:
    - 의견/분석 추가
    - 반박/동의 표현
    - 맥락 제공
    - 바이럴 확대

  structure:
    - 원본 트윗에 대한 관점 제시
    - 추가 정보/인사이트
    - 질문으로 토론 유도

  templates:
    agree: "이 말에 100% 동의. 추가하자면..."
    disagree: "존중하지만 다른 관점도 있습니다..."
    expand: "여기에 맥락을 더하면..."
    react: "이건 정말 중요한 포인트."
```

### 4. 투표 (Poll)

```yaml
poll:
  specs:
    options: "2-4개"
    duration: "5분 - 7일"

  use_cases:
    - 의견 수렴
    - 참여 유도
    - 콘텐츠 방향 결정
    - 트렌드 파악

  template: |
    [질문]

    투표해주세요 👇

  best_practices:
    - 명확하고 간결한 질문
    - 상호 배타적인 선택지
    - 결과 공유로 후속 콘텐츠
```

## 트윗 작성 공식

### 구조별 템플릿

**1. 인사이트/팁형**
```
[핵심 인사이트 한 문장]

왜냐하면:
[이유]

실천법:
[구체적 액션]
```

**2. 숫자/리스트형**
```
[주제] 5가지 진실:

1. [포인트 1]
2. [포인트 2]
3. [포인트 3]
4. [포인트 4]
5. [포인트 5]

어떤 것에 가장 공감하시나요?
```

**3. 반박/논쟁형**
```
"[일반적 믿음]"

틀렸습니다.

[반박 논거]

진짜는 이겁니다:
[올바른 관점]
```

**4. 관찰/인사이트형**
```
[관찰한 현상]

이게 의미하는 것:
[해석]

그래서:
[시사점]
```

**5. 경험담형**
```
[상황 설명]

결과:
[어떻게 됐는지]

배운 점:
[교훈]
```

## 훅(Hook) 공식

```yaml
hook_patterns:
  bold_claim: |
    "[강한 주장]."
    예: "대부분의 마케팅 조언은 쓸모없다."

  number: |
    "[숫자] [성과/경험]에서 배운 것:"
    예: "100개 스타트업 투자 후 깨달은 것:"

  question: |
    "[도발적 질문]?"
    예: "왜 아무도 이 말을 안 할까요?"

  story_start: |
    "[시점], [상황]이었습니다."
    예: "2년 전, 저는 파산 직전이었습니다."

  unpopular: |
    "인기 없는 의견:"
    "아무도 동의 안 하겠지만:"

  thread_intro: |
    "[주제]에 대해 알아야 할 모든 것 🧵"
    "[N]가지 [주제] 진실 (스레드 👇)"
```

## 바이럴 요소

```yaml
viral_elements:
  controversy: |
    - 기존 통념에 반하는 주장
    - 논쟁을 유발하는 관점
    - "인기 없는 의견" 프레이밍

  relatability: |
    - 공감할 수 있는 경험
    - "나도 그래" 반응 유도
    - 보편적 고민/상황

  utility: |
    - 즉시 적용 가능한 팁
    - 저장할 가치가 있는 정보
    - 다른 사람에게 공유하고 싶은 내용

  emotion: |
    - 놀라움/충격
    - 영감/동기부여
    - 유머
    - 분노/좌절 (주의해서 사용)

  timing: |
    - 트렌딩 토픽 연결
    - 시사 이슈 코멘트
    - 계절/이벤트 활용
```

## X 알고리즘 최적화

```yaml
algorithm_tips:
  do:
    - 답글에 적극 응답 (첫 1시간 중요)
    - 다른 계정과 진정성 있는 소통
    - 일관된 주제 (알고리즘 학습)

    - 팔로워 언팔로워 반복

  links:
    strategy: "답글에 링크 추가"

```
