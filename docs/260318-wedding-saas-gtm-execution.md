### 1-3. 비교 섹션 — "무료 청첩장 vs 우리 서비스"

| 항목 | 무료 청첩장 | 우리 서비스 |
|------|-----------|------------|
| 가격 | 무료 (종이 청첩장 구매 조건) | 4.9만원 (독립 구매) |
| 템플릿 | 140종+ (정적 레이아웃) | 테마 기반 + 블록 에디터 자유 편집 |
| 인터랙티브 효과 | 없음 또는 기본 | 파티클, 터미널 인트로, 사운드 디자인 |
| 이스터에그/게임 | 없음 | 아케이드 게임, 글리치 테마 등 |
| AI 커스터마이징 | 없음 | 텍스트 명령으로 스타일/레이아웃/이미지 변경 |
| Guest Snap (사진 공유) | 일부 지원 | Resumable 업로드 + 오프라인 큐 + 무제한 |
| 방명록 | 기본 | 실시간 + 비밀 글 + 마스터 비밀번호 |
| 듀얼 오디오 | 없음 | Quick Start + Full Quality 자동 전환 |

### 1-6. FAQ 섹션 (예상 질문 13개)

**가격/결제**

**Q1. 무료로 만들어보고 결제 안 해도 되나요?**
A. 네, 블록 에디터로 청첩장을 완성하는 것까지 완전히 무료입니다. 공유 링크를 발행할 때만 4.9만원이 결제됩니다. 마음에 들지 않으면 결제할 필요 없습니다.

**Q2. 4.9만원 외에 추가 비용이 있나요?**
A. 기본 기능은 모두 포함입니다. AI 커스터마이징 크레딧이 기본 10회 제공되며, 추가가 필요하면 10회/1,900원에 구매할 수 있습니다. 커스텀 도메인 연결은 1,900원입니다. 축의금 송금 수수료는 0원입니다.

**Q3. 환불 가능한가요?**
A. 결제 후 공유 링크를 아직 발행하지 않았다면 100% 환불됩니다. 발행 후 7일 이내에는 50% 환불이 가능합니다. 결혼식 이후에는 환불이 불가합니다.

**기능/사용**

**Q4. 코딩을 못하는데 터미널 인트로 같은 기능을 쓸 수 있나요?**
A. 물론입니다. 모든 기능은 블록 에디터에서 ON/OFF와 스타일 선택만으로 적용됩니다. 코딩 지식이 전혀 필요하지 않습니다.

**Q5. 카카오톡으로 공유하면 미리보기가 잘 나오나요?**
A. 커플 사진과 결혼 정보가 자동 반영된 동적 OG 이미지를 생성합니다. 카카오톡, 문자, 인스타그램 DM 등 어디로 보내도 깔끔한 미리보기가 표시됩니다.

**Q6. 결혼식 당일에 청첩장이 안 열리면 어떡하나요?**
A. CDN 캐싱과 정적 페이지 폴백 시스템을 운영하고 있어 안정적으로 동작합니다. 결혼 시즌에는 업타임 모니터링을 강화합니다.

**Q7. 청첩장 데이터는 얼마나 보관되나요?**
A. 결혼식 이후 2주간 무료로 보관됩니다. 보관 기간 만료 전 데이터를 다운로드하실 수 있으며, 유료 연장도 가능합니다. Guest Snap 사진/영상도 동일하게 결혼식 후 2주간 무료 보관됩니다.

**차별화**

**Q8. 무료 청첩장도 많은데 왜 돈을 내야 하나요?**
A. 무료 서비스는 정적 템플릿 기반입니다. 파티클 이펙트, 터미널 인트로, 이스터에그 게임, AI 커스터마이징 같은 인터랙티브 경험은 무료 서비스에서 제공되지 않습니다. 150명 이상의 하객에게 보여질 청첩장에 커피 10잔 값의 투자로 "이거 어디서 만들었어?"라는 반응을 만들어 보세요.

**Q9. 이스터에그는 뭔가요?**
A. 청첩장 안에 숨겨진 특별한 페이지입니다. 아케이드 게임, 글리치 테마 등 하객들이 발견하면 즐거워할 요소를 넣을 수 있습니다. 커플만의 비밀 메시지 페이지로 활용할 수도 있습니다.

**Q10. AI 커스터마이징은 어떻게 동작하나요?**
A. 채팅창에 "이 섹션 색감을 따뜻하게 바꿔줘", "갤러리를 슬라이드로 바꿔줘", "벚꽃 배경 일러스트 만들어줘" 같은 자연어 명령을 입력하면 AI가 즉시 반영합니다. 1회당 1크레딧이 소모됩니다.

**Q11. 결혼식이 끝나면 청첩장이 사라지나요?**
A. 결혼식 이후 2주간 무료로 보관됩니다. 보관 기간 만료 전 데이터를 다운로드하실 수 있으며, 유료 연장도 가능합니다.

**Q12. 결제 후 수정은 몇 번이나 되나요?**
A. 무제한 수정 가능합니다. 다만 AI 커스터마이징 기능 사용 시에만 크레딧이 차감됩니다. 블록 에디터에서 직접 수정하는 것은 횟수 제한이 없습니다.

**Q13. 인터랙티브 효과 때문에 로딩이 느리진 않나요?**
A. 모든 인터랙티브 효과는 성능 최적화가 완료되어 있습니다. 3G 네트워크에서도 3초 이내에 로딩됩니다. Canvas 렌더링과 Web Audio는 사용자 인터랙션 시점에 지연 로딩되어 초기 로딩에 영향을 주지 않습니다.

### 2-1. disquiet.io — 프로덕트 소개 글 초안

**제목**: 개인 청첩장 사이드 프로젝트, 사업이 되기까지

**본문 초안:**

---

안녕하세요. 올해 결혼을 앞두고 직접 모바일 청첩장을 만든 개발자입니다.

**시작은 단순했습니다.**
시중 모바일 청첩장이 다 비슷해 보여서, "내가 만들면 좀 더 재밌게 할 수 있지 않을까?" 싶어서 Next.js로 직접 만들기 시작했습니다.

**그런데 반응이 예상 밖이었습니다.**
- 터미널 인트로 + 신디사이저 사운드 → "이거 뭐야, 개발자 감성 미쳤다"
- 마우스를 따라다니는 벚꽃 파티클 → "예쁘다 진짜, 어떻게 한 거야?"
- 숨겨진 아케이드 게임 → "이스터에그 찾았다! 이거 공유해야 해"
- 하객 50명 이상이 "이거 어디서 만들었어?"라고 물어봄

주변에서 "이거 팔아봐라"는 말을 진지하게 하기 시작했고, 실제로 몇 분은 "나도 만들어달라"고 요청하셨습니다.

**사이드 프로젝트에서 SaaS로 전환하기로 했습니다.**

핵심 가설:
- "모바일 청첩장은 무료"라는 한국 시장의 강한 인식에도, 무료에서 불가능한 경험이라면 4.9만원을 지불할 사람들이 있을 것
- 1명이 구매하면 150명 이상의 하객에게 자연스럽게 노출되는 바이럴 구조

Build Free, Pay to Publish 전환 메커니즘:
- 블록 에디터로 무료 제작 → 마음에 들면 공유 링크 발행 시 결제
- 결제 장벽을 최대한 뒤로 미뤄서 전환율을 높이는 전략

**기술 스택:**
- Next.js 16 + React 19 + TypeScript
- Supabase Pro (DB, Auth, Realtime, Storage)
- Cloudflare R2 (Guest Snap 대량 업로드)
- Gemini 3.0 Flash (AI 커스터마이징)
- Vercel Pro (배포)

**현재 단계:**
Phase 0 — 지인 대상 Concierge MVP로 시장 반응 검증 중.

피드백, 질문, "나도 써보고 싶다" 모두 환영합니다.

## 5. 초기 가격 실험 설계

### 5-1. A/B 테스트 설계 — "4.9만원" 검증

#### 테스트 가설

**H1: 가격 수용성 테스트**
- 귀무가설: 3.9만원과 4.9만원 간 전환율 차이가 없다
- 대립가설: 3.9만원이 4.9만원 대비 전환율이 유의미하게 높다
- 목적: 1만원 차이가 전환율에 미치는 영향 측정

**H2: 가격 앵커링 테스트**
- 귀무가설: "정가 7.9만원 → 런칭 특가 4.9만원" 표기와 "4.9만원" 단독 표기 간 전환율 차이가 없다
- 대립가설: 앵커링 표기가 전환율을 유의미하게 높인다
- 목적: 할인 프레이밍의 효과 측정

#### 테스트 설계

| 항목 | H1 (가격 수용성) | H2 (가격 앵커링) |
|------|-----------------|-----------------|
| 변수 | 결제 금액 | 가격 표시 방식 |
| 대조군 (A) | 4.9만원 | "4.9만원" 단독 표기 |
| 실험군 (B) | 3.9만원 | "~~7.9만원~~ 런칭 특가 4.9만원" |
| 지표 | 결제 전환율 | 결제 전환율 |
| 최소 표본 | 각 그룹 200명 (제작 시작 기준) | 각 그룹 200명 |
| 기간 | 4-8주 (트래픽 의존) | 4-8주 |
| 유의 수준 | 95% (p < 0.05) | 95% |
| 노출 방식 | 결제 페이지에서 무작위 배정 | 랜딩 + 결제 페이지에서 무작위 배정 |

#### 실행 순서

1. **런칭 1-2주**: 가격 앵커링 테스트(H2) 먼저 실행 (가격 변경 없이 표기만 테스트)
2. **런칭 3-8주**: 충분한 데이터 확보 후 가격 수용성 테스트(H1) 실행
3. **런칭 8주 후**: 두 테스트 결과를 종합하여 최종 가격 결정

#### 의사결정 기준

| 시나리오 | H1 결과 | H2 결과 | 액션 |
|---------|---------|---------|------|
| 둘 다 유의미 | 3.9만원 > 4.9만원 | 앵커링 효과 있음 | "~~5.9만원~~ 3.9만원" 또는 수익 극대화 시뮬레이션 |
| H1만 유의미 | 3.9만원 > 4.9만원 | 앵커링 효과 없음 | 3.9만원으로 인하, 단독 표기 |
| H2만 유의미 | 차이 없음 | 앵커링 효과 있음 | 4.9만원 유지 + 앵커링 표기 |
| 둘 다 무의미 | 차이 없음 | 효과 없음 | 4.9만원 유지 (마진 보호) |

> **주의**: 초기 트래픽이 적으면 통계적 유의성 확보에 시간이 걸립니다.
> 월 방문자 400명 미만이면 테스트 기간을 8주 이상으로 연장하거나,
> 정성적 피드백(사용자 인터뷰)을 병행하여 판단합니다.
