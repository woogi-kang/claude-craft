# 모바일 청첩장 SaaS 사업 컨셉

## 배경

기존 개인 모바일 청첩장 프로젝트(`wedding-invitation`)에 대한 주변 반응이 매우 긍정적.
"유니크하다", "잘 만들었다", "팔아보라"는 피드백을 다수 받음.
이를 SaaS 형태의 모바일 청첩장 에디터로 확장하여 사업화 검토.

## 기존 프로젝트 분석

### 기존 기술 스택 (개인 프로젝트)
- Next.js 16 (App Router) + React 19 + TypeScript
- Tailwind CSS v4 + Framer Motion
- Firebase Firestore (실시간 방명록)
- Google Drive API (Guest Snap 업로드)
- Resend (RSVP 이메일)
- Web Audio API (신디사이저 사운드)
- Canvas API (파티클 이펙트)
- Vercel 배포

### SaaS 전환 기술 스택
- **Vercel Pro** — 호스팅/배포, 서버리스, 커스텀 도메인
- **Supabase Pro** — DB (PostgreSQL), Auth, Realtime (방명록), Storage (갤러리/AI 이미지)
- **Cloudflare R2** — Guest Snap 전용 (하객 사진/영상 대량 업로드, egress 0원)
- **Gemini 3.0 Flash** — AI 커스터마이징 API (저비용 고효율)
- Firebase/Google Drive **제거** → Supabase + R2로 대체

### 핵심 기능
- **13개 섹션**: Hero, 인사말, 커플소개, 인터뷰, 갤러리, 웨딩정보, 위치, 계좌, 방명록, Guest Snap, 공유, 영상, 푸터
- **RSVP 시스템**: 참석 여부 + 식사 선택 + 이메일 알림
- **실시간 방명록**: Firebase onSnapshot, 비밀번호 삭제, 마스터 비밀번호
- **Guest Snap**: Resumable 업로드, Magic Byte 검증, 오프라인 큐(IndexedDB), IP 기반 Rate Limiting
- **듀얼 오디오**: Quick Start + Full Quality 자동 전환
- **마우스 트레일**: Canvas 기반 꽃잎 파티클 (벚꽃/장미/하트)
- **터미널 인트로**: Web Audio 신디사이저 + 순차 애니메이션
- **이스터에그**: 아케이드 게임(`/invitation/arcade`), 글리치 테마(`/invitation/glitch`)

### 차별점 (시중 서비스 대비)
- 시중 청첩장 서비스에서 찾기 어려운 인터랙티브 요소
- 전문 서비스급 업로드 시스템 (Resumable + 오프라인)
- 이스터에그를 통한 엔터테인먼트 요소
- 개발자 감성의 터미널 인트로 + 사운드 디자인
- No-App, No-Login 메신저 최적화 (카카오톡 즉시 접근)

### 한국 시장 특성

**"모바일 청첩장은 무료"라는 강한 소비자 인식**이 한국 시장의 가장 큰 특징.

- 바른손카드: 종이 청첩장 구매/회원가입 시 140여 종 모바일 청첩장 **무료** 제공
- 필수 기능: 계좌번호 복사 + 카카오페이 송금 연동
- 송금 수수료: 과거 1,000~2,000원 논란 → 현재 주요 업체 **수수료 0원**
- 사진 확대 방지, 실시간 무제한 수정이 셀링 포인트


### 기술 전환 요구사항

**Phase 0 — Concierge MVP (1-2주)**
- 테마 1종 + 수동 셋업 (Google Form으로 정보 수집)
- 결제: 토스 송금 (수동)
- 목표: 시장 반응 검증 (5~10건)

**Phase 1 — Self-Serve MVP (6-8주)**
- `constants.ts` → Supabase DB + Row-Level Security
- Firebase → Supabase Realtime (방명록), Supabase Storage (갤러리/AI 이미지)
- Google Drive → Cloudflare R2 (Guest Snap — 하객 사진/영상 대량 업로드)
- 슬러그 기반 동적 라우팅 (`/invitation/{slug}`)
- 블록 에디터 MVP (섹션 드래그앤드롭 + 기본 레이아웃 편집)
- 2-3개 테마 (botanical, modern, glitch)
- 결제 연동 (포트원 — 카카오페이 간편결제 최우선)

**Phase 2 — AI 커스터마이징 + 성장 (2-3개월)**
- AI 커스터마이징 기능 (Gemini 3.0 Flash API 연동, 크레딧 시스템)
- 블록 에디터 고도화 (섹션 내부 레이아웃 자유 편집)
- 테마 추가 (5-10개)
- 분석 대시보드 (방문자 수, RSVP 통계)
- 카카오톡 알림 연동
- 보이스오버 기능
- A/B 가격 테스트

**Phase 3 — 확장 (시장 반응에 따라)**
- B2B 화이트라벨 or 웨딩 플랫폼 (전환 조건 참조)
- 비주얼 에디터 (드래그앤드롭)
- 해외 시장 (일본, 동남아)

## CAC 및 LTV 전략

### 핵심 과제: 1회성 이벤트의 낮은 LTV

결혼은 인생에서 보통 1번 → 구독 모델 적용 어려움 → 지속적 신규 고객 유치 필요.

### Viral Loop 추정

```
1명 청첩장 구매 → 150명 하객에게 공유
→ 하객 중 0.5%가 향후 결혼 예정 (0.75명)
→ 이 중 30%가 우리 서비스 인지/기억 (0.225명)
→ 이 중 20%가 실제 구매 전환 (0.045명)

Viral 계수: ~0.045 (약한 Viral, 단독으로는 성장 불충분)
→ 유료 마케팅 + SEO + 커뮤니티로 보완 필수
```

### 대응 전략

| 전략 | 설명 |
|------|------|
| 입소문(Viral) 활용 | 1명이 100~200명에게 링크 전송 → Blended CAC 절감 |
| 이벤트 초대장 확장 | 돌잔치, 생일, 회갑 등 → LTV 증가 |
| B2B 채널 (Phase 3) | 웨딩플래너/스튜디오 제휴 → 안정적 반복 유입 |
| SEO + 콘텐츠 | "프리미엄 모바일 청첩장" 롱테일 키워드 선점 |
