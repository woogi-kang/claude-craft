# Agent Orchestration

## 원칙
- 독립적인 조회 작업은 병렬 실행 우선
- 도메인 에이전트와 스킬을 적극 활용
- 단순 파일 검색은 Glob/Grep 직접 사용 (Agent 불필요)
- 복잡한 탐색은 Explore 서브에이전트 활용

## 모델 라우팅
- 아키텍처 설계, 복잡한 추론: Opus
- 일반 구현, 코드 작성: Sonnet
- 탐색, 단순 검색, 분류: Haiku

## 컨텍스트 관리
- 큰 파일은 필요한 부분만 읽기 (offset/limit)
- 서브에이전트로 컨텍스트 보호
- 중간 결과를 파일에 저장하여 컨텍스트 절약
