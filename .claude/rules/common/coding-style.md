# Coding Style

## 일반 원칙
- 기존 코드 패턴과 컨벤션을 따른다
- 과도한 엔지니어링을 피한다 — 요청된 변경만 수행
- 불필요한 주석, docstring, 타입 어노테이션을 추가하지 않는다
- 한 번만 쓰이는 헬퍼/유틸리티/추상화를 만들지 않는다
- 비슷한 코드 3줄이 조기 추상화보다 낫다
- 사용하지 않는 코드는 주석 처리 대신 완전히 삭제

## 에러 처리
- 시스템 경계(사용자 입력, 외부 API)에서만 검증
- 발생할 수 없는 시나리오에 대한 에러 핸들링을 추가하지 않는다
- 내부 코드와 프레임워크 보장을 신뢰한다

## 보안
- OWASP Top 10 취약점 방지 (XSS, SQL Injection, Command Injection 등)
- API 키, 시크릿은 절대 커밋하지 않는다
- 보안 취약 코드를 발견하면 즉시 수정
