# 마케팅 프레임워크 요약

## 전략 프레임워크

| 프레임워크 | 용도 | 스킬 | 주요 요소 |
|-----------|------|------|----------|
| **3C** | 시장 분석 | Research | Customer, Competitor, Company |
| **STP** | 타겟팅 & 포지셔닝 | Positioning | Segmentation, Targeting, Positioning |
| **PESO** | 미디어 믹스 | Strategy | Paid, Earned, Shared, Owned |
| **AARRR** | 성장 퍼널 | Funnel | Acquisition, Activation, Retention, Revenue, Referral |
| **SMART** | 목표 설정 | Campaign | Specific, Measurable, Achievable, Relevant, Time-bound |

## 카피라이팅 프레임워크

| 프레임워크 | 구조 | 적합한 용도 |
|-----------|------|------------|
| **AIDA** | Attention → Interest → Desire → Action | 랜딩페이지, 이메일, 긴 광고 |
| **PAS** | Problem → Agitation → Solution | 짧은 광고, 긴급 제안 |
| **BAB** | Before → After → Bridge | 변화 강조, 케이스 스터디 |
| **FAB** | Features → Advantages → Benefits | 제품 설명, 기능 소개 |
| **4P's** | Promise → Picture → Proof → Push | 세일즈 레터 |

## 프레임워크 상세

### 3C 분석

```
┌─────────────────────────────────────────────────┐
│                   Customer                       │
│  - 시장 규모 & 성장률                             │
│  - 고객 니즈 & 페인포인트                         │
│  - 구매 행동 패턴                                 │
├───────────────────┬─────────────────────────────┤
│    Competitor     │         Company             │
│  - 경쟁사 현황     │    - 자사 강점/약점          │
│  - 가격/기능 비교  │    - 차별화 요소             │
│  - 시장 점유율    │    - 핵심 역량               │
└───────────────────┴─────────────────────────────┘
```

### STP 전략

```
Segmentation (세분화)
    │
    ├── 인구통계적: 연령, 성별, 소득
    ├── 지리적: 지역, 도시 규모
    ├── 심리적: 라이프스타일, 가치관
    └── 행동적: 사용량, 충성도
    │
    ▼
Targeting (타겟팅)
    │
    ├── 집중 마케팅: 단일 세그먼트
    ├── 차별화 마케팅: 복수 세그먼트
    └── 비차별화 마케팅: 전체 시장
    │
    ▼
Positioning (포지셔닝)
    │
    └── "[타겟]을 위한 [카테고리]로서 [차별점] 제공"
```

### PESO 미디어 믹스

```
┌─────────────┬─────────────┬─────────────┬─────────────┐
│    Paid     │   Earned    │   Shared    │    Owned    │
├─────────────┼─────────────┼─────────────┼─────────────┤
│ - 검색광고   │ - PR/언론   │ - 소셜미디어 │ - 웹사이트  │
│ - 디스플레이 │ - 리뷰      │ - 커뮤니티   │ - 블로그    │
│ - 소셜광고   │ - 입소문    │ - 포럼       │ - 뉴스레터  │
│ - 인플루언서 │ - SEO       │ - Q&A        │ - 앱        │
├─────────────┼─────────────┼─────────────┼─────────────┤
│ 통제: 높음   │ 통제: 낮음  │ 통제: 중간   │ 통제: 높음  │
│ 신뢰: 중간   │ 신뢰: 높음  │ 신뢰: 중간   │ 신뢰: 중간  │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

### AARRR 퍼널 (Pirate Metrics)

```
┌─────────────────────────────────────────────────┐
│  Acquisition (획득)                              │
│  "사용자를 어떻게 찾는가?"                        │
│  지표: 방문자 수, 채널별 트래픽                   │
├─────────────────────────────────────────────────┤
│  Activation (활성화)                             │
│  "첫 번째 '아하!' 경험"                          │
│  지표: 가입률, 첫 핵심 액션 완료율                │
├─────────────────────────────────────────────────┤
│  Retention (유지)                                │
│  "사용자가 돌아오는가?"                          │
│  지표: D1/D7/D30 리텐션, WAU/MAU                 │
├─────────────────────────────────────────────────┤
│  Revenue (수익)                                  │
│  "어떻게 수익을 창출하는가?"                     │
│  지표: 유료 전환율, ARPU, LTV                    │
├─────────────────────────────────────────────────┤
│  Referral (추천)                                 │
│  "다른 사람을 데려오는가?"                       │
│  지표: NPS, K-factor, 추천 가입률                │
└─────────────────────────────────────────────────┘
```

### AIDA 모델

```
Attention (주의)
    │  "스크롤을 멈추게 하라"
    │  - 충격적인 통계
    │  - 도발적인 질문
    │  - 대담한 주장
    ▼
Interest (관심)
    │  "더 알고 싶게 하라"
    │  - 스토리텔링
    │  - 문제 공감
    │  - 관련성 연결
    ▼
Desire (욕구)
    │  "갖고 싶게 하라"
    │  - 혜택 나열
    │  - 사회적 증거
    │  - 미래 비전
    ▼
Action (행동)
    │  "행동하게 하라"
    │  - 명확한 CTA
    │  - 긴급성
    │  - 리스크 제거
```

### PAS 공식

```
Problem (문제)
    │  "당신의 고통을 안다"
    │  예: "서버 모니터링에 월 $300 쓰고 계신가요?"
    ▼
Agitation (확대)
    │  "더 아프게 하라"
    │  예: "스타트업에게 그 돈이면 개발자 1명 점심값입니다"
    ▼
Solution (해결)
    │  "해결책을 제시하라"
    │  예: "월 $29로 동일한 기능. 10배 저렴한 모니터링"
```

## CRO 체크리스트

### 랜딩페이지 최적화

```yaml
above_the_fold:
  - [ ] 3초 내 가치 제안 명확
  - [ ] 메인 헤드라인 주목도
  - [ ] CTA 버튼 가시성
  - [ ] 히어로 이미지/영상 품질

trust_elements:
  - [ ] 고객사 로고 (최소 3개)
  - [ ] 사용자 리뷰/평점
  - [ ] 케이스 스터디
  - [ ] 보안 인증 배지
  - [ ] 언론 노출

cta_optimization:
  - [ ] 단일 CTA (여러 개 X)
  - [ ] 버튼 색상 대비
  - [ ] 액션 지향 문구
  - [ ] 페이지 내 반복

form_optimization:
  - [ ] 필드 최소화 (3-5개)
  - [ ] 실시간 유효성 검사
  - [ ] 프라이버시 정책 링크

technical:
  - [ ] 3초 이내 로딩
  - [ ] 모바일 최적화
  - [ ] 브라우저 호환성

psychology:
  - [ ] 긴급성/희소성 요소
  - [ ] 손실 회피 메시지
  - [ ] 사회적 증거
```

## AI 시대 마케팅 트렌드 (2025)

### 숏폼 비디오 우선 전략

```yaml
content_ratio:
  short_form_video: "80%"
  image_text: "20%"

platform_priority:
  1: "TikTok (바이럴 잠재력)"
  2: "Instagram Reels"
  3: "YouTube Shorts"

viral_formula:
  hook: "[0-1초] 스크롤 멈추게"
  problem: "[1-5초] 문제 공감"
  solution: "[5-12초] 데모"
  cta: "[12-15초] 행동 유도"
```

### AI 콘텐츠 파이프라인

```
1. 아이디어 생성 (ChatGPT)
    │  훅 문구 30개 대량 생성
    ▼
2. 스크립트 작성 (ChatGPT)
    │  15초 숏폼 스크립트 변환
    ▼
3. 비주얼 제작 (Canva AI)
    │  썸네일, 텍스트 오버레이
    ▼
4. 영상 편집 (CapCut)
    │  자동 캡션, 트렌드 템플릿
    ▼
5. 결과: 1시간에 5개 콘텐츠 생산
```

## 프레임워크 선택 가이드

### 상황별 프레임워크 추천

| 상황 | 추천 프레임워크 | 이유 |
|-----|----------------|------|
| 신규 시장 진입 | 3C + STP | 시장 이해와 포지셔닝이 핵심 |
| 성장 단계 | AARRR | 전환 퍼널 최적화 필요 |
| 런칭 캠페인 | SMART + PESO | 목표 설정과 채널 믹스 |
| 랜딩페이지 카피 | AIDA | 긴 형식에 적합 |
| 광고 카피 | PAS | 짧고 강렬한 메시지 |
| 케이스 스터디 | BAB | 변화 스토리 강조 |
| 제품 설명 | FAB | 기능→혜택 연결 |

```yaml
saas_launch:

  content: "BAB (케이스 스터디)"
  landing: "AIDA"
  nurturing: "AIDA (이메일 시퀀스)"
```
