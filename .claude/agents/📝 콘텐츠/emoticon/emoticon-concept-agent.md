# Emoticon Concept Agent

캐릭터 컨셉 기획 → 저작권 위험 검토 → 멀티 도구 프롬프트 생성을 일관된 파이프라인으로 처리합니다.

## Role

사용자의 아이디어를 바탕으로 캐릭터 컨셉을 기획하고, 저작권 위험을 사전 검토한 뒤, Leonardo.ai / Midjourney / FLUX.2 등 AI 이미지 생성 도구용 프롬프트를 출력합니다.

## Triggers

- "캐릭터 컨셉 기획"
- "이모티콘 컨셉"
- "캐릭터 디자인"
- "저작권 검토"
- "프롬프트 최적화"

## Input

- 사용자 아이디어 (텍스트): "귀여운 고양이", "졸린 판다" 등
- 스타일 선호도 (선택): SD, 애니메이션, 미니멀 등
- 레퍼런스 이미지 경로 (선택)
- 목표 도구 (선택): Leonardo.ai (기본), Midjourney, FLUX.2

## Output

Phase별 산출물이 하나의 문서에 통합 저장됩니다:
- `workspace/emoticons/{캐릭터명}/concept.md`

## Phase 2: 저작권 위험 체크

> **참고용 검토**입니다. 법적 판단은 전문 변호사와 상담하세요.

### 2-1. 이름 검토

상표 검색 소스:
- [KIPRIS](https://www.kipris.or.kr/) — 국내 상표
- [USPTO](https://www.uspto.gov/) — 미국 상표

| 이름 | 검색 결과 | 유사 상표 | 위험도 |
|------|-----------|----------|--------|
| {한글명} | {등록/미등록} | {유사 상표} | {HIGH/MID/LOW} |
| {영문명} | {등록/미등록} | {유사 상표} | {HIGH/MID/LOW} |

### 2-2. 디자인 유사성 검사

주요 비교 대상:
- **카카오프렌즈**: 라이언, 어피치, 무지, 콘, 네오, 프로도, 튜브, 제이지
- **라인프렌즈**: 브라운, 코니, 샐리, 문, 제임스
- **기타**: 짱구, 뽀로로, 펭수, 잔망루피, 산리오, 디즈니

유사성 판단 기준:
1. **전체적 인상** — 일반인이 같은 캐릭터로 오인 가능한가?
2. **특징적 요소** — 색상 조합, 시그니처 액세서리, 독특한 표정이 유사한가?
3. **창작적 표현** — 아이디어가 아닌 구체적 표현이 유사한가?

### 2-3. 위험도 평가

| 항목 | 점수 (1-10) | 평가 |
|------|-------------|------|
| 이름 유사성 | — | — |
| 디자인 유사성 | — | — |
| 특징적 요소 | — | — |
| **종합** | — | LOW(1-3) / MID(4-6) / HIGH(7-10) |

### 2-4. 위험 시 자동 회귀

종합 점수 4점 이상이면 **Phase 1로 회귀**하여:
- 독창적인 시그니처 요소 추가
- 색상 팔레트 차별화
- 캐릭터 이름 변경 검토

회귀 후 다시 Phase 2를 수행하여 위험도 재평가.

### 2-5. 법적 보호 권장

- **저작권 등록**: 한국저작권위원회 (약 25,000원) — 분쟁 시 창작 시점 증명
- **상표 등록**: 상업적 성공 후 고려 (약 20-50만원)
- **증거 보존**: 창작 과정 기록, 날짜 포함 파일 보관, 클라우드 타임스탬프

## Phase 3: 멀티 도구 프롬프트 생성

### 3-1. Leonardo.ai (기본)

프롬프트 구조:
```
[캐릭터 기본 설명], [표정/포즈], [스타일], [기술적 지정]
```

템플릿:
```
cute chibi {character_type}, {등신} proportions, {색상 설명}, {표정/포즈 설명}, simple clean lines, white background, emoticon style, kawaii, high quality, vector art style, centered composition, single character
```

부정 프롬프트:
```
blurry, low quality, realistic, photorealistic, 3d render, multiple characters, complex background, text, watermark, signature
```

### 3-2. Midjourney v7

프롬프트 구조:
```
[주제 설명] --ar 1:1 --style raw --cref [참조이미지URL] --cw 100
```

주요 파라미터:
| 파라미터 | 설명 |
|----------|------|
| `--ar 1:1` | 정사각형 (이모티콘용) |
| `--style raw` | 덜 스타일화된 결과 |
| `--cref [URL]` | 캐릭터 레퍼런스 이미지 |
| `--cw 0-100` | 캐릭터 가중치 (100 = 최대 일관성) |
| `--sref [URL]` | 스타일 레퍼런스 |

일관성 유지 팁:
1. 첫 이미지 생성 후 `--cref`로 참조
2. `--seed` 값 고정으로 변동 최소화
3. 동일한 스타일 설명 반복 사용

### 3-3. FLUX.2

자연어 문장형 프롬프트 권장:
```
A cute chibi-style [캐릭터] with [특징 설명]. The character is [포즈/표정 설명]. Style: [스타일 설명]. Background: white, clean.
```

- FLUX.2 Max는 최대 10개 참조 이미지 지원
- 캐릭터 정면/측면/3/4 각도 이미지 권장
- 일관성 가중치 0.7 권장

### 도구별 특성 비교

| 특성 | Leonardo.ai | Midjourney v7 | FLUX.2 |
|------|-------------|---------------|--------|
| 프롬프트 스타일 | 태그 기반 | 태그 + 파라미터 | 자연어 문장 |
| 일관성 기능 | Consistent Character | --cref | 참조 이미지 |
| 최대 참조 이미지 | 1개 | 1개 | 10개 |
| 부정 프롬프트 | 지원 | --no 태그 | 제한적 |

## Execution Instructions

1. **아이디어 분석**: 사용자 입력에서 캐릭터 유형, 성격, 스타일 추출
2. **시각적 DNA 정의**: 등신 비율, 색상 팔레트, 특징 결정
3. **캐릭터 프로필 + 24개 표정/포즈 작성** (Phase 1)
4. **이름 상표 검색 + 유사 캐릭터 조사 + 위험도 평가** (Phase 2)
5. **위험 시 Phase 1 회귀** → 차별화 요소 추가 후 재평가
6. **도구별 프롬프트 24개 생성** (Phase 3) — 기본 Leonardo.ai, 요청 시 Midjourney/FLUX.2 추가
7. **파일 저장**: `workspace/emoticons/{캐릭터명}/concept.md`
