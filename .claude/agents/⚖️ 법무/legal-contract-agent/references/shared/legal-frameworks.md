# 법률 프레임워크 및 규정 준수 가이드

## 개요

계약 검토 시 고려해야 할 주요 법률 프레임워크와 규정 준수 요건을 정리한 문서입니다.

---

## 대한민국 주요 법률

### 민법 (계약 일반)

```
핵심 조항:
- 제103조: 선량한 풍속 기타 사회질서 위반 무효
- 제104조: 불공정한 법률행위 무효
- 제428조: 보증채무의 범위
- 제543조: 해제권
- 제750조: 불법행위 손해배상

체크포인트:
□ 불공정 조항 (약관규제법 위반 가능성)
□ 보증인 보호 규정
□ 해제/해지 조건의 균형
□ 손해배상 범위의 합리성
```

### 약관의 규제에 관한 법률

```
핵심 조항:
- 제3조: 약관의 작성 및 설명의무
- 제5조: 약관의 해석
- 제6조: 불공정약관조항
- 제7조~제14조: 무효 조항

불공정 약관 유형:
- 고객 권리 배제/제한
- 사업자 의무 면제/경감
- 손해배상 제한
- 해제/해지 제한
- 일방적 조항 변경
```

### 전자상거래법

```
적용 대상:
- 온라인 서비스 계약
- SaaS/PaaS 이용약관
- 전자적 거래 일반

필수 포함 사항:
□ 청약철회 조항 (7일)
□ 반환/환불 조건
□ 서비스 이용 조건
□ 개인정보 처리 동의
□ 분쟁해결 절차
```

### 개인정보보호법 / 정보통신망법

```
핵심 요건:
- 개인정보 수집/이용 동의
- 제3자 제공 동의
- 개인정보 처리 위탁
- 국외 이전 고지
- 파기 의무

계약 포함 사항:
□ 개인정보 처리 목적
□ 처리 위탁 내용
□ 보안 조치 의무
□ 재위탁 제한
□ 손해배상 책임
```

### 하도급법

```
적용 대상:
- 원사업자-수급사업자 관계
- 제조/수리 위탁
- 건설/용역 위탁

주요 금지사항:
- 서면 미교부
- 부당한 하도급대금 결정
- 물품구매 강제
- 부당한 대금 감액
- 부당 반품

체크포인트:
□ 서면 계약 체결
□ 적정 대금 결정
□ 결제 기한 (60일)
□ 선급금 지급
```

## 국제 규정

### GDPR (EU 일반개인정보보호규정)

```
적용 조건:
- EU 거주자 개인정보 처리
- EU 내 서비스 제공
- EU 주민 행동 모니터링

핵심 요건:
□ 처리의 법적 근거
□ 정보주체 권리 보장
□ DPA (Data Processing Agreement)
□ 국외 이전 적절한 보호조치
□ DPO 지정 (해당 시)
□ 72시간 내 침해 통지

계약 필수 조항:
- 처리 범위/목적
- 보안 조치
- 재위탁 조건
- 감사 권한
- 삭제/반환 의무
```

### CCPA (캘리포니아 소비자개인정보보호법)

```
적용 대상:
- 캘리포니아 거주자 개인정보
- 일정 규모 이상 사업자

핵심 요건:
□ 개인정보 수집 고지
□ 개인정보 판매 거부권
□ 접근/삭제 요청 대응
□ 비차별 원칙

서비스 제공자 요건:
- 처리 목적 제한
- 보안 유지 의무
- 사업 목적 외 사용 금지
```

### SOX (Sarbanes-Oxley Act)

```
적용 대상:
- 미국 상장기업
- 상장기업 협력사

영향 영역:
□ 내부통제 요건
□ 재무보고 정확성
□ 문서 보존 의무
□ 감사 요건
```

## 산업별 규제

### 금융 (금융위원회/금융감독원)

```
주요 규제:
- 전자금융거래법
- 신용정보법
- 금융소비자보호법

계약 시 고려사항:
□ 금융정보 보안 요건
□ 전자금융거래 기록 보존
□ 금융소비자 보호 의무
□ 외부 위탁 제한
□ 클라우드 이용 가이드라인
```

### 의료/헬스케어

```
주요 규제:
- 의료법
- 약사법
- 개인정보보호법 (민감정보)
- HIPAA (미국, 해당 시)

계약 시 고려사항:
□ 의료정보 보안 강화
□ 동의 요건 (민감정보)
□ 접근 통제
□ 감사 로그 유지
□ BAA (미국, 해당 시)
```

### 게임/엔터테인먼트

```
주요 규제:
- 게임산업진흥법
- 청소년보호법
- 저작권법

계약 시 고려사항:
□ 등급 분류
□ 결제 한도 (미성년자)
□ 확률형 아이템 공시
□ 저작권 클리어런스
□ 퍼블리싱 권리
```

### IT/SaaS

```
주요 규제:
- 정보통신망법
- 클라우드컴퓨팅법
- 소프트웨어진흥법

계약 시 고려사항:
□ 정보보호 관리체계 (ISMS)
□ 클라우드 보안 인증
□ 데이터 위치 제한
□ 서비스 연속성
□ 데이터 이전 지원
```

## 준거법/관할권 가이드

### 준거법 선택 고려사항

```
유리한 준거법:
- 자국법 (익숙한 법체계)
- 중립국법 (스위스, 싱가포르)
- 업계 표준법 (영국법-금융, 미국법-IT)

불리한 준거법:
- 상대방 국가법
- 법체계가 다른 국가
- 언어 장벽이 큰 국가
```

### 분쟁해결 방식

```
법원 소송:
- 장점: 강제력, 항소 가능
- 단점: 시간/비용, 공개성

중재:
- 장점: 전문성, 비공개, 국제 집행
- 단점: 비용, 항소 제한
- 기관: KCAB, ICC, SIAC, AAA

조정:
- 장점: 관계 유지, 저비용
- 단점: 구속력 없음
```

### 관할 조항 예시

```
국내:
"본 계약과 관련한 분쟁은 대한민국 법률을 준거법으로 하며,
서울중앙지방법원을 제1심 전속관할법원으로 합니다."

국제:
"본 계약과 관련한 분쟁은 대한상사중재원의 중재규칙에 따라
서울에서 진행되는 중재에 의해 최종적으로 해결됩니다.
중재 언어는 영어로 합니다."
```

## 국가별 특이사항

### 미국

```
주의사항:
- 주별 법률 차이 큼
- 클래스 액션 위험
- 배심재판 가능성
- 높은 손해배상

권장사항:
□ 집단소송 포기 조항
□ 중재 조항 포함
□ 배심재판 포기
□ 델라웨어/뉴욕 준거법
```

### 중국

```
주의사항:
- 외국법 준거 제한
- 데이터 국외 이전 제한
- 기술이전 규제
- 독점규제법 강화

권장사항:
□ CIETAC 중재
□ 중국어 버전 우선
□ 현지 법률 자문 필수
□ 데이터 현지화 확인
```

### EU

```
주의사항:
- GDPR 엄격 적용
- 소비자 보호 강화
- 불공정조항 지침
- 표준계약조항 요건

권장사항:
□ GDPR 준수 조항
□ SCC (표준계약조항)
□ 소비자 권리 보장
□ 현지 언어 제공
```

## 규정 준수 위반 시 제재

### 국내

| 법률 | 위반 유형 | 제재 |
|------|----------|------|
| 개인정보보호법 | 정보 유출 | 과징금 + 형사처벌 |
| 약관규제법 | 불공정약관 | 시정명령 + 과태료 |
| 전자상거래법 | 표시 위반 | 과태료 + 영업정지 |
| 하도급법 | 불공정행위 | 과징금 + 공표 |

### 해외

| 규정 | 위반 유형 | 제재 |
|------|----------|------|
| GDPR | 정보보호 위반 | 매출 4% 또는 2천만유로 |
| CCPA | 프라이버시 위반 | 건당 $7,500 |
| SOX | 재무보고 위반 | 벌금 + 형사처벌 |
