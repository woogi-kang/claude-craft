# 위험 분석 (Risk Analysis) Framework

## 개요

계약 위험 분석을 위한 체계적인 프레임워크입니다. Risk Matrix를 활용하여 위험 조항을 식별하고 우선순위를 결정합니다.

---

## Risk Matrix

```
                        영향도 (Impact)
                   낮음      보통      높음
              +---------+---------+---------+
         높음 |  Medium |  High   | Critical|
   발생       |   L3    |   H2    |   C1    |
   가능성     +---------+---------+---------+
   (Likeli-  보통 |  Low    | Medium  |  High   |
   hood)      |   L1    |   M2    |   H1    |
              +---------+---------+---------+
         낮음 |  Low    |  Low    | Medium  |
              |   L0    |   L2    |   M1    |
              +---------+---------+---------+
```

## 심각도 정의

### Critical (즉시 대응 필요)

```
정의: 서명 불가, 법적 리스크 또는 중대한 비즈니스 손실 가능

특징:
- 계약 거부 또는 즉각적인 수정 필요
- 경영진 의사결정 필요
- 법률 전문가 검토 필수

예시:
- 무제한 손해배상 책임
- 경쟁사 유리 독점 조항
- IP 전체 양도
- 준거법/관할 심각한 불리
```

### High (수정 강력 권고)

```
정의: 불리한 조건으로 비즈니스에 상당한 영향

특징:
- 서명 전 수정 강력 권고
- 협상 우선순위 높음
- 대안 조항 제시 필요

예시:
- 과도한 책임 제한
- 일방적 해지 조항
- 불합리한 SLA
- 비대칭적 면책
```

### Medium (개선 권장)

```
정의: 개선 여지가 있는 조항으로 협상 고려

특징:
- 수용 가능하나 개선 여지
- 협상력에 따라 시도
- 다른 조건과 트레이드오프 가능

예시:
- 긴 통지 기간
- 자동 갱신 조항
- 제한적 감사 권한
- 일방적 변경 조항
```

### Low (수용 가능)

```
정의: 표준적이거나 수용 가능한 조항

특징:
- 업계 표준 범위
- 추가 협상 불필요
- 참고 사항으로 기록

예시:
- 표준 면책 조항
- 합리적 기밀유지
- 일반적 통지 조항
```

## 위험 카테고리

### 1. 재무적 위험 (Financial Risk)

```
평가 요소:
- 손해배상 한도
- 로열티/요금 조건
- 지연 이자/위약금
- 보험 요건
- 세금 처리

체크리스트:
□ 총 재무 노출 규모는?
□ 최악의 시나리오 비용은?
□ 보험 커버리지 여부?
□ 예산 영향도는?
```

### 2. 운영적 위험 (Operational Risk)

```
평가 요소:
- SLA 및 성과 기준
- 서비스 연속성
- 전환/Exit 조건
- 변경 관리 절차
- 리소스 의존도

체크리스트:
□ 일상 운영에 미치는 영향?
□ 서비스 중단 시 대안?
□ 전환 비용과 기간?
□ 인력 의존도?
```

### 3. 법적 위험 (Legal Risk)

```
평가 요소:
- 규정 준수 의무
- 면책 범위
- 분쟁 해결 조항
- 준거법/관할
- 지재권 귀속

체크리스트:
□ 법적 분쟁 가능성?
□ 관할권 불리 여부?
□ 규제 위반 위험?
□ IP 분쟁 가능성?
```

### 4. 전략적 위험 (Strategic Risk)

```
평가 요소:
- 경쟁 제한 조항
- 독점/종속 조건
- 시장 접근 제한
- 기술 종속성
- 협력 관계 영향

체크리스트:
□ 장기 전략에 미치는 영향?
□ 경쟁사 이점 제공?
□ 시장 기회 제한?
□ 파트너십 영향?
```

### 5. 평판 위험 (Reputational Risk)

```
평가 요소:
- 공개 의무
- 브랜드 연관
- 고객 영향
- 미디어 노출
- 규제기관 관계

체크리스트:
□ 공개 시 평판 영향?
□ 고객 신뢰도 영향?
□ 미디어 관심 가능성?
□ 규제기관 반응?
```

## 위험 평가 프로세스

### Step 1: 조항 식별

```
1. 계약서 전체 스캔
2. 위험 카테고리별 조항 추출
3. 비표준/비대칭 조항 마킹
4. 누락된 표준 조항 확인
```

### Step 2: 심각도 평가

```
각 조항에 대해:
1. 발생 가능성 평가 (Low/Medium/High)
2. 영향도 평가 (Low/Medium/High)
3. Risk Matrix에 따른 심각도 결정
4. 카테고리 분류
```

### Step 3: 우선순위 결정

```
우선순위 기준:
1. Critical → 즉시 대응
2. High → 협상 필수 항목
3. Medium → 협상 시도 항목
4. Low → 모니터링 항목
```

### Step 4: 대응 전략 수립

```
대응 옵션:
- 조항 삭제 요청
- 수정안 제시
- 상호 조건 추가
- 범위 제한
- 예외 조항 추가
- 단서 조건 추가
```

## 일반적인 위험 조항 유형

### 책임 관련

| 조항 유형 | 위험 신호 | 권고 대응 |
|----------|----------|----------|
| 무제한 책임 | "unlimited liability" | 연간 요금 기준 한도 |
| 간접손해 포함 | "consequential damages" | 배제 또는 제한 |
| 일방적 면책 | 한쪽만 면책 의무 | 상호 면책 |
| 보증 배제 | "AS IS" 제공 | 기본 보증 추가 |

### 해지 관련

| 조항 유형 | 위험 신호 | 권고 대응 |
|----------|----------|----------|
| 편의 해지 없음 | 기간 만료만 가능 | 편의 해지 추가 |
| 즉시 해지 | 시정 기회 없음 | 시정 기간 추가 |
| Lock-in 기간 | 장기 의무 기간 | 기간 단축 또는 중도 해지 |
| Exit 비용 | 해지 시 위약금 | 비용 제한/면제 |

### 지재권 관련

| 조항 유형 | 위험 신호 | 권고 대응 |
|----------|----------|----------|
| IP 전체 양도 | "assign all rights" | 라이선스로 변경 |
| Work for Hire | 저작권 자동 이전 | 범위 제한 |
| Grant-back | 개량 발명 양도 | 라이선스로 변경 |
| IP 보증 없음 | 침해 면책 없음 | 면책 조항 추가 |

## 위험 완화 전략

### 계약 조건 협상

```
1. 한도 설정: 무제한 → 연간 요금의 12개월
2. 범위 제한: 전체 → 특정 상황만
3. 예외 추가: 면책 → 고의/중과실 제외
4. 상호성: 일방적 → 상호 의무
5. 시정 기회: 즉시 해지 → 30일 시정 기간
```

### 보험 활용

```
검토 보험 유형:
- 전문인배상책임보험 (E&O)
- 사이버보험
- 일반배상책임보험
- 이사임원배상책임보험 (D&O)
```

### 운영적 대응

```
- 정기 계약 검토 일정
- 해지 통지 알림 설정
- SLA 모니터링 체계
- 벤더 평가 프로세스
```
