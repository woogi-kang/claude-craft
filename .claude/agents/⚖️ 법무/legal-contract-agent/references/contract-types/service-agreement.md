# 서비스 계약 (MSA/SaaS) Reference

## 개요

서비스 계약은 B2B 서비스 제공 관계를 규정하는 계약으로, MSA(Master Service Agreement), SaaS Agreement, 업무위탁계약 등이 포함됩니다.

---

## 서비스 계약 유형

| 유형 | 설명 | 주요 특징 |
|------|------|----------|
| **MSA** | 기본 계약 + SOW | 유연한 범위 확장 |
| **SaaS Agreement** | 클라우드 서비스 | 구독 모델, 데이터 처리 |
| **업무위탁계약** | 특정 업무 위탁 | 인력/결과물 관리 |
| **유지보수계약** | 지속적 지원 | SLA, 대응 시간 |

---

## 핵심 검토 항목

### 1. 서비스 범위 (Scope of Services)

```
체크포인트:
□ 서비스 범위가 명확히 정의되어 있는가?
□ 제외 서비스가 명시되어 있는가?
□ 범위 변경 절차가 규정되어 있는가?
□ SOW(Statement of Work) 형식과 절차

위험 신호:
- "합리적으로 필요한 모든 서비스" → 범위 확대 위험
- 변경 관리 조항 없음 → Scope Creep 위험
```

### 2. SLA (Service Level Agreement)

```
핵심 지표:
- Uptime/Availability: 99.9% 이상
- Response Time: P1(1시간), P2(4시간), P3(24시간)
- Resolution Time: 심각도별 차등
- Scheduled Maintenance: 사전 통지 기간

체크포인트:
□ SLA 지표가 측정 가능한가?
□ SLA 미달 시 구제수단 (Credit/환불)
□ 면책 사유가 합리적인가?
□ 보고 주기와 형식
```

### 3. 요금 및 결제 (Fees and Payment)

```
체크포인트:
□ 요금 구조가 명확한가 (고정/변동/혼합)?
□ 요금 인상 조건과 한도
□ 결제 조건 (선불/후불, 주기)
□ 지연 이자/연체료
□ 세금 처리 방식
```

### 4. 책임 제한 (Limitation of Liability)

```
표준 구조:
- 직접손해: 연간 요금의 12개월분 한도
- 간접손해: 전면 배제
- 예외: 고의/중과실, 비밀유지 위반, 지재권 침해

체크포인트:
□ 손해배상 한도가 합리적인가?
□ 면책 예외 사항이 명확한가?
□ 무한책임 조항 여부
□ 보험 요건
```

### 5. 지식재산권 (Intellectual Property)

```
체크포인트:
□ 기존 IP 귀속이 명확한가?
□ 신규 개발물 귀속 규정
□ 라이선스 범위와 조건
□ 제3자 IP 사용 시 책임

위험 신호:
- 모든 결과물 고객 귀속 → 서비스 개선 제약
- Work for Hire 조항 → 재사용 불가
```

### 6. 데이터 및 보안 (Data and Security)

```
체크포인트:
□ 데이터 소유권 명확한가?
□ 데이터 처리 범위와 목적
□ 보안 요건 (SOC2, ISO27001 등)
□ 데이터 위치 제한
□ 데이터 반환/삭제 조항
□ 개인정보처리 위탁 규정
```

### 7. 해지 (Termination)

```
체크포인트:
□ 해지 사유가 균형적인가?
□ 사전 통지 기간
□ 편의 해지(Termination for Convenience) 조건
□ 해지 시 결과 (데이터 반환, 이행 지원)
□ 존속 조항 (Survival)
```

## 협상 포지션별 전략

### 서비스 제공자 (Vendor)

```
우선순위:
1. 책임 제한 조항 확보
2. SLA 면책 사유 확대
3. 지재권 유지
4. 요금 인상 권한

주장 포인트:
- 간접손해 전면 배제
- Force Majeure 범위 확대
- 자동 갱신 조항
- 선불 결제
```

### 서비스 수령자 (Customer)

```
우선순위:
1. SLA 강화 및 Credit 확보
2. 데이터 소유권 확보
3. 해지 유연성
4. 가격 동결/상한선

주장 포인트:
- SLA 미달 시 환불
- 데이터 이전 지원
- 편의 해지 권한
- 후불 결제
```

## 서비스 계약 체크리스트

### 서비스 제공자용

```
□ 서비스 범위가 명확히 제한되어 있는가?
□ SLA가 달성 가능한 수준인가?
□ 책임 제한 조항이 있는가?
□ 지재권이 보호되고 있는가?
□ 요금 인상 조항이 있는가?
□ 자동 갱신 조항이 있는가?
```

### 서비스 수령자용

```
□ SLA가 측정 가능하고 구제수단이 있는가?
□ 데이터 소유권이 확보되어 있는가?
□ 해지 조건이 합리적인가?
□ 전환 지원(Transition Assistance) 조항이 있는가?
□ 가격 상한선이 있는가?
□ 감사 권한이 있는가?
```

## SLA 템플릿

### 가용성 (Availability)

| 등급 | 가용성 | 허용 다운타임/월 | Credit |
|------|--------|----------------|--------|
| Tier 1 | 99.99% | 4.3분 | 25% |
| Tier 2 | 99.9% | 43.2분 | 10% |
| Tier 3 | 99.5% | 3.6시간 | 5% |

### 대응 시간

| 심각도 | 설명 | 대응 시간 | 해결 목표 |
|--------|------|----------|----------|
| P1 (Critical) | 서비스 전체 장애 | 15분 | 4시간 |
| P2 (High) | 주요 기능 장애 | 1시간 | 8시간 |
| P3 (Medium) | 부분 기능 장애 | 4시간 | 24시간 |
| P4 (Low) | 경미한 이슈 | 24시간 | 72시간 |

## 일반적인 위험 조항

| 조항 | 위험 수준 | 설명 |
|------|----------|------|
| 무제한 책임 | Critical | 책임 한도 협상 필수 |
| 자동 갱신 + 요금 인상 | High | 갱신 조건 명확화 |
| 일방적 서비스 변경 | High | 사전 통지/동의 요건 |
| 데이터 귀속 불명확 | High | 명시적 규정 필요 |
| SLA Credit 없음 | Medium | 구제수단 추가 |

## 표준 문구 예시

### 책임 제한

```
어느 당사자도 상대방에게 간접손해, 특별손해, 결과적 손해,
징벌적 손해 또는 일실이익에 대해 책임을 지지 않습니다.
본 계약에 따른 각 당사자의 총 책임은 청구 원인 발생일
직전 12개월간 지급된 요금을 초과하지 않습니다.

단, 다음의 경우에는 위 제한이 적용되지 않습니다:
(a) 고의 또는 중대한 과실
(b) 비밀유지 의무 위반
(c) 지식재산권 침해에 대한 면책 의무
```

### 해지 조항

```
1. 편의 해지: 각 당사자는 30일 전 서면 통지로 본 계약을
   해지할 수 있습니다.
2. 사유 해지: 상대방이 본 계약을 중대하게 위반하고
   서면 통지 후 30일 이내에 시정하지 않는 경우,
   통지 당사자는 즉시 계약을 해지할 수 있습니다.
3. 해지 효과: 해지 시 서비스 제공자는 고객 데이터를
   30일간 보관 후 안전하게 삭제합니다.
```
