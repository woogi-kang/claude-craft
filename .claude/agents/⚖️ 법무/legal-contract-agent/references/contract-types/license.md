# 라이선스 계약 Reference

## 개요

라이선스 계약은 지식재산권(특허, 저작권, 상표 등)의 사용 권한을 부여하는 계약으로, 소프트웨어 라이선스, 기술이전, 콘텐츠 라이선스 등이 포함됩니다.

---

## 라이선스 유형

| 유형 | 설명 | 주요 특징 |
|------|------|----------|
| **소프트웨어 라이선스** | SW 사용권 부여 | 설치/사용 제한, 업데이트 |
| **기술이전** | 특허/노하우 이전 | 로열티, 기술지원 |
| **콘텐츠 라이선스** | 저작물 사용권 | 사용 범위, 2차 저작 |
| **상표 라이선스** | 상표 사용권 | 품질관리, 영역 제한 |

## 핵심 검토 항목

### 1. 라이선스 범위 (Scope of License)

```
체크포인트:
□ 라이선스 대상이 명확히 정의되어 있는가?
□ 독점/비독점 여부
□ 지역적 범위 (전 세계/특정 지역)
□ 사용 목적 제한
□ 양도/서브라이선스 가능 여부

위험 신호:
- "합리적으로 필요한 모든 권리" → 범위 불명확
- 서브라이선스 무제한 허용 → 통제 상실 위험
```

### 2. 라이선스 유형 상세

```
독점 (Exclusive):
- 라이선서도 사용 불가
- 높은 로열티
- 최소 실적 의무 일반적

독점적 (Sole):
- 라이선서는 사용 가능
- 다른 라이선시 없음

비독점 (Non-exclusive):
- 다수 라이선시 가능
- 낮은 로열티
- 최소 실적 없음
```

### 3. 로열티 (Royalty)

```
로열티 구조:
- 일시금 (Lump-sum): 계약 시 일괄 지급
- 경상 로열티 (Running Royalty): 매출/판매량 기준
- 최소 로열티 (Minimum Royalty): 최소 보장 금액
- 마일스톤: 목표 달성 시 지급

체크포인트:
□ 로열티 산정 기준이 명확한가?
□ 순매출/총매출 정의
□ 공제 항목 (반품, 할인 등)
□ 감사 권한
□ 최소 로열티 조건
```

### 4. 지식재산권 보호

```
체크포인트:
□ IP 소유권 귀속이 명확한가?
□ 개량 발명 (Improvements) 귀속
□ IP 침해 시 대응 의무
□ 면책 (Indemnification) 범위
□ 제3자 권리 침해 보증

위험 신호:
- 개량 발명 전부 양도 → Grant-back 과도
- IP 보증 없음 → 침해 위험 부담
```

### 5. 기술지원 및 교육 (Support & Training)

```
체크포인트:
□ 기술지원 범위와 기간
□ 교육 프로그램 내용
□ 추가 지원 비용
□ 문서/매뉴얼 제공
□ 업데이트/업그레이드 포함 여부
```

### 6. 기간 및 종료 (Term & Termination)

```
체크포인트:
□ 라이선스 기간 (영구/기간제)
□ 갱신 조건
□ 해지 사유
□ 해지 후 잔존 권리
□ 재고 처리 기간 (Sell-off Period)
```

## 협상 포지션별 전략

### 라이선서 (Licensor)

```
우선순위:
1. IP 보호 강화
2. 로열티 극대화
3. 사용 범위 제한
4. 품질 관리 권한

주장 포인트:
- 독점 시 높은 로열티/최소 실적
- 개량 발명 공유/양도
- 감사 권한 확보
- 해지 시 즉시 사용 중단
```

### 라이선시 (Licensee)

```
우선순위:
1. 사용 범위 확대
2. 로열티 합리화
3. IP 보증 확보
4. 존속 권리 확보

주장 포인트:
- 서브라이선스 권한
- 개량 발명 자체 보유
- 해지 후 Sell-off Period
- 경쟁 라이선스 없음 보장
```

## 라이선스 유형별 체크리스트

### 소프트웨어 라이선스

```
□ 사용자 수/설치 수 제한
□ 서버/클라우드 사용 조건
□ 소스코드 접근 여부
□ 수정/파생물 권리
□ 역설계(Reverse Engineering) 제한
□ 업데이트/패치 포함 여부
□ 유지보수 조건
```

### 기술이전 계약

```
□ 이전 기술 명세 (Technology Description)
□ 노하우 문서화 범위
□ 기술지원 기간과 범위
□ 기술인력 파견 조건
□ 실시권 범위
□ 최소 로열티 조건
□ 개량기술 처리
```

### 콘텐츠 라이선스

```
□ 사용 매체/플랫폼 범위
□ 영역/기간 제한
□ 독점/비독점 여부
□ 2차 저작물 권리
□ 편집/수정 권한
□ 크레딧 표시 요건
□ 제3자 권리 클리어런스
```

## 일반적인 위험 조항

| 조항 | 위험 수준 | 설명 |
|------|----------|------|
| 무제한 Grant-back | Critical | 개량 발명 양도 범위 제한 |
| IP 보증 없음 | High | 침해 면책 조항 추가 |
| 일방적 해지권 | High | 해지 사유 균형 맞추기 |
| 로열티 감사 없음 | Medium | 감사 권한 추가 |
| 영구 독점 | Medium | 기간 제한 또는 해지 조건 |

## 로열티 계산 예시

### 경상 로열티 산정

```
순매출 정의:
순매출 = 총매출 - (반품 + 할인 + 세금 + 운송비)

로열티 산정:
- 순매출 $1,000,000
- 로열티율 5%
- 로열티 = $50,000
- 최소 로열티 = $30,000 (더 큰 금액 적용)
- 실제 지급 = $50,000
```

### 마일스톤 지급

```
마일스톤 예시:
- 계약 체결: $100,000
- 제품 출시: $200,000
- 매출 $1M 달성: $300,000
- 매출 $5M 달성: $500,000
```

## 표준 문구 예시

### 라이선스 부여

```
라이선서는 라이선시에게 본 계약에 정의된 영역 내에서
본 계약에 첨부된 기술 명세서에 기재된 라이선스 기술을
사용하여 라이선스 제품을 제조, 사용, 판매, 판매 청약
및 수입할 수 있는 비독점적, 양도 불가능한 라이선스를
부여합니다.
```

### 로열티 조항

```
라이선시는 각 분기 종료 후 30일 이내에 해당 분기 동안의
순매출의 5%에 해당하는 경상 로열티를 라이선서에게
지급합니다. 어떠한 경우에도 연간 로열티는 미화
$50,000 이상이어야 합니다 (최소 로열티).
```

### IP 면책

```
라이선서는 제3자가 라이선스 기술이 해당 제3자의
특허권, 저작권 또는 기타 지식재산권을 침해한다고
주장하는 소송으로부터 라이선시를 방어하고
면책합니다. 단, 라이선시가 (a) 즉시 서면으로
통지하고, (b) 방어 통제권을 라이선서에게 부여하며,
(c) 합리적인 협조를 제공하는 것을 조건으로 합니다.
```
