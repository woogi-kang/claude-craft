# Notion Core Formatting

Notion 문서 작성의 핵심 원칙과 포맷팅 규칙.

## Core Principles

1. **Scannable**: 헤딩(`H1`-`H3`)으로 구조화, 긴 텍스트보다 리스트 선호
2. **Action-Oriented**: 읽고 끝나는 것이 아닌, 다음 행동이 명확한 문서
3. **Notion-Native**: Notion 블록 기능(콜아웃, 토글, 데이터베이스)을 적극 활용

## Document Metadata

모든 문서 최상단에 메타데이터 테이블 포함:

```markdown
| 속성 | 값 |
|------|-----|
| 태그 | API, 가이드, 트러블슈팅 |
| 담당자 | @name |
| 상태 | 작성중 / 검토필요 / 배포됨 / Deprecated |
| 최종수정 | YYYY-MM-DD |
```

## Callout Blocks

| 용도 | 형식 | Notion 배경색 |
|------|------|--------------|
| 정보/팁 | `💡 [내용]` | 회색/파랑 |
| 주의사항 | `⚠️ [내용]` | 노랑 |
| 금지/치명적 | `🚨 [내용]` | 빨강 |
| 성공/완료 | `✅ [내용]` | 초록 |
| 질문/확인필요 | `❓ [내용]` | 보라 |

**예시:**
```
💡 이 API는 v2.0부터 사용 가능합니다.

⚠️ 프로덕션 환경에서는 반드시 환경변수를 사용하세요.

🚨 절대 API 키를 코드에 하드코딩하지 마세요.
```

## Toggle Blocks

긴 내용은 토글로 접어서 가독성 확보:

```markdown
▶️ 상세 에러 로그 보기
   [접힌 내용: 에러 로그, 긴 코드, 부가 설명]

▶️ 전체 응답 예시
   [접힌 내용: JSON response]
```

**토글 사용 기준:**
- 코드 블록 > 30줄
- 부가적인 상세 설명
- 고급 사용자용 옵션
- 에러 로그, 디버그 정보

## Code Blocks

```markdown
```language
// 파일경로: src/example.dart
// 설명: 간단한 설명

실제 코드
```
```

**필수 규칙:**
- 언어 지정 필수 (`dart`, `python`, `javascript`, `bash` 등)
- 파일 경로를 주석으로 명시
- 핵심 라인에 `// ← 주목` 주석 추가

## Tables

| 상황 | 권장 형식 |
|------|----------|
| 비교/대조 | 테이블 |
| 3개 이상 속성 | 테이블 |
| Key-Value | 테이블 |
| 순차적 단계 | 번호 리스트 |
| 병렬 항목 | 불릿 리스트 |

## Heading Hierarchy

```
# H1: 문서 제목 (1개만)
## H2: 주요 섹션
### H3: 하위 섹션
```

**규칙:**
- H1은 문서당 1개
- H4 이하 사용 지양 → 대신 Bold나 토글 활용
- 헤딩 건너뛰기 금지 (H1 → H3 ✗)

## Writing Style

| 항목 | 규칙 |
|------|------|
| 문체 | 명령형/동사형 ("~하십시오", "~한다") |
| 문장 길이 | 1문장 1주제, 40자 이내 권장 |
| 전문용어 | 첫 등장 시 정의 또는 링크 제공 |
| 숫자/버전 | 구체적으로 명시 (v2.0, 3개, 5분) |

## Visual Markers

다이어그램이나 이미지가 필요한 위치 표시:

```markdown
[Image: 시스템 아키텍처 다이어그램]
[Image: 사용자 플로우 차트]
[Diagram: API 호출 시퀀스]
```

## Maintenance Footer

문서 하단에 유지보수 정보 포함:

```markdown
---
📝 **유지보수 노트**
- 이 문서는 [조건: API 버전 변경 시] 업데이트 필요
- 관련 문서: [링크]
- 문의: @담당자
```

## Quick Reference

```
구조:     H1 > H2 > H3 (skip 금지)
강조:     💡정보 ⚠️주의 🚨금지 ✅완료
숨김:     ▶️ 토글로 긴 내용 접기
코드:     ```lang + 파일경로 주석
테이블:   비교/속성 → 테이블, 순서 → 번호리스트
메타:     상단 속성 테이블 필수
유지보수: 하단 업데이트 조건 명시
```
