# Validation Skill (Shared)

콘텐츠 유형에 관계없이 적용되는 데이터 검증 공통 방법론입니다.

## Triggers

- "데이터 검증", "팩트체크", "출처 확인", "콘텐츠 검증"

---

## 공통 검증 프레임워크

### 1. 출처 신뢰도 검증 (Source Credibility Check)

출처의 신뢰도를 5단계로 평가합니다:

| Level | 등급 | 출처 유형 | 설명 |
|-------|------|----------|------|
| **5** | 최고 | 정부기관, 학술논문, 공인기관 보고서 | 공신력 최상위 |
| **4** | 높음 | 주요 언론사, 산업 리서치 기관 | 검증된 전문 기관 |
| **3** | 보통 | 전문 블로그, 기업 공식 자료 | 일반적 신뢰 수준 |
| **2** | 낮음 | 일반 블로그, 커뮤니티 글 | 추가 검증 필요 |
| **1** | 미검증 | 출처 불명, 익명 자료 | 사용 비권장 |

### 2. 팩트체크 프로세스 (5단계 검증)

| 단계 | 검증 항목 | 검증 방법 | 합격 기준 |
|------|----------|----------|----------|
| **1차** | 출처 존재 여부 | URL 접근성 확인, 원문 존재 확인 | 원문 접근 가능 |
| **2차** | 데이터 최신성 | 발행일 확인 | 발행일 기준 6개월 이내 권장 |
| **3차** | 교차 검증 | 최소 2개 이상 독립 출처 확인 | 2+ 출처에서 일치 |
| **4차** | 맥락 검증 | 인용이 원문 맥락과 일치 여부 | 맥락 왜곡 없음 |
| **5차** | 수치 검증 | 통계 데이터 산출 방법 확인 | 방법론 명확 |

### 3. 검증 상태 정의

| 상태 | 설명 | 사용 권장 |
|-----|------|---------|
| `verified` | 모든 검증 통과 | 자유롭게 사용 |
| `partially_verified` | 일부 검증 통과 | 주의하여 사용 |
| `unverified` | 검증 불가 | 사용 비권장 |
| `disputed` | 상충되는 정보 존재 | 사용 금지 |

### 4. 검증 결과 구조

```json
{
  "validation_result": {
    "item_id": "R001",
    "status": "verified|partially_verified|unverified|disputed",
    "credibility_score": 0.92,
    "credibility_level": 4,
    "checks_passed": ["source_exists", "recent_data", "cross_verified"],
    "checks_failed": [],
    "warnings": [],
    "recommendation": "safe_to_use|use_with_caveat|do_not_use",
    "verified_at": "2025-01-04T10:30:00Z"
  }
}
```

## 검증 워크플로우 (공통)

```
Research Skill에서 데이터 수신
         │
         ▼
   [1차] 출처 존재 확인
         │
         ├─ 실패 → 출처 불명 태그 → 사용 비권장
         ▼
   [2차] 최신성 확인
         │
         ├─ 경고 → 오래된 데이터 주의 태그
         ▼
   [3차] 교차 검증
         │
         ├─ 경고 → 단일 출처 주의 태그
         ▼
   [4차] 맥락 검증
         │
         ├─ 실패 → 맥락 왜곡 태그 → 사용 금지
         ▼
   [5차] 수치 검증
         │
         ├─ 경고 → 방법론 불명확 태그
         ▼
   최종 검증 결과 생성
```

## 공통 검증 체크리스트

### 팩트 체크
- [ ] 인용된 통계/수치의 출처가 명시되어 있는가?
- [ ] 원본 데이터와 일치하는가?
- [ ] 데이터가 최신인가? (6개월 이내 권장)
- [ ] 2개 이상의 출처로 교차 검증되었는가?

### 맥락 검증
- [ ] 인용문이 원문과 정확히 일치하는가?
- [ ] 맥락이 왜곡되지 않았는가?

### 링크 검증
- [ ] 모든 링크가 유효한가?
- [ ] 올바른 페이지로 연결되는가?

## 검증 보고서 템플릿

```markdown
# 데이터 검증 보고서

## 요약
- 총 검증 항목: N개
- 검증 통과: N개 (%)
- 부분 통과: N개 (%)
- 검증 실패: N개 (%)

## 검증 통과 항목
| ID | 내용 요약 | 신뢰도 | 출처 |
|----|---------|-------|------|

## 주의 필요 항목
| ID | 경고 내용 | 권장 조치 |
|----|----------|---------|

## 사용 불가 항목
| ID | 실패 사유 | 대안 |
|----|----------|-----|
```

---

## 공통 주의사항

- 검증 실패 항목도 기록 보관 (추후 참조용)
- 검증 결과는 절대적이 아니며, 최종 판단은 사용자 몫
- 빠른 검증이 필요한 경우 1-3차 검증만 수행 가능 (quick_validate 모드)

## 콘텐츠 유형별 오버라이드

각 에이전트별 스킬에서 다음 항목을 콘텐츠 유형에 맞게 구체화합니다:

| 항목 | Presentation | Social Media |
|------|-------------|--------------|
| 유형별 검증 기준 | 투자피치/세미나/내부보고별 차등 | 브랜드 일관성/민감도 추가 |
| 추가 검증 | 수치 검증 강화 | 브랜드 세이프티, 저작권 체크 |
| 중복 체크 | - | 과거 콘텐츠 중복, 표절 체크 |
| 다음 단계 | Structure Skill | Compliance Skill |
| 검증 수준 | 발표 유형별 최소 기준 상이 | 리스크 매트릭스 기반 |
