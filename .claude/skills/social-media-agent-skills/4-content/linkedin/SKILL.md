# LinkedIn 콘텐츠 작성 가이드

## 플랫폼 특성

```yaml
linkedin_profile:
  audience: "비즈니스 전문가, 의사결정권자, 구직자"
  content_lifespan: "24-72시간 (알고리즘), 검색으로 장기 노출"
  algorithm_priority:
    - 체류 시간 (Dwell Time)
    - 댓글 (특히 긴 댓글)
    - 공유 및 리포스트
    - 첫 1시간 참여율
  best_times:
    weekday: "07:30-08:30, 12:00-13:00, 17:00-18:00"
    weekend: "참여율 낮음, 피하기 권장"
  audience_mindset: "학습, 네트워킹, 커리어 성장"
```

## 콘텐츠 유형별 가이드

### 1. 텍스트 포스트

```yaml
text_post:
  specs:
    max_length: 3000자
    optimal_length: 1200-1500자
    line_breaks: "필수 (가독성)"

  structure:
    hook: |
      첫 2-3줄 (접히기 전 표시)
      - 강력한 첫 문장
      - 호기심 유발
      - "더 보기" 클릭 유도

    body: |
      본문
      - 한 문단 = 1-2문장
      - 충분한 줄바꿈
      - 스토리텔링 또는 리스트
      - 구체적 사례/데이터

    cta: |
      마무리
      - 핵심 요약
      - 질문으로 댓글 유도
      - 행동 촉구

  formatting:
    - 짧은 문단 (1-2줄)
    - 빈 줄로 구분
    - 리스트에 숫자 또는 기호
    - 이모지 절제 (1-3개)
```

### 2. 캐러셀 (Document Post)

```yaml
linkedin_carousel:
  specs:
    format: "PDF (권장), PPT"
    slides: "8-15장 권장"
    ratio: "4:5 또는 1:1"
    file_size: "100MB 이하"

  structure:
    slide_1: |
      표지
      - 강렬한 제목
      - 부제목으로 가치 명시
      - 깔끔한 디자인

    slide_2_n: |
      본문
      - 한 슬라이드 = 한 포인트
      - 큰 텍스트 (모바일 가독성)
      - 최소한의 디자인 요소

    last_slide: |
      CTA
      - 요약 또는 핵심 메시지
      - 팔로우/연결 요청
      - 댓글 유도 질문

  design_tips:
    - 폰트 크기: 최소 24pt
    - 한 슬라이드 텍스트: 50단어 이하
    - 브랜드 컬러 일관성
    - 슬라이드 번호 표시
```

### 3. 아티클 (Article)

```yaml
linkedin_article:
  specs:
    length: "1500-3000단어 권장"
    format: "장문 블로그 형식"

  use_cases:
    - 심층 인사이트/분석
    - 케이스 스터디
    - 산업 트렌드 리포트
    - 개인 브랜딩 에세이

  structure:
    - 강력한 헤드라인
    - 도입 (문제 제기)
    - 본문 (섹션별 구분)
    - 결론 (핵심 요약)
    - CTA

  seo_tips:
    - 키워드 포함 제목
    - 소제목 활용 (H2, H3)
    - 이미지 Alt 텍스트
```

### 4. 이미지/인포그래픽 포스트

```yaml
image_post:
  specs:
    ratio: "1200x627 (링크), 1080x1080 (단일)"
    format: "PNG, JPG"

  types:
    quote_card: "인용문 + 깔끔한 배경"
    infographic: "데이터 시각화"
    chart: "그래프/차트"
    behind_scenes: "팀/업무 사진"

  caption:
    - 이미지 맥락 설명
    - 인사이트 추가
    - 질문으로 참여 유도
```

## 캡션 작성 공식

### 구조별 템플릿

**1. 경험담/스토리형**
```
[연도/시점] 전, 저는 [상황]이었습니다.

[문제/도전]
모든 게 잘못되고 있었죠.

[전환점]
그때 한 가지를 깨달았습니다:
[인사이트]

[결과]
그 이후로 [구체적 성과]를 이뤘습니다.

[교훈]
배운 점은 이것입니다:
→ [교훈 1]
→ [교훈 2]
→ [교훈 3]

여러분도 비슷한 경험이 있으신가요?
댓글로 공유해주세요.
```

**2. 인사이트/팁형**
```
[주제]에 대해 아무도 말하지 않는 것이 있습니다.

지난 [기간] 동안 [경험]을 하면서 깨달은 것:

1. [인사이트 1]
   - [설명]

2. [인사이트 2]
   - [설명]

3. [인사이트 3]
   - [설명]

이 중 가장 공감되는 것은 무엇인가요?
```

**3. 반박/논쟁형**
```
"[일반적인 믿음]"

이 말, 정말일까요?

저는 동의하지 않습니다.
그 이유는:

[반박 논거 1]

[반박 논거 2]

[대안적 관점]

당신의 생각은 어떠신가요?
동의하시나요, 반대하시나요?
```

**4. 데이터/리서치형**
```
[충격적인 통계]%

이 숫자가 의미하는 것은:

[분석 1]

[분석 2]

[시사점]

출처: [신뢰할 수 있는 출처]

이 데이터에 대한 당신의 해석은?
```

**5. 축하/마일스톤형**
```
오늘 특별한 소식을 공유합니다.

[마일스톤/성과]

이 여정에서 배운 것:

→ [교훈 1]
→ [교훈 2]
→ [교훈 3]

함께해주신 모든 분들께 감사드립니다.
특히 [감사 대상]에게 감사를.

다음 목표는 [다음 단계]입니다.
```

## 톤앤매너 가이드

```yaml
linkedin_tone:
  voice:
    - 전문적이지만 인간적
    - 신뢰감 있는
    - 겸손하면서 자신감 있는
    - 가치 중심적

  language:
    - "~합니다" 체 (격식)
    - 업계 용어 적절히 사용
    - 데이터/근거 제시
    - 명확하고 직접적인 표현

  emoji_guide:
    frequency: "포스트당 1-3개"
    placement: "강조점 또는 리스트 시작"
    appropriate:
      - "→" (포인트 표시)
      - "✓" (체크리스트)
      - "📊" (데이터)
      - "💡" (인사이트)
    avoid:
      - 과도한 이모지
      - 캐주얼한 이모지 (😂, 🔥)
      - 의미 없는 장식

  formatting:
    - 첫 줄에 훅 (클릭 유도)
    - 1-2문장마다 줄바꿈
    - 리스트로 가독성 확보
    - 핵심 문장 강조
```

## 훅(Hook) 공식

효과적인 첫 줄 패턴:

```yaml
hook_patterns:
  contrarian: |
    "[일반적 믿음]은 틀렸습니다."
    "모두가 [X]라고 하지만, 저는 동의하지 않습니다."

  statistic: |
    "[충격적 숫자]%의 [대상]이 [행동]합니다."
    "지난 [기간], [구체적 성과]를 달성했습니다."

  question: |
    "[도발적 질문]?"
    "왜 [현상]이 일어나는 걸까요?"

  story: |
    "[연도] 전, 저는 [상황]이었습니다."
    "어제 [경험]을 했습니다."

  confession: |
    "솔직히 말씀드리겠습니다."
    "아무도 말하지 않는 진실이 있습니다."

  list_preview: |
    "[N]년간 [경험]에서 배운 [N]가지 교훈:"
    "[역할]로서 깨달은 [N]가지 진실:"
```

## CTA 모음

```yaml
linkedin_ctas:
  engagement:
    - "이 주제에 대한 당신의 생각은?"
    - "동의하시나요? 댓글로 알려주세요."
    - "여러분의 경험은 어떠셨나요?"
    - "1-10점 중 몇 점을 주시겠습니까?"

  connection:
    - "이 주제에 관심 있으시면 연결 요청 보내주세요."
    - "DM으로 더 이야기 나눠요."

  share:
    - "도움이 됐다면 네트워크에 공유해주세요."
    - "리포스트로 더 많은 분들과 나눠주세요."

  follow:
    - "매주 [주제]에 대해 공유합니다. 팔로우하세요."
    - "더 많은 인사이트를 원하시면 🔔 눌러주세요."

  resource:
    - "상세 가이드는 댓글에 링크 달아두었습니다."
    - "무료 템플릿이 필요하시면 댓글 남겨주세요."
```

## LinkedIn 알고리즘 최적화

```yaml
algorithm_tips:
  do:
    - 첫 1시간 내 적극적 댓글 응답
    - 다른 포스트에 의미 있는 댓글 남기기
    - 일관된 발행 스케줄
    - 네이티브 콘텐츠 (외부 링크 최소화)
    - 첫 줄에 강력한 훅

  dont:
    - 첫 포스트에 외부 링크 포함
    - 발행 직후 편집 (알고리즘 리셋)
    - 과도한 해시태그 (3-5개 이하)
    - 참여 유도 스팸 ("좋아요 누르면 PDF 드려요")
    - 팔로워 수 언급 자랑

  external_links:
    strategy: "첫 댓글에 링크 추가"
    reason: "링크 포함 포스트 도달률 40% 하락"
```
