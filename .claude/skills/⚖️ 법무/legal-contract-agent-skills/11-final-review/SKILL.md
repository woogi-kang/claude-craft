# Final Review Skill

모든 분석 결과를 종합하여 서명 전 최종 검토를 수행합니다.

## 최종 검토 영역

### 1. 종합 위험 평가

```yaml
risk_summary:
  critical_count: 0      # 서명 불가 수준
  high_count: 0          # 수정 강력 권고
  medium_count: 0        # 개선 권장
  low_count: 0           # 참고

  overall_risk: ""       # 높음/중간/낮음
  recommendation: ""     # 수용/조건부수용/거부
```

### 2. 핵심 조건 확인

```yaml
key_terms_check:
  - term: "계약 금액"
    value: ""
    verified: true/false

  - term: "계약 기간"
    value: ""
    verified: true/false

  - term: "손해배상 상한"
    value: ""
    verified: true/false
```

### 3. 체크리스트 완료 확인

```yaml
checklist_completion:
  required_items: 20
  passed: 18
  failed: 2
  completion_rate: 90%
```

### 4. 미해결 이슈

```yaml
open_issues:
  - issue: ""
    severity: "Critical/High/Medium"
    status: "Open/Resolved/Accepted"
    action: ""
```

## 서명 권고 기준

### 서명 가능 (✅)

```
- Critical 위험: 0개
- High 위험: 모두 해결 또는 수용 가능
- 체크리스트: 필수 항목 100% 충족
- 협상 목표: 핵심 목표 달성
```

### 조건부 서명 (⚠️)

```
- Critical 위험: 0개
- High 위험: 일부 잔존 (수용 가능 범위)
- 체크리스트: 95% 이상 충족
- 협상 목표: 최소 목표 달성
```

### 서명 불가 (❌)

```
- Critical 위험: 1개 이상 미해결
- 또는 체크리스트 필수 항목 미충족
- 또는 협상 최소 목표 미달성
```

## 워크플로우

```
1. 이전 분석 결과 수집
      │
      ├─ Risk Assessment
      ├─ Summary Extract
      ├─ Checklist
      ├─ Redline 결과 (있는 경우)
      │
      ▼
2. 미해결 이슈 확인
      │
      ├─ Critical 이슈 상태
      ├─ High 이슈 상태
      │
      ▼
3. 핵심 조건 최종 확인
      │
      ├─ 금액, 기간, 책임 등
      ├─ 숫자/날짜 정확성
      │
      ▼
4. 서명 권고 결정
      │
      ├─ 수용/조건부/거부
      ├─ 잔존 리스크 명시
      │
      ▼
5. 최종 리포트 생성
   → workspace/work-legal/{project}/reports/{project}-final-review.md
```
