# Legal Context Intake Skill

계약 검토/생성의 기초가 되는 컨텍스트를 수집합니다.
충분한 컨텍스트 수집 = 정확한 분석 결과

## 왜 중요한가?

```
컨텍스트 없이 시작
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
"계약서 검토해줘" → Generic한 분석

컨텍스트 충분히 수집
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Context Intake → 상황에 맞는 정확한 분석
```

## 수집 항목

### 1. 필수 정보 (Must Have)

```yaml
contract_info:
  type: ""                    # NDA, MSA, License, Employment 등
  title: ""                   # 계약서 제목
  version: ""                 # 버전 (초안, 상대방안, 수정안, 최종안)
  language: ""                # 언어 (한국어, 영어, 이중언어)
  page_count: ""              # 페이지 수 (대략적)

parties:
  our_party:
    name: ""                  # 우리측 당사자명
    role: ""                  # 역할 (제공자/수령자, 라이센서/라이센시, 갑/을)
  counter_party:
    name: ""                  # 상대방 당사자명
    type: ""                  # 유형 (대기업, 중소기업, 스타트업, 개인, 정부)
```

### 2. 권장 정보 (Should Have)

```yaml
context:
  purpose: ""                 # 계약 목적
  background: ""              # 배경 설명
  deal_value: ""              # 계약 규모 (금액, 기간)
  urgency: ""                 # 긴급도 (높음/보통/낮음)

negotiation:
  position: ""                # 협상력 (강함/대등/약함)
  flexibility: ""             # 유연성 (높음/보통/낮음)
  relationship: ""            # 관계 (신규/기존/전략적)

legal:
  jurisdiction: ""            # 관할권 (대한민국, 미국 등)
  governing_law: ""           # 준거법
  industry: ""                # 업종 (IT, 금융, 의료, 제조 등)
  regulations: []             # 관련 규제 (개인정보보호법, 금융규제 등)
```

### 3. 선택 정보 (Nice to Have)

```yaml
preferences:
  risk_tolerance: ""          # 위험 수용도 (보수적/중립/적극적)
  priority_terms: []          # 중요 조항 (가격, 기간, 책임제한 등)
  must_haves: []              # 필수 조항
  deal_breakers: []           # 수용 불가 조항

history:
  previous_contracts: ""      # 기존 계약 유무
  known_issues: []            # 알려진 이슈
  counter_party_concerns: []  # 상대방 우려사항
```

## 인터뷰 질문 템플릿

### Quick Start (3분)

```
1. 어떤 종류의 계약서인가요? (NDA, 서비스계약, 라이선스 등)
2. 우리는 어떤 역할인가요? (서비스 제공자/수령자, 정보 제공자/수령자)
3. 상대방은 누구인가요? (회사명, 규모)
4. 협상력은 어느 정도인가요? (강함/대등/약함)
5. 가장 중요한 조항은 무엇인가요?
```

### Deep Dive (10분)

```
계약 개요
─────────────────────────────────
1. 계약의 목적은 무엇인가요?
2. 계약 금액/규모는 얼마인가요?
3. 예상 계약 기간은 얼마인가요?
4. 긴급하게 검토가 필요한가요?

당사자 관계
─────────────────────────────────
5. 상대방과 기존 거래 관계가 있나요?
6. 상대방의 협상 스타일은 어떤가요?
7. 이 계약이 불성립되면 대안이 있나요?

법적 고려사항
─────────────────────────────────
8. 어느 나라/지역 법률이 적용되나요?
9. 특별히 적용되는 규제가 있나요? (개인정보, 금융 등)
10. 과거에 비슷한 계약에서 문제가 있었나요?

우선순위
─────────────────────────────────
11. 반드시 확보해야 할 조항은?
12. 절대 수용 불가한 조항은?
13. 양보 가능한 영역은?
```

## 계약 유형별 추가 질문

### NDA (비밀유지계약)

```
- 쌍방 NDA인가요, 일방 NDA인가요?
- 어떤 정보를 보호하려고 하나요?
- 정보 공유 기간은 얼마나 되나요?
- 비밀유지 기간은 얼마를 원하나요?
```

### 서비스 계약 (MSA/SaaS)

```
- 어떤 서비스를 제공/수령하나요?
- SLA(서비스수준계약)가 필요한가요?
- 데이터 처리가 포함되나요?
- 지적재산권 이슈가 있나요?
```

### 라이선스 계약

```
- 독점 라이선스인가요?
- 라이선스 범위(지역, 기간, 용도)는?
- 서브라이선스 가능 여부는?
- 로열티 구조는 어떻게 되나요?
```

### 고용/용역 계약

```
- 정규직인가요, 계약직인가요?
- 비경쟁/비밀유지 조항이 필요한가요?
- 결과물의 IP 귀속은 어떻게 하나요?
- 해지 조건은 어떻게 설정하나요?
```

## 워크플로우

```
1. 계약서 수신 (파일 또는 텍스트)
      │
      ▼
2. 계약 유형 자동 식별
      │
      ├─ 식별 불가 → 사용자에게 질문
      │
      ▼
3. 필수 정보 확인
      │
      ├─ 부족 → 인터뷰 질문
      │
      ▼
4. 권장 정보 수집 (가능한 만큼)
      │
      ▼
5. 컨텍스트 문서 생성
      │
      ▼
6. 사용자 확인 & 보완
      │
      ▼
7. 최종 컨텍스트 저장
   → workspace/work-legal/{project}/context/{project}-context.md
```
