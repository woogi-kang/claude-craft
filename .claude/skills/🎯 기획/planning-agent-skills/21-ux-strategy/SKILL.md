## 5. 인터랙션 패턴

### 핵심 인터랙션

| 패턴 | 용도 | 피드백 |
|------|------|--------|
| Click | 주요 액션 | 시각적 변화 |
| Hover | 부가 정보 | 툴팁/프리뷰 |
| Drag | 정렬/이동 | 그림자/스냅 |
| Swipe | 네비게이션 | 페이지 전환 |

### 피드백 원칙

```
1. 즉각적 (< 100ms): 버튼 클릭
2. 중간 (< 1s): 로딩 시작 표시
3. 지연 (> 1s): 진행 상황 표시
```

### 애니메이션 원칙

| 용도 | 지속시간 | 이징 |
|------|---------|------|
| 마이크로인터랙션 | 100-200ms | ease-out |
| 페이지 전환 | 200-300ms | ease-in-out |
| 모달 | 200ms | ease-out |
| 토스트 | 3s 표시 | ease |

---

## 6. 정보 계층

### 시각적 계층

```
Level 1: 핵심 액션/정보 ← 가장 눈에 띄게
Level 2: 주요 콘텐츠
Level 3: 보조 정보
Level 4: 메타 정보 ← 가장 subtle
```

### 읽기 패턴

| 패턴 | 용도 | 적용 |
|------|------|------|
| F-패턴 | 텍스트 중심 | 블로그, 문서 |
| Z-패턴 | 랜딩 페이지 | 마케팅 |
| 구텐베르크 | 간단한 레이아웃 | 폼 |

## 7. 접근성 전략

### 접근성 목표

| 항목 | 목표 | 체크 |
|------|------|------|
| WCAG 레벨 | {level} | ⬜ |
| 키보드 네비게이션 | 100% | ⬜ |
| 스크린 리더 | 호환 | ⬜ |
| 색상 대비 | 4.5:1+ | ⬜ |

### 접근성 원칙

1. **인식 가능**: 모든 정보는 여러 방식으로 전달
2. **운용 가능**: 키보드만으로 모든 기능 사용
3. **이해 가능**: 예측 가능한 동작
4. **견고함**: 다양한 환경 지원

## 8. 반응형 전략

### 브레이크포인트

| 디바이스 | 너비 | 레이아웃 |
|----------|------|----------|
| Mobile | 0-767px | 1 column |
| Tablet | 768-1023px | 2 column |
| Desktop | 1024px+ | Multi-column |

### Mobile-First 원칙

```
1. 모바일에서 먼저 설계
2. 핵심 기능만 우선
3. 터치 타겟 48px 이상
4. 엄지 영역 고려
```
