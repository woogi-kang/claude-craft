## 3. 팀 구조 옵션

### Option A: 최소 팀 (1-2명)

```
┌─────────────────────────────────────┐
│           Founder/PM                │
│    (기획 + 개발 + 디자인 일부)       │
├─────────────────────────────────────┤
│                                     │
│         풀스택 개발자                │
│    (FE + BE + DevOps)               │
│                                     │
├─────────────────────────────────────┤
│          외주/파트타임               │
│       (디자인, 마케팅)               │
└─────────────────────────────────────┘
```

**장점**: 빠른 의사결정, 낮은 비용
**단점**: 병목, 번아웃 위험
**적합**: 검증 단계, 사이드 프로젝트

### Option B: 밸런스 팀 (3-5명)

```
┌─────────────────────────────────────┐
│              PM/CPO                  │
│         (제품 방향, 우선순위)         │
├──────────────────┬──────────────────┤
│                  │                  │
│   FE Developer   │   BE Developer   │
│                  │                  │
├──────────────────┴──────────────────┤
│            Designer                  │
│         (UI/UX, 브랜딩)              │
└─────────────────────────────────────┘
```

**장점**: 역할 분리, 효율적
**단점**: 커뮤니케이션 오버헤드
**적합**: MVP 후 성장 단계

### Option C: 성장 팀 (5-10명)

```
┌─────────────────────────────────────────────────────────────────┐
│                           CTO / PM                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐          │
│  │   FE Team    │  │   BE Team    │  │ Design Team  │          │
│  │              │  │              │  │              │          │
│  │  Lead (1)    │  │  Lead (1)    │  │  Lead (1)    │          │
│  │  Dev (1-2)   │  │  Dev (1-2)   │  │  UI (1)      │          │
│  └──────────────┘  └──────────────┘  └──────────────┘          │
│                                                                  │
│  ┌──────────────┐  ┌──────────────┐                            │
│  │     QA       │  │   DevOps     │                            │
│  │    (1)       │  │    (0.5)     │                            │
│  └──────────────┘  └──────────────┘                            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 🎯 인터랙티브 가이드

### 작성 전 확인 질문

**Q1. 현재 팀이 있나요?**
- 있음 → 역량 Gap 분석 진행
- 없음 → 필수 역할 정의부터 시작

**Q2. 예산 제약이 있나요?**
- 있음 → 예산 내 최적 구성 설계
- 없음 → 이상적 구성 후 우선순위 조정

**Q3. 외주 활용을 고려하고 있나요?**
- 고려함 → 내부/외부 역할 구분 설계
- 안 함 → 풀타임 채용 계획 수립

### 의사결정 포인트

| 시점 | 확인 내용 | 사용자 프롬프트 |
|------|----------|----------------|
| 역할 정의 | 필수 인력 | "MVP 개발에 꼭 필요한 역할 3개는?" |
| Gap 분석 | 우선순위 | "가장 먼저 채워야 할 역량 Gap은?" |
| 채용 계획 | 시기 | "첫 채용을 언제까지 완료해야 하나요?" |
| 협업 구조 | 의사소통 | "팀 커뮤니케이션 도구와 방식은?" |
