# Information Architecture Skill

서비스의 정보 구조와 네비게이션을 설계합니다.
사용자가 원하는 정보를 쉽게 찾을 수 있는 구조를 만듭니다.

## 출력 템플릿

```markdown
# {Project Name} - 정보 구조 (IA)

## 1. IA Overview

### 설계 원칙

1. **{principle_1}**: {description}
2. **{principle_2}**: {description}
3. **{principle_3}**: {description}

### 사용자 멘탈 모델

```
{mental_model_description}
```

## 2. 사이트맵 (Sitemap)

### 전체 구조

```
{Service Name}
│
├── 🏠 홈
│   └── 메인 대시보드
│
├── 📊 {Section 1}
│   ├── {Page 1-1}
│   ├── {Page 1-2}
│   └── {Page 1-3}
│       ├── {Sub-page 1-3-1}
│       └── {Sub-page 1-3-2}
│
├── 👤 {Section 2}
│   ├── {Page 2-1}
│   ├── {Page 2-2}
│   └── {Page 2-3}
│
├── ⚙️ 설정
│   ├── 계정 설정
│   ├── 알림 설정
│   └── 보안 설정
│
└── ❓ 도움말
    ├── FAQ
    ├── 가이드
    └── 고객 지원
```

### 상세 사이트맵

| Depth | 페이지 | URL | 설명 | 우선순위 |
|-------|--------|-----|------|----------|
| L1 | 홈 | / | 메인 랜딩/대시보드 | 🔴 |
| L1 | {Section 1} | /{section1} | {description} | 🔴 |
| L2 | {Page 1-1} | /{section1}/{page1} | {description} | 🔴 |
| L2 | {Page 1-2} | /{section1}/{page2} | {description} | 🟡 |
| L3 | {Sub-page} | /{section1}/{page1}/{sub} | {description} | 🟢 |

## 3. 네비게이션 구조

### 글로벌 네비게이션 (GNB)

```
┌─────────────────────────────────────────────────────────────────┐
│  [Logo]    [Nav 1]  [Nav 2]  [Nav 3]  [Nav 4]    [User] [CTA]  │
└─────────────────────────────────────────────────────────────────┘
```

| 위치 | 항목 | 링크 | 노출 조건 |
|------|------|------|----------|
| 좌측 | 로고 | / | 항상 |
| 중앙 | {Nav 1} | /{path1} | 항상 |
| 중앙 | {Nav 2} | /{path2} | 로그인 시 |
| 중앙 | {Nav 3} | /{path3} | 로그인 시 |
| 우측 | 프로필 | /profile | 로그인 시 |
| 우측 | CTA 버튼 | /{action} | 비로그인 시 |

### 로컬 네비게이션 (LNB)

**{Section 1} LNB**
```
┌─────────────┐
│ {Section 1} │
├─────────────┤
│ • Page 1-1  │ ← Active
│ • Page 1-2  │
│ • Page 1-3  │
│   ├ Sub 1   │
│   └ Sub 2   │
└─────────────┘
```

### 유틸리티 네비게이션

| 항목 | 위치 | 기능 |
|------|------|------|
| 검색 | 헤더 | 전체 검색 |
| 알림 | 헤더 | 알림 확인 |
| 설정 | 프로필 드롭다운 | 설정 접근 |
| 로그아웃 | 프로필 드롭다운 | 로그아웃 |

### 푸터 네비게이션

| 섹션 | 링크 |
|------|------|
| 회사 정보 | 소개, 채용, 블로그 |
| 서비스 | 기능, 가격, 가이드 |
| 지원 | FAQ, 문의, 상태 |
| 법적 | 이용약관, 개인정보처리방침 |
```

```
### URL 네이밍 규칙

| 규칙 | 예시 |
|------|------|

| 패턴 | 예시 | 설명 |
|------|------|------|
| /{resource} | /projects | 목록 |

| /{resource}/:id/edit | /projects/123/edit | 수정 폼 |
```

## 🎯 인터랙티브 가이드

### 작성 전 확인 질문

**Q1. 서비스의 핵심 기능이 정의되었나요?**
- 정의됨 → 기능 기반 사이트맵 설계
- 미정의 → "사용자가 가장 자주 할 행동 3가지는?"

**Q2. 권한별 접근이 다른가요?**
- 다름 → 권한별 IA 구분 필요
- 동일 → 단일 IA 구조로 진행

**Q3. 모바일과 웹의 구조가 다른가요?**
- 다름 → 플랫폼별 네비게이션 설계
- 동일 → 반응형 IA로 통합 설계

### 의사결정 포인트

| 시점 | 확인 내용 | 사용자 프롬프트 |
|------|----------|----------------|
| 사이트맵 | 깊이 | "3단계 이내로 원하는 페이지에 도달 가능한가요?" |
| 네비게이션 | 우선순위 | "GNB에 들어갈 가장 중요한 5개 메뉴는?" |
| URL 구조 | 일관성 | "URL만 보고 페이지 내용을 예상할 수 있나요?" |
| 검증 | 카드소팅 | "사용자 테스트로 IA를 검증할 예정인가요?" |
