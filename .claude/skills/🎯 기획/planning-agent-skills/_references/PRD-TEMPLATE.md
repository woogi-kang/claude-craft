## PRD 작성 팁

### Good PRD 특징
```
✅ 명확한 문제 정의
✅ 측정 가능한 성공 기준
✅ 구체적인 사용자 스토리
✅ 명확한 범위 (포함/제외)
✅ 검증 가능한 Acceptance Criteria
✅ 리스크와 가정 명시
```

### Bad PRD 특징
```
❌ "사용하기 쉬워야 한다" (모호함)
❌ "빨라야 한다" (측정 불가)
❌ "모든 사용자를 위해" (타겟 불명확)
❌ 기술 구현 방식 지정 (How vs What)
❌ 변경 이력 없음
```
