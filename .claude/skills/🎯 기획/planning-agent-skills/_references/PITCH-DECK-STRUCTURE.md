# Pitch Deck Structure Reference

## 슬라이드 구조

### 표준 12장 구조

```
1.  Cover           - 첫인상
2.  Problem         - 공감
3.  Solution        - 해결책
4.  Market          - 기회
5.  Product         - 데모
6.  Business Model  - 수익화
7.  Traction        - 검증
8.  Go-to-Market    - 성장
9.  Competition     - 차별화
10. Team            - 신뢰
11. Financials      - 계획
12. Ask             - 요청
```

### 간단 버전 (5-7장)

```
1. Cover + Problem
2. Solution + Product
3. Market + Traction
4. Team
5. Ask
```

## 각 슬라이드 가이드

### 1. Cover

```
목적: 기억에 남는 첫인상

포함 요소:
─────────────────────────────────────
□ 로고
□ 회사명
□ 태그라인 (한 문장)
□ 발표자 정보

예시:
┌─────────────────────────────────────┐
│                                     │
│            [LOGO]                   │
│                                     │
│         Company Name                │
│    "AI로 업무를 자동화하다"          │
│                                     │
│       홍길동 | CEO                   │
│       contact@company.com           │
│                                     │
└─────────────────────────────────────┘
```

### 2. Problem

```
목적: 투자자가 공감할 수 있는 문제 제시

포함 요소:
─────────────────────────────────────
□ 구체적인 문제 설명
□ 영향받는 사람/비즈니스
□ 현재 해결책의 한계
□ 문제의 비용/규모

구조 옵션:

A) 스토리텔링
"김대리는 매일 3시간을 반복 업무에..."

B) 통계
"87%의 직장인이 반복 업무로 고통"

C) 인용
"'매일 같은 일의 반복이 지겹다' - 실제 인터뷰"
```

### 3. Solution

```
목적: 명확하고 간결한 해결책

포함 요소:
─────────────────────────────────────
□ How it works (3단계 이하)
□ 핵심 가치 제안
□ Before/After 비교

구조:

     Problem              Solution
─────────────────────────────────────
수작업 3시간    →    자동화 10분
오류 발생       →    99.9% 정확도
비용 100만원    →    비용 10만원
```

### 4. Market (TAM/SAM/SOM)

```
목적: 충분히 큰 시장 기회 증명

포함 요소:
─────────────────────────────────────
□ TAM (전체 시장)
□ SAM (접근 가능 시장)
□ SOM (목표 시장)
□ 성장률
□ Why Now (타이밍)

시각화:

          TAM: $100B
         ┌─────────────────┐
         │                 │
         │   SAM: $10B     │
         │  ┌───────────┐  │
         │  │           │  │
         │  │ SOM: $1B  │  │
         │  │ ┌───────┐ │  │
         │  │ │       │ │  │
         │  │ └───────┘ │  │
         │  └───────────┘  │
         └─────────────────┘
```

### 5. Product

```
목적: 실제 제품이 어떻게 작동하는지

포함 요소:
─────────────────────────────────────
□ 스크린샷 또는 데모
□ 핵심 기능 (3개 이하)
□ 사용자 흐름

팁:
- 실제 작동하는 제품 보여주기
- 핵심 기능에만 집중
- 너무 많은 기능 나열 금지
```

### 6. Business Model

```
목적: 어떻게 돈을 버는지 설명

포함 요소:
─────────────────────────────────────
□ 수익 모델
□ 가격 구조
□ 단위 경제 (LTV, CAC)
□ Gross Margin

예시:

수익 모델: SaaS 구독
─────────────────────────────────────
Basic:      $29/월     (개인)
Pro:        $99/월     (팀)
Enterprise: Custom     (기업)

단위 경제:
LTV:        $1,200
CAC:        $300
LTV/CAC:    4x
Payback:    4개월
```

### 7. Traction

```
목적: 검증된 성과와 모멘텀

포함 요소:
─────────────────────────────────────
□ 핵심 지표 (매출, 사용자, 성장률)
□ 성장 그래프
□ 주요 고객/파트너
□ 마일스톤

성장 그래프:

Revenue ($K)
│
50├                     ●
│                    ╱
30├                 ●
│                ╱
20├             ●
│          ╱
10├       ●
│      ╱
5├   ●
└─────────────────────────
  Jan  Mar  Jun  Sep  Dec

+200% MoM 성장
```

### 8. Go-to-Market

```
목적: 어떻게 성장할 것인지

포함 요소:
─────────────────────────────────────
□ 채널 전략
□ 성장 레버
□ 파트너십

예시:

채널 믹스:
Organic (SEO, 콘텐츠):    40%
Paid (Google, Meta):      30%
Referral:                 20%
Partnership:              10%
```

### 9. Competition

```
목적: 경쟁 환경과 우위 설명

포함 요소:
─────────────────────────────────────
□ 경쟁사 분석
□ 차별화 포인트
□ 경쟁 우위 (Moat)

2x2 매트릭스:

             Feature Rich
                 │
         ┌───────┼───────┐
         │   A   │ [Us]★ │
Expensive│───────┼───────│Affordable
         │   B   │   C   │
         └───────┼───────┘
                 │
             Feature Light

차별화: AI 기반 자동화 + 50% 저렴
```

### 10. Team

```
목적: 왜 이 팀이 성공할 수 있는지

포함 요소:
─────────────────────────────────────
□ 핵심 멤버 소개
□ 관련 경험
□ 어드바이저 (있다면)

예시:

┌─────────────┬─────────────┬─────────────┐
│ [사진]      │ [사진]      │ [사진]      │
│ 홍길동      │ 김철수      │ 이영희      │
│ CEO        │ CTO        │ CPO        │
│             │             │             │
│ Ex-Google   │ Ex-Meta     │ Ex-Kakao    │
│ 10년 경력   │ 8년 경력    │ 7년 경력    │
└─────────────┴─────────────┴─────────────┘

Founder-Market Fit:
"팀 전체가 이 문제를 직접 경험했습니다"
```

### 11. Financials

```
목적: 재무 전망과 자금 사용 계획

포함 요소:
─────────────────────────────────────
□ 매출/비용 전망 (3년)
□ 자금 사용 계획
□ 주요 가정

예시:

재무 전망 (단위: 억원)
         Year 1   Year 2   Year 3
매출       5       20       80
비용       8       15       40
Net       -3        5       40

자금 사용:
Product/Engineering:   60%
Sales/Marketing:       30%
Operations:            10%
```

### 12. Ask

```
목적: 명확한 요청

포함 요소:
─────────────────────────────────────
□ 투자 요청 금액
□ 라운드 정보
□ 자금 용도
□ 다음 마일스톤
□ 연락처

예시:

┌─────────────────────────────────────┐
│                                     │
│    투자 요청: 30억원 (Series A)      │
│                                     │
│    자금 용도:                        │
│    - 제품 개발 강화                  │
│    - 영업팀 확장                     │
│    - 마케팅 확대                     │
│                                     │
│    다음 마일스톤:                    │
│    - MAU 10만 달성                   │
│    - 연 매출 100억                   │
│                                     │
│    contact@company.com              │
│                                     │
└─────────────────────────────────────┘
```

## 디자인 원칙

### Do

```
✅ 한 슬라이드 한 메시지
✅ 큰 폰트 (24pt+)
✅ 충분한 여백
✅ 일관된 디자인
✅ 고품질 이미지
✅ 명확한 차트
```

### Don't

```
❌ 텍스트 빽빽이
❌ 작은 폰트
❌ 불필요한 애니메이션
❌ 너무 많은 색상
❌ 저해상도 이미지
❌ 복잡한 차트
```

```
Traction:        1분
GTM:             30초

Financials:      30초
Ask:             30초
```

### 예상 질문 준비

```
□ 경쟁사와의 차이점은?
□ 왜 지금 이 시장인가?
□ CAC/LTV는?
□ 팀의 강점은?
□ 가장 큰 리스크는?
□ Exit 전략은?
□ 기존 투자자는 누구?
```
