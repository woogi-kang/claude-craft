# Pricing Models Reference

## 가격 모델 유형

### 1. Subscription (구독)

```
특징: 정기적 반복 결제
적합: SaaS, 콘텐츠, 서비스

장점                          단점
─────────────────────────────────────────
예측 가능한 수익               초기 진입 장벽
높은 LTV                      이탈률 관리 필요
안정적 캐시플로우              가격 인상 어려움
```

**예시 구조**
| Tier | 가격 | 포함 |
|------|------|------|
| Basic | $9/월 | 기본 기능 |
| Pro | $29/월 | + 고급 기능 |
| Enterprise | Custom | + 전담 지원 |

### 2. Freemium

```
특징: 무료 + 유료 프리미엄
적합: 네트워크 효과 제품, 바이럴

장점                          단점
─────────────────────────────────────────
낮은 진입 장벽                 전환율 도전 (2-5%)
바이럴 효과                   무료 사용자 비용
넓은 퍼널                     가치 차별화 어려움
```

**전환 전략**
| 제한 | 무료 | 유료 |
|------|------|------|
| 기능 | 기본 | 전체 |
| 용량 | 1GB | 100GB |
| 사용자 | 3명 | 무제한 |
| 지원 | 셀프 | 프리미엄 |

### 3. Usage-Based (사용량 기반)

```
특징: 사용한 만큼 지불
적합: API, 클라우드, 인프라

장점                          단점
─────────────────────────────────────────
공정한 가격                   예측 어려움
진입 장벽 낮음                수익 변동성
성장과 수익 연동              비용 폭발 위험
```

**예시 구조**
| 단위 | 가격 |
|------|------|
| API 호출 1K | $0.01 |
| 스토리지 1GB | $0.05/월 |
| 대역폭 1GB | $0.10 |

### 4. Tiered Pricing (단계별)

```
특징: 사용량 구간별 가격
적합: 다양한 규모의 고객

장점                          단점
─────────────────────────────────────────
명확한 업그레이드 경로         복잡할 수 있음
다양한 고객 수용               경계 효과
```

**예시 구조**
| Tier | 범위 | 가격 |
|------|------|------|
| Starter | 0-1,000 | $19/월 |
| Growth | 1,001-10,000 | $49/월 |
| Scale | 10,001-50,000 | $99/월 |
| Enterprise | 50,001+ | Custom |

### 5. Per-Seat (사용자당)

```
특징: 사용자 수 × 단가
적합: B2B SaaS, 협업 도구

장점                          단점
─────────────────────────────────────────
이해하기 쉬움                 Shadow IT 위험
성장과 수익 연동              공유 계정 유인
```

**예시 구조**
| Plan | 가격/유저 | 최소 |
|------|----------|------|
| Team | $10/월 | 5명 |
| Business | $25/월 | 10명 |
| Enterprise | Custom | 50명+ |

### 6. One-Time (일회성)

```
특징: 단일 구매
적합: 소프트웨어, 템플릿, 코스

장점                          단점
─────────────────────────────────────────
단순한 거래                   예측 어려운 수익
높은 마진                     재구매 유도 필요
```

### 7. Hybrid (혼합)

```
특징: 여러 모델 조합
적합: 복잡한 제품, Enterprise

예시: 기본료 + 사용량 + 애드온
```

## 가격 전략

### 1. Penetration (침투)

```
전략: 낮은 가격으로 시장 진입
목표: 빠른 시장 점유율 확보
리스크: 수익성, 가격 인상 어려움

적합 상황:
- 경쟁이 치열한 시장
- 네트워크 효과가 중요한 제품
- 충분한 자금이 있는 경우
```

### 2. Skimming (고가)

```
전략: 높은 가격에서 시작, 점진적 인하
목표: 얼리어답터에서 최대 마진
리스크: 경쟁사 진입

적합 상황:
- 혁신적인 제품
- 명확한 차별점
- 가격 민감도 낮은 고객
```

### 3. Value-Based (가치 기반)

```
전략: 고객이 얻는 가치 기준
목표: 가치와 가격 정렬

공식:
가격 = 고객이 얻는 가치 × 가치 점유율 (10-30%)
```

### 4. Competitive (경쟁 기준)

```
전략: 경쟁사 가격 기준
옵션:
- 동일 가격 (경쟁)
- 낮은 가격 (저가 전략)
- 높은 가격 (프리미엄)
```

## 가격 심리학

### 앵커링 (Anchoring)

```
효과: 첫 번째 가격이 기준점 역할

전략:
- 높은 가격을 먼저 보여주기
- "원래 $99, 지금 $49"
- 가장 비싼 플랜을 먼저 배치
```

### 참조 가격

```
효과: 비교 대상 제공

전략:
- 3개 플랜 (미끼 효과)
- 경쟁사 가격 비교
- ROI 계산기 제공
```

### 매직 넘버

```
$9.99 vs $10.00
$97 vs $100
$29, $49, $99 (익숙한 가격대)
```

### Decoy Effect (미끼 효과)

```
     Small    Medium    Large
가격   $10      $15       $18
용량   100MB    200MB     500MB

Medium이 미끼 → Large 선택 유도
```

## 가격 최적화

### A/B 테스트

```
테스트 가능 항목:
- 가격 포인트 ($9 vs $12)
- 플랜 구조 (2개 vs 3개)
- 플랜 이름
- 기능 묶음
- 할인 표시 방식
```

### Van Westendorp PSM

```
질문:
1. 너무 싸서 품질이 의심되는 가격?
2. 저렴하다고 느끼는 가격?
3. 비싸다고 느끼는 가격?
4. 너무 비싸서 고려하지 않는 가격?

→ 최적 가격대 도출
```

## 플랜 설계

### 좋은 플랜 구조

```
✅ 명확한 차별화
✅ 업그레이드 경로
✅ 가치 기반 네이밍
✅ 3개 이하 옵션
✅ 추천 플랜 표시
```

### 플랜 이름 예시

| 패턴 | 예시 |
|------|------|
| 성장 단계 | Starter, Growth, Enterprise |
| 사용자 유형 | Personal, Team, Business |
| 기능 레벨 | Basic, Pro, Premium |
| 크기 | Small, Medium, Large |

### Feature Gating

| 전략 | 설명 | 예시 |
|------|------|------|
| 기능 제한 | 일부 기능 잠금 | "Pro에서 사용 가능" |
| 용량 제한 | 저장/사용량 제한 | "5GB까지 무료" |
| 사용자 제한 | 팀 규모 제한 | "3명까지 무료" |
| 시간 제한 | 기간 제한 | "14일 무료 체험" |
| 품질 제한 | 출력 품질 제한 | "워터마크 포함" |

## 단위 경제

### 핵심 지표

```
LTV (고객 생애 가치)
= ARPU × 평균 구독 기간
= ARPU / 월간 이탈률

CAC (고객 획득 비용)
= 마케팅 비용 / 신규 고객 수

LTV/CAC 목표: > 3x
Payback 목표: < 12개월
```

### 건전한 단위 경제

| 지표 | 좋음 | 나쁨 |
|------|------|------|
| LTV/CAC | > 3 | < 1 |
| Payback | < 12개월 | > 24개월 |
| Gross Margin | > 70% | < 50% |
| Net Revenue Retention | > 100% | < 80% |

## 가격 변경

### 가격 인상 전략

```
1. 기존 고객 보호 (Grandfather)
2. 사전 충분한 공지
3. 가치 증가와 함께
4. 단계적 인상
5. 옵션 제공
```

### 커뮤니케이션

```
Good: "더 나은 서비스를 위해 가격을 조정합니다.
      기존 고객님은 12개월간 현재 가격으로..."

Bad:  "가격이 인상됩니다."
```
