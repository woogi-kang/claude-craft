# 이력서 (Resume) 형식

1-2페이지 분량의 간결한 형식. 채용 담당자가 빠르게 훑어볼 수 있도록 구성.

## 구조

```markdown
# [이름]

[직함/희망 포지션]

📧 [이메일] | 📱 [전화번호] | 🔗 [GitHub/LinkedIn]

---

## Summary

[2-3문장으로 핵심 역량과 경력 요약]

---

## Experience

### [회사명] | [직급]
**YYYY.MM - YYYY.MM** | [위치]

- 성과 중심 bullet point 1 (정량적 수치)
- 성과 중심 bullet point 2 (정량적 수치)
- 성과 중심 bullet point 3 (정량적 수치)

### [회사명] | [직급]
**YYYY.MM - YYYY.MM** | [위치]

- 성과 중심 bullet point 1
- 성과 중심 bullet point 2

---

## Skills

**Languages**: Python, Dart, JavaScript
**Frameworks**: Flutter, FastAPI, React
**Tools**: Git, Docker, AWS

---

## Education

**[학위]** | [학교명] | YYYY
```

## 핵심 원칙

### 1. 6초 규칙
채용 담당자는 평균 6초간 이력서를 훑어봄.
→ 상단 1/3에 가장 중요한 정보 배치

### 2. One Page Rule
- 경력 10년 미만: 1페이지
- 경력 10년 이상: 2페이지 (최대)

### 3. Bullet Point 규칙
- 회사당 3-5개 bullet
- 각 bullet은 1-2줄
- [동사] + [업무] + [성과/수치] 구조

## Bullet Point 예시

❌ Bad:
```
- 웹 개발 담당
- 팀원들과 협업
- 다양한 프로젝트 참여
```

✅ Good:
```
- MAU 50만 서비스의 프론트엔드 개발 리드, 페이지 로딩 속도 40% 개선
- 5명 팀 리드로 3개 프로젝트 동시 진행, 전체 일정 준수율 95% 달성
- CI/CD 파이프라인 구축으로 배포 시간 2시간 → 15분 단축
```

## ATS 최적화

ATS(Applicant Tracking System) 통과를 위한 규칙:

1. **표준 섹션명 사용**: Experience, Skills, Education
2. **키워드 매칭**: 채용 공고의 기술 스택 포함
3. **특수문자 최소화**: 심플한 포맷
4. **PDF 저장**: 레이아웃 유지
