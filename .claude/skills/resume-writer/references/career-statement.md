# 경력기술서 (Career Statement) 형식

상세한 경력 기술이 필요한 경우 사용. 기술 면접, 경력직 채용 시 활용.

## 구조

```markdown
# 경력기술서

## 인적사항

| 항목 | 내용 |
|------|------|
| 성명 | [이름] |
| 연락처 | [전화번호] |
| 이메일 | [이메일] |
| GitHub | [URL] (선택) |
| 포트폴리오 | [URL] (선택) |

---

## 경력 요약

총 경력: X년 Y개월
핵심 역량: [3-5개 키워드]

---

## 경력 사항

### [회사명] (YYYY.MM - YYYY.MM)

**직급/직책**: [직급] | [부서/팀]

**담당 업무**:
[1-2문장으로 전체 역할 요약]

#### 프로젝트 1: [프로젝트명]

**기간**: YYYY.MM - YYYY.MM
**규모**: 팀원 N명 / 예산 N원 / 사용자 N명
**역할**: [구체적 역할]

**배경 및 목표**:
[왜 이 프로젝트를 했는지, 해결하려던 문제]

**주요 성과**:
- 성과 1: [정량적 수치 포함]
- 성과 2: [정량적 수치 포함]
- 성과 3: [정량적 수치 포함]

**수행 업무**:
- [구체적 업무 내용]
- [구체적 업무 내용]

**기술 스택**:
[기술 | 프레임워크 | 도구]

**문제 해결 사례** (선택):
- 문제: [발생한 문제]
- 해결: [해결 방법]
- 결과: [개선된 결과]

---

## 기술 스택

### Expert (3년 이상)
- 기술 1, 기술 2

### Proficient (1-3년)
- 기술 3, 기술 4

### Familiar (1년 미만)
- 기술 5, 기술 6

---

## 교육 및 자격

- [학위/자격증] | [기관] | YYYY.MM
- [학위/자격증] | [기관] | YYYY.MM
```

## 작성 팁

1. **프로젝트당 1페이지 이내**: 핵심만 간결하게
2. **역순 정렬**: 최신 경력부터
3. **정량적 성과 필수**: 모든 프로젝트에 숫자 포함
4. **기술 숙련도 구분**: Expert/Proficient/Familiar

## 분량 가이드

| 경력 | 권장 분량 |
|------|----------|
| 1-3년 | 2-3페이지 |
| 3-7년 | 3-5페이지 |
| 7년+ | 5-7페이지 |

최근 3년 경력에 70% 분량 할애 권장.
