## 2. 컨센서스 이슈 (양측 동의)

### ⚠️ MAJOR: URL/라우팅 구조 중복 및 확장성 문제
**합의율: 100%**

| 관점 | Claude | Gemini |
|------|--------|--------|
| 이슈 | `/projects/:id`와 `/projects/:id/overview` 중복 | 하드코딩된 네비게이션이 확장성 제한 |
| 위치 | URL 구조, 사이트맵 | LNB Navigation |
| 제안 | 표준 RESTful 패턴 적용, 리다이렉트 명시 | 동적 네비게이션 시스템, 플러그인 구조 고려 |

**권장 조치**:
- `/projects/:id` → `/projects/:id/overview` 리다이렉트 명시
- 네비게이션 항목을 설정 기반으로 동적 생성하여 확장성 확보

---

### 💡 SUGGESTION: 대시보드 페이지네이션/필터링 부재
**합의율: 100%**

| 관점 | Claude | Gemini |
|------|--------|--------|
| 이슈 | 대시보드에 페이지네이션 언급 없음 | 프로젝트 증가 시 스케일 전략 부재 |
| 제안 | 프로젝트 수 증가 대비 필터/검색/페이징 추가 | 가상 스크롤 또는 필터 기반 탐색 도입 |

**권장 조치**: 대시보드에 프로젝트 검색, 필터(상태, 날짜), 페이지네이션 추가 계획 수립

## 3. Claude 단독 발견 사항

### ⚠️ MAJOR: 비밀번호 재설정 토큰 처리 미정의
- **위치**: /reset-password 페이지
- **이슈**: URL 토큰 처리 방식 (쿼리 파라미터 vs 경로) 미정의
- **제안**: `/reset-password?token=xxx` 형식 명시 및 만료 처리 정의

### ⚠️ MAJOR: 프로젝트 간 전환 패턴 부재
- **위치**: 프로젝트 네비게이션
- **이슈**: 프로젝트 내에서 다른 프로젝트로 전환하는 방법 미정의
- **제안**: GNB 프로젝트 드롭다운 또는 전역 프로젝트 스위처 추가

### 📝 MINOR: Settings 네이밍 혼란
- **위치**: 사이트맵
- **이슈**: `/settings` (계정)과 `/projects/:id/settings` (프로젝트) 혼동 가능
- **제안**: `/account/*` 또는 명확한 네이밍 컨벤션 적용

### 📝 MINOR: 쿼리 파라미터 조합 규칙 미정의
- **위치**: URL 구조
- **이슈**: 다중 필터 조합 시 행동 미정의
- **제안**: `?platform=android&priority=high` 조합 규칙 문서화

### 📝 MINOR: 401 Unauthorized 처리 누락
- **위치**: 에러 페이지
- **이슈**: 403만 있고 401 (인증 필요) 처리 없음
- **제안**: 401 → 로그인 페이지 리다이렉트 로직 추가

## 4. Gemini 단독 발견 사항

### 💡 SUGGESTION: TC 상세 페이지 액션 라우팅
- **위치**: /projects/:id/testcases/:tcId
- **이슈**: TC 수정/삭제 등 액션이 모달인지 별도 라우트인지 미정의
- **제안**: `/testcases/:tcId/edit` 또는 인라인 편집 방식 명시

### 💡 SUGGESTION: 분석/추적성 페이지 구조 미정의
- **위치**: Analytics, Traceability 페이지
- **이슈**: [P1], [P2] 표시만 있고 구체적 구조 없음
- **제안**: MVP 범위가 아니더라도 기본 와이어프레임 수준 정의 권장

| Claude | Gemini |
|--------|--------|

## 6. 개선 우선순위

### P0 (즉시)
1. URL 중복 해결 (리다이렉트 규칙 명시)
2. 비밀번호 재설정 토큰 처리 방식 정의

### P1 (MVP 전)
3. 대시보드 페이지네이션/필터링 설계
4. 프로젝트 스위처 UI 패턴 정의
5. 401 Unauthorized 처리 추가

### P2 (MVP 후)
6. TC 액션 라우팅 방식 확정
7. Analytics/Traceability 페이지 기본 구조 정의
8. 네비게이션 동적 확장 시스템 설계

## 7. 메타 분석

**Claude vs Gemini 관점 차이:**
- **Claude**: 엣지 케이스 중심 (토큰 처리, 401, 쿼리 파라미터 조합)
- **Gemini**: 스케일 및 확장성 중심 (동적 네비게이션, 대시보드 스케일)

**특이사항**: 두 LLM 모두 Critical 이슈 없음 - IA 문서가 전반적으로 양호함

**리뷰 신뢰도**: 높음 (상호 보완적 관점으로 포괄적 검토 완료)
