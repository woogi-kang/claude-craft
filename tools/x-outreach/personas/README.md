# X-Outreach Personas 운영 가이드

이 문서는 `/tools/x-outreach/personas` 디렉토리의 모든 파일을 실제 에이전트/툴 운영에 연결하기 위한 기준 문서입니다.

## 1) 목적

이 페르소나 체계의 목적은 아래 3가지입니다.

- 계정별 말투/어투를 분리해서 자연스러운 운영 유지
- 계정별 검색 키워드를 분리해 타겟을 차등 수집
- 결과를 `persona_id` 단위로 추적해 성과 최적화

## 2) 디렉토리 구성

- `base_policy.md`: 모든 계정 공통 안전/표현 정책
- `p01_price.md`: 가격 분석형 페르소나
- `p02_beginner_guide.md`: 입문 가이드형 페르소나
- `p03_procedure_explainer.md`: 시술 설명형 페르소나
- `p04_risk_care.md`: 리스크/사후 케어형 페르소나
- `p05_lifestyle.md`: 라이프스타일형 페르소나
- `style_lexicon.yaml`: 페르소나별 선호 어미/어휘/금지어
- `accounts.yaml`: 계정 ↔ 페르소나 ↔ 키워드 프로필 매핑
- `keyword_matrix.yaml`: 페르소나별 키워드 세트(검증 지표 포함)
- `keyword_validation_2026-02-23.md`: 키워드 실측 검증 리포트

## 3) 페르소나 개요

### p01_price (가격 분석형)

- 역할: 가격/구성/추가비용 중심 비교
- 스타일: 담백, 숫자 중심, 과장 없는 설명
- 키워드 성격: 요금/가성비/가격대/추가금

주요 키워드 번역:

- `韓国 美容 コスパ` → 한국 미용 가성비
- `韓国 美容 料金` → 한국 미용 요금
- `韓国 施術 価格` → 한국 시술 가격
- `韓国 追加料金` → 한국 추가 비용

### p02_beginner_guide (입문 가이드형)

- 역할: 초보 사용자 동선/준비/체크리스트 안내
- 스타일: 친절, 쉽게 설명, 불안 완화
- 키워드 성격: 처음/추천/어디/일본어 대응

주요 키워드 번역:

- `韓国 皮膚科 初めて` → 한국 피부과 첫 방문
- `韓国 皮膚科 おすすめ` → 한국 피부과 추천
- `初渡韓 美容` → 첫 방한 미용
- `日本語対応 皮膚科 韓国` → 일본어 대응 한국 피부과

### p03_procedure_explainer (시술 설명형)

- 역할: 시술 간 차이, 목적별 선택 가이드
- 스타일: 중립, 비교 구조, 설명형 문장
- 키워드 성격: 시술명/다운타임/효과

주요 키워드 번역:

- `ポテンツァ` → 포텐자
- `レーザー治療` → 레이저 치료
- `ピコレーザー` → 피코레이저
- `ピコトーニング` → 피코토닝
- `リジュラン 韓国` → 한국 리쥬란

### p04_risk_care (리스크 케어형)

- 역할: 불만/부작용/분쟁 맥락의 공감 + 체크리스트
- 스타일: 공감 우선, 신중, 과장 금지
- 키워드 성격: 실패/후회/트러블/추가금

주요 키워드 번역:

- `韓国 追加料金` → 한국 추가 비용
- `韓国 皮膚科 トラブル` → 한국 피부과 트러블
- `韓国 皮膚科 失敗` → 한국 피부과 실패
- `韓国 施術 後悔` → 한국 시술 후회

### p05_lifestyle (라이프스타일형)

- 역할: 일상 톤에서 자연스럽게 뷰티 대화 연결
- 스타일: 캐주얼, 인간적, 부담 없는 질문형
- 키워드 성격: 후기/체감/일상형 표현

주요 키워드 번역:

- `韓国 美容 よかった` → 한국 미용 만족 후기
- `#韓国美容` → #한국미용
- `渡韓美容 レポ` → 방한 미용 리포트
- `韓国美容` → 한국 미용

## 4) 키워드 파일 해석 규칙

### `keyword_matrix.yaml`

각 프로필은 아래 2층 구조를 사용합니다.

- `primary`: 기본 수집에 우선 사용
- `secondary`: 결과가 적거나 편향될 때 보강용

지표 의미:

- `recent_30d`: 샘플링된 상위 트윗 중 최근 30일 내 게시 수
- 값이 0이어도 완전 삭제하지 않고 `secondary`로 보관 가능

### `accounts.yaml`

핵심 필드:

- `account_id`: 실제 운영 계정 식별자
- `persona_id`: 문체/내용 페르소나
- `keyword_profile`: 검색 키워드 세트 프로필
- `global_fallback_keywords`: 어떤 프로필에서도 최소 검색량을 보장하는 공통 백업 키워드

## 5) 에이전트 적용 순서 (권장)

각 메시지 생성 시 아래 순서로 컨텍스트를 조합합니다.

1. `base_policy.md` 로드
2. `accounts.yaml`에서 `account_id` 매핑 조회
3. `p0x_*.md` 로드 (말투/콘텐츠 정책)
4. `style_lexicon.yaml` 로드 (선호 어미/금지어)
5. 스테이지별 프롬프트(`reply`/`dm`/`post`)와 결합
6. 생성 후 규칙 검증(길이/URL/금지어/반복도)

프롬프트 조합 예시(개념):

```text
[base_policy]
[persona_profile]
[style_lexicon for persona]
[stage_prompt]
[user/tweet context]
```

## 6) 검색 키워드 적용 순서 (권장)

계정별 검색 실행 시:

1. `keyword_profile.primary`에서 4~6개 랜덤 샘플
2. `keyword_profile.secondary`에서 1~2개 보강
3. 결과 부족 시 `global_fallback_keywords`에서 2~3개 추가
4. 최근 24~72시간 중복 키워드는 우선순위 하향

권장 운영 파라미터:

- 1회 실행당 계정별 키워드 6~9개
- 동일 키워드 연속 사용 최소화
- 계정 간 동일 키워드 동시 사용 비율 제한 (예: 40% 이하)

## 7) 생성 검증 체크리스트

### 공통

- `base_policy` 위반 없음
- 금지어(`style_lexicon`) 미포함
- 같은 계정 최근 N개 메시지와 고유성 확보

### Reply

- 상대 맥락 1개 이상 반영
- 정보 과밀하지 않음(핵심 1~2개)
- 질문형 CTA로 마무리

### DM

- 첫 DM URL 없음
- 과도한 판매/압박 문구 없음
- 캐릭터 길이 제한 준수 (`accounts.yaml > stage_overrides`)

### Post

- 페르소나 톤 일관성 유지
- 주제 중복 과다 방지
- 인간적인 문장 흐름 유지

## 8) 로깅/메타데이터 규칙

생성 결과에 최소 아래 메타데이터를 남기는 것을 권장합니다.

- `account_id`
- `persona_id`
- `persona_version`
- `keyword_profile`
- `stage` (`reply|dm|post`)
- `style_signature` (짧은 해시 또는 라벨)

## 9) 월간 유지보수 루틴

- 월 1회: `keyword_validation_YYYY-MM-DD.md` 신규 생성
- 월 1회: `keyword_matrix.yaml`의 `primary/secondary` 재분류
- 주 1회: 페르소나별 성과 비교(응답률/차단률/전환률)
- 필요 시: 저성과 키워드 교체, 고성과 키워드 확장

## 10) 변경 규칙

파일 수정 시 아래 원칙을 지킵니다.

- `version` 값 증가 또는 `validated_on` 날짜 갱신
- 변경 이유를 커밋 메시지에 명시
- `accounts.yaml`과 `keyword_matrix.yaml` 동시 정합성 확인

---

필수 운영 요약:

- 문체는 `persona md`에서,
- 금지/선호 어휘는 `style_lexicon.yaml`에서,
- 검색 분리는 `accounts.yaml + keyword_matrix.yaml`에서,
- 안전은 항상 `base_policy.md`에서 강제한다.
